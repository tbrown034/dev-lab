00 The Syllabus — React + Frameworks Deep Dive

You already know React. You've built apps with Next.js, written TypeScript, styled with Tailwind. This course isn't about teaching you useState. It's about teaching you why useState works the way it does — and everything that follows from that understanding.

Who This Course Is For

This course is built for a specific person: someone who can build React apps but wants to understand what's happening beneath the surface. You know the syntax. You want the mental models.

Think of it like investigative journalism. You've been reporting on React — using it, quoting its documentation, following its conventions. Now you're going to investigate it. Pull back the curtain. Read the source code. Understand the architecture decisions that shape everything you build.

The goal isn't more syntax. It's the ability to look at any React problem — a performance issue, a hydration mismatch, a state management decision — and reason about it from first principles instead of Googling stack traces.

What You'll Learn

Phase 1: React Internals (Lessons 01-04)

The foundation. How React actually works under the hood.

  • Lesson 01 — The React Mental Model: React as a rendering engine. Why "UI = f(state)" is the most important equation in frontend development. Components as pure functions of their props. The declarative paradigm and what it costs you.
  • Lesson 02 — Fiber & Reconciliation: The virtual DOM is a marketing term. What React actually maintains is a fiber tree. You'll learn how reconciliation works, why keys matter at an algorithmic level, and what "concurrent" rendering actually means.
  • Lesson 03 — Hooks Deep Dive: Hooks aren't magic — they're array slots on a fiber node. You'll trace through the implementation of useState and useEffect, understand why the rules of hooks exist, and build custom hooks that compose cleanly.
  • Lesson 04 — State Management: Context, Zustand, Jotai, and the signals debate. When global state helps, when it hurts, and the architectural patterns that prevent state from becoming a mess.

Phase 2: The Server Revolution (Lessons 05-08)

React Server Components changed everything. This phase covers the new paradigm.

  • Lesson 05 — Server Components: The mental model shift from "everything runs in the browser" to "some code never ships to the client." The serialization boundary. Why RSC exists and what problems it solves.
  • Lesson 06 — Next.js App Router: The file system as your routing API. Layouts, loading states, error boundaries, parallel routes. How App Router implements the RSC vision.
  • Lesson 07 — Data Fetching: Server actions, React Query, SWR, streaming with Suspense. The decision tree for choosing the right approach.
  • Lesson 08 — Performance: When React.memo helps (rarely). When useMemo helps (sometimes). Code splitting, bundle analysis, and the React Profiler. Measuring before optimizing.

Phase 3: Production Patterns (Lessons 09-12)

Building real applications. Testing, forms, framework choices, and a capstone.

  • Lesson 09 — Forms & Validation: Why controlled inputs exist. React Hook Form for performance. Zod for type-safe validation. Server-side validation that doesn't duplicate client logic.
  • Lesson 10 — Testing: Testing Library's philosophy (test behavior, not implementation). MSW for API mocking. Playwright for E2E. What to test and what not to.
  • Lesson 11 — Framework Comparison: Next.js vs Remix vs Astro vs SvelteKit. Not "which is best" but "which architecture fits which problem."
  • Lesson 12 — Capstone: Build a full-stack data journalism dashboard. Next.js 16, RSC, streaming data, D3 visualizations, authentication, and deployment. Everything in one project.

How Each Lesson Works

Every lesson follows the same structure, designed around how you actually learn:

  1. Mental Model First: An analogy or framework for thinking about the concept. No code yet. Just the "why."
  2. The Concept in Depth: Technical explanation with diagrams, comparisons, and real code. This is the "how."
  3. Code Examples: Real TypeScript/JSX that you can run. Not toy examples — patterns you'd use in production.
  4. Quizzes: Check your understanding before moving on. Hidden answers so you can test yourself.
  5. Exercises: Build something that forces you to apply the mental model.

The journalism analogy runs throughout this course. You're investigating React. Each lesson is a beat: you develop sources (documentation, source code), gather evidence (experiments, benchmarks), and build a story (mental model) that explains what you've found. The best developers, like the best journalists, can explain complex systems simply because they understand them deeply.

Prerequisites

This course assumes you can already:

  • Build a React app with functional components and hooks
  • Use Next.js for routing and basic data fetching
  • Write TypeScript (at least basic types, interfaces, generics)
  • Style with Tailwind CSS or CSS modules
  • Use Git and npm/pnpm for project management

If any of those feel shaky, go build a small project first. This course won't re-teach the basics — it builds on them.

What Success Looks Like

By the end of this course, you should be able to:

  1. Explain React's rendering model to a junior developer without using jargon
  2. Debug performance issues by reading the Profiler and understanding why components re-render
  3. Architect a Next.js application with proper server/client boundaries, streaming, and caching
  4. Choose the right tool for state management, data fetching, form handling, and testing
  5. Compare frameworks based on architectural tradeoffs, not Twitter hype
  6. Build a production-grade full-stack app that demonstrates all of the above

The real test: when you encounter a new React feature or a framework you've never used, you should be able to reason about what it does and why it exists — without reading the docs first. That's what mental models buy you.

Before You Start: Self-Assessment

1. Can you explain what happens between calling setState and seeing the update on screen?

If you said "React re-renders the component" — that's the surface answer. The deep answer involves fiber trees, reconciliation, commit phases, and batching. Lesson 02 covers this in detail.

2. Why can't you call hooks inside an if statement?

Because hooks are stored in a linked list on the fiber node, indexed by call order. Conditional hooks would shift the indices. Lesson 03 makes this completely clear.

3. What's the difference between a Server Component and a Client Component in Next.js?

Server Components run only on the server and send serialized JSX (not JavaScript) to the client. Client Components ship their JS bundle to the browser. The boundary is defined by "use client" directives. Lesson 05 goes deep on this.

4. When should you use React.memo?

Almost never, unless you've profiled and confirmed that a component re-renders expensively with the same props. Most re-renders are cheap. React.memo adds comparison overhead on every render. Lesson 08 covers when memoization actually helps.
