06 Iterators & Generators — The Serialized Story

← Back to lessons

Generators are functions that can pause and resume. They produce values one at a time, on demand. Combined with async, they unlock powerful patterns for streaming data, paginated APIs, and lazy evaluation.

The Mental Model: A Serialized Investigation

Imagine you're publishing an investigation as a serial — one part per week. You don't write all 10 parts upfront. You write part 1, publish it, and pause. The audience reads it. When they want the next part, you resume writing and publish part 2. This continues until the investigation is complete.

A generator function works exactly this way. It produces one value (yield), pauses, and waits for the consumer to ask for the next value (.next()). It remembers where it left off each time.

The Iterator Protocol

Before generators, let's understand what they build on. The iterator protocol is a contract:

// Any object with a .next() method that returns { value, done } is an iterator
const countdown = {
  current: 3,
  next() {
    if (this.current > 0) {
      return { value: this.current--, done: false };
    }
    return { value: undefined, done: true };
  }
};

countdown.next();  // { value: 3, done: false }
countdown.next();  // { value: 2, done: false }
countdown.next();  // { value: 1, done: false }
countdown.next();  // { value: undefined, done: true }

The Iterable Protocol

An iterable is any object with a [Symbol.iterator] method that returns an iterator:

const newsWire = {
  stories: ["Election Update", "Market Close", "Weather Alert"],

  [Symbol.iterator]() {
    let index = 0;
    const stories = this.stories;

    return {
      next() {
        if (index < stories.length) {
          return { value: stories[index++], done: false };
        }
        return { done: true };
      }
    };
  }
};

// Now you can use for...of
for (const story of newsWire) {
  console.log(story);
}

// And spread
const allStories = [...newsWire];

// And destructuring
const [first, second] = newsWire;

Built-in iterables: String, Array, Map, Set, TypedArray, arguments, NodeList. When you write for...of, spread (...), or destructuring, JavaScript calls [Symbol.iterator]() under the hood. That's the mechanism.

Generator Functions

Writing iterators manually is verbose. Generators give you the same result with much less code:

function* countdown(start) {
  while (start > 0) {
    yield start;   // Pause here. Return { value: start, done: false }
    start--;       // Resume here on next .next() call
  }
  // Implicit return: { value: undefined, done: true }
}

const counter = countdown(3);  // Returns a generator object (an iterator)
counter.next();  // { value: 3, done: false }
counter.next();  // { value: 2, done: false }
counter.next();  // { value: 1, done: false }
counter.next();  // { value: undefined, done: true }

// Generators are iterable — you can use for...of
for (const n of countdown(5)) {
  console.log(n);  // 5, 4, 3, 2, 1
}

yield is Two-Way

You can send values INTO a generator through .next(value):

function* interview() {
  const name = yield "What's your name?";
  const age = yield `Hello ${name}, how old are you?`;
  return `${name} is ${age} years old`;
}

const gen = interview();
gen.next();           // { value: "What's your name?", done: false }
gen.next("Trevor");   // { value: "Hello Trevor, how old are you?", done: false }
gen.next(30);         // { value: "Trevor is 30 years old", done: true }

How it works: yield is an expression that pauses AND returns a value outward. The next .next(value) call resumes the generator AND injects value as the result of the yield expression. It's bidirectional communication.

Practical Generator Patterns

1. Infinite Sequences

function* fibonacci() {
  let a = 0, b = 1;
  while (true) {        // Infinite! But safe because it's lazy
    yield a;
    [a, b] = [b, a + b];
  }
}

// Take only what you need
function take(n, iterable) {
  const result = [];
  for (const value of iterable) {
    result.push(value);
    if (result.length >= n) break;
  }
  return result;
}

take(8, fibonacci());  // [0, 1, 1, 2, 3, 5, 8, 13]

2. yield* — Delegating to Another Generator

function* sections() {
  yield "Politics";
  yield "Technology";
}

function* fullPaper() {
  yield "Front Page";
  yield* sections();     // Delegates to sections()
  yield "Classifieds";
}

[...fullPaper()];  // ["Front Page", "Politics", "Technology", "Classifieds"]

3. Lazy Pipelines

function* filter(iterable, predicate) {
  for (const item of iterable) {
    if (predicate(item)) yield item;
  }
}

function* map(iterable, transform) {
  for (const item of iterable) {
    yield transform(item);
  }
}

// Process a million items lazily — only computes what's needed
const numbers = function*() {
  for (let i = 1; i <= 1_000_000; i++) yield i;
};

const result = take(5,
  map(
    filter(numbers(), n => n % 2 === 0),  // Even numbers
    n => n * n                             // Squared
  )
);
// [4, 16, 36, 64, 100] — only computed 10 values total, not 1,000,000

Async Generators & for-await-of

Combine generators with async to iterate over async data sources:

async function* fetchPages(baseUrl) {
  let page = 1;
  let hasMore = true;

  while (hasMore) {
    const res = await fetch(`${baseUrl}?page=${page}`);
    const data = await res.json();
    yield data.results;

    hasMore = data.next !== null;
    page++;
  }
}

// Consume with for-await-of
for await (const pageResults of fetchPages("/api/articles")) {
  console.log(`Got ${pageResults.length} articles`);
  // Process each page as it arrives
  // Don't need to load ALL pages into memory
}

This is how streaming works. Async generators are the primitive behind Node.js readable streams, Server-Sent Events, and React Server Components' streaming. When you see data arriving in chunks, there's likely an async iterable underneath.

Real-World: Streaming Chat API

async function* streamChat(prompt) {
  const res = await fetch("/api/chat", {
    method: "POST",
    body: JSON.stringify({ prompt }),
  });

  const reader = res.body.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    yield decoder.decode(value);
  }
}

// Display tokens as they arrive
for await (const chunk of streamChat("Explain closures")) {
  outputDiv.textContent += chunk;
}

Knowledge Check

1. What is the difference between an iterator and an iterable?

An iterator is an object with a .next() method that returns { value, done }. An iterable is an object with a [Symbol.iterator]() method that returns an iterator. Arrays are iterables. The object returned by arr[Symbol.iterator]() is the iterator.

2. Why are infinite generators safe when infinite loops are not?

Generators are lazy — they only execute up to the next yield when you call .next(). An infinite while(true) loop in a generator only runs one iteration per .next() call. A regular infinite loop blocks the call stack forever because it never yields control back to the event loop.

3. When would you use for await...of instead of Promise.all?

Use for await...of when you're consuming a stream of values that arrive over time (paginated API, file stream, SSE). Use Promise.all when you have a known set of concurrent operations to fire at once. The key difference: for await processes sequentially as values arrive; Promise.all runs everything concurrently.
