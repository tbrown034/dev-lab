08 TypeScript Generics — The Story Template

← Back to lessons

Generics are TypeScript's most powerful feature. They let you write code that works with any type while preserving type safety. Every React hook, every utility type, every library you use is built on generics.

The Mental Model: Story Templates

A news organization has story templates: "PERSON did THING at PLACE on DATE." The template is fixed, but the specifics get filled in for each story. A generic type works the same way — the structure is defined once, and the specific types get filled in at use time.

Array<T> is a template: "a list of T." Array<string> fills in T with string. Array<User> fills in T with User. The structure (list operations: push, pop, map, filter) stays the same. Only the element type changes.

Generic Functions

// Without generics: you lose type information
function first(arr: any[]): any {
  return arr[0];
}
const x = first([1, 2, 3]);  // x: any (useless!)

// With generics: type flows through
function first<T>(arr: T[]): T {
  return arr[0];
}
const x = first([1, 2, 3]);       // x: number (inferred!)
const y = first(["a", "b"]);      // y: string
const z = first<boolean>([true]);  // z: boolean (explicit)

Generics preserve the connection between input and output. Without generics, the return type is disconnected from the argument type. With generics, TypeScript tracks the type through the function. This is why useState<number>(0) gives you a number setter — the generic flows through.

Multiple Type Parameters

function pair<A, B>(first: A, second: B): [A, B] {
  return [first, second];
}

const p = pair("hello", 42);  // type: [string, number]

// Real example: Map
const map = new Map<string, User>();
map.set("trevor", { name: "Trevor", age: 30 });
map.get("trevor");  // User | undefined

Generic Constraints

Sometimes you need to restrict what types T can be:

// T must have a .length property
function longest<T extends { length: number }>(a: T, b: T): T {
  return a.length >= b.length ? a : b;
}

longest("hello", "world");         // OK: strings have .length
longest([1, 2], [1, 2, 3]);        // OK: arrays have .length
longest(10, 20);                    // Error: numbers don't have .length

// T must be a key of the object
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { name: "Trevor", age: 30 };
getProperty(user, "name");   // type: string
getProperty(user, "age");    // type: number
getProperty(user, "email");  // Error: "email" is not a key of user

keyof is your best friend. It extracts the union of all keys from a type. keyof User is "name" | "age" | "email". Combined with extends, it ensures you can only access properties that actually exist. This is the foundation of type-safe property access.

Conditional Types

Types can have if/else logic:

// Syntax: T extends U ? X : Y
type IsString<T> = T extends string ? "yes" : "no";

type A = IsString<string>;    // "yes"
type B = IsString<number>;    // "no"
type C = IsString<"hello">;   // "yes" (literal extends string)

Distributive Conditional Types

// When T is a union, conditional types distribute across the union
type ToArray<T> = T extends any ? T[] : never;

type D = ToArray<string | number>;
// = (string extends any ? string[] : never) | (number extends any ? number[] : never)
// = string[] | number[]

// Compare with non-distributive (wrapped in tuple):
type ToArrayND<T> = [T] extends [any] ? T[] : never;
type E = ToArrayND<string | number>;
// = (string | number)[]  — one mixed array, not a union of arrays

Extract and Exclude (Built-in Conditional Types)

type Status = "draft" | "published" | "archived" | "deleted";

// Exclude: remove members from a union
type ActiveStatus = Exclude<Status, "deleted">;
// "draft" | "published" | "archived"

// Extract: keep only matching members
type PublicStatus = Extract<Status, "published" | "archived">;
// "published" | "archived"

// NonNullable: remove null and undefined
type MaybeString = string | null | undefined;
type DefinitelyString = NonNullable<MaybeString>;
// string

Mapped Types

Transform every property of a type according to a rule:

// Make all properties optional (this is how Partial works!)
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

// Make all properties readonly
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

// Make all properties nullable
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};

interface User {
  name: string;
  age: number;
}

type NullableUser = Nullable<User>;
// { name: string | null; age: number | null }

Key Remapping (as clause)

// Create getter methods for every property
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

type UserGetters = Getters<User>;
// { getName: () => string; getAge: () => number }

// Filter properties by type
type StringKeys<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};

type UserStrings = StringKeys<User>;
// { name: string } — age is filtered out

The infer Keyword

infer lets you extract types from within other types. Think of it as a capture group in regex, but for types:

// Extract return type of a function
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type A = MyReturnType<() => string>;    // string
type B = MyReturnType<(x: number) => boolean>;  // boolean

// Extract element type from an array
type ElementOf<T> = T extends (infer E)[] ? E : never;

type C = ElementOf<string[]>;     // string
type D = ElementOf<(number | boolean)[]>;  // number | boolean

// Extract Promise value type
type Unwrap<T> = T extends Promise<infer V> ? V : T;

type E = Unwrap<Promise<string>>;  // string
type F = Unwrap<number>;           // number (not a Promise, return as-is)

// Deep unwrap (recursive!)
type DeepUnwrap<T> = T extends Promise<infer V> ? DeepUnwrap<V> : T;
type G = DeepUnwrap<Promise<Promise<string>>>;  // string

infer is pattern matching for types. The pattern is defined by the extends clause. infer R says "whatever matches this position, call it R." This is how TypeScript extracts the element type from arrays, the return type from functions, and the value type from Promises.

Real-World Generic Patterns

Type-Safe Event Emitter

type EventMap = {
  login: { userId: string; timestamp: number };
  logout: { userId: string };
  error: { message: string; code: number };
};

class TypedEmitter<Events extends Record<string, any>> {
  private handlers = new Map<string, Function[]>();

  on<K extends keyof Events>(event: K, handler: (data: Events[K]) => void) {
    const list = this.handlers.get(event as string) || [];
    list.push(handler);
    this.handlers.set(event as string, list);
  }

  emit<K extends keyof Events>(event: K, data: Events[K]) {
    const list = this.handlers.get(event as string) || [];
    list.forEach(fn => fn(data));
  }
}

const emitter = new TypedEmitter<EventMap>();
emitter.on("login", (data) => {
  // data is { userId: string; timestamp: number }
  console.log(data.userId);
});
emitter.emit("login", { userId: "trevor", timestamp: Date.now() });
emitter.emit("login", { userId: "trevor" });  // Error: missing timestamp

Type-Safe API Client

type Routes = {
  "/api/users":    { GET: User[]; POST: User };
  "/api/users/:id": { GET: User; PUT: User; DELETE: void };
  "/api/articles": { GET: Article[]; POST: Article };
};

async function api<
  P extends keyof Routes,
  M extends keyof Routes[P]
>(path: P, method: M): Promise<Routes[P][M]> {
  const res = await fetch(path, { method: method as string });
  return res.json();
}

const users = await api("/api/users", "GET");     // User[]
const user = await api("/api/users/:id", "GET");  // User
await api("/api/users", "DELETE");                 // Error: DELETE not on /api/users

Knowledge Check

1. What type does this resolve to?

type X = "hello" extends string ? true : false;
true. The literal type "hello" extends (is assignable to) string, so the conditional resolves to the true branch.

2. Explain why Partial is implemented as a mapped type.

Partial<T> is { [K in keyof T]?: T[K] }. It iterates over every key K in T (via keyof T), keeps the same value type (T[K]), and adds the ? modifier to make each property optional. It needs to be a mapped type because it transforms each property individually.

3. What does infer do in T extends Promise<infer U> ? U : T?

infer U creates a new type variable U that captures whatever type is inside the Promise<>. If T is Promise<string>, then U is inferred as string. If T doesn't match the Promise<...> pattern, the false branch returns T unchanged.

Exercise: Build a Type-Safe Form Validator

Create a generic validation system:

// Define your schema type
type Schema = {
  name: { type: "string"; required: true };
  age: { type: "number"; required: false };
  email: { type: "string"; required: true };
};

// Build these utility types:

// 1. SchemaToType — converts schema to the actual data type
type UserData = SchemaToType<Schema>;
// { name: string; age?: number; email: string }

// 2. ValidationErrors — maps each key to string | undefined
type UserErrors = ValidationErrors<Schema>;
// { name?: string; age?: string; email?: string }

// 3. RequiredKeys — extracts only the required field names
type Required = RequiredKeys<Schema>;
// "name" | "email"

Hints: You'll need conditional types, mapped types, and key remapping.
