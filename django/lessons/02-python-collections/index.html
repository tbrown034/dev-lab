02 Python Collections & Control Flow

← Back to lessons

You already know arrays, objects, and loops. This lesson maps every JavaScript collection and control flow pattern to its Python equivalent — plus Python's secret weapon: comprehensions.

Lists — Python's Arrays

A Python list is JavaScript's Array. Same concept, same bracket syntax, slightly different method names.

JavaScript
// Creating arrays
const nums = [1, 2, 3, 4, 5];
const mixed = [1, "two", true, null];
const nested = [[1, 2], [3, 4]];
const empty = [];

// Common operations
nums.push(6);           // append
nums.pop();             // remove last
nums.unshift(0);        // prepend
nums.splice(1, 1);      // remove at index
nums.concat([7, 8]);    // combine
nums.indexOf(3);        // find index
nums.includes(3);       // check membership
nums.length;            // size
nums.reverse();         // reverse in place
nums.sort();            // sort in place
Python
# Creating lists
nums = [1, 2, 3, 4, 5]
mixed = [1, "two", True, None]
nested = [[1, 2], [3, 4]]
empty = []

# Common operations
nums.append(6)          # push
nums.pop()              # remove last
nums.insert(0, 0)       # prepend (insert at index)
del nums[1]             # remove at index
nums + [7, 8]           # combine (or nums.extend)
nums.index(3)           # find index
3 in nums               # check membership
len(nums)               # size
nums.reverse()          # reverse in place
nums.sort()             # sort in place

Method name translations: .push() → .append(). .includes() → in operator. .length → len(). .indexOf() → .index(). Python uses built-in functions (len, sorted, reversed) where JS uses methods.

Slicing — Python's Supercharged .slice()

JavaScript's .slice() lets you extract part of an array. Python's slice syntax does the same thing but with far more power — and it works on strings, lists, and tuples uniformly.

JavaScript
const arr = [0, 1, 2, 3, 4, 5, 6, 7];

arr.slice(2, 5);     // [2, 3, 4]
arr.slice(0, 3);     // [0, 1, 2]
arr.slice(-2);       // [6, 7]
arr.slice();         // shallow copy

// No built-in "every Nth" or reverse
// Need: arr.filter((_, i) => i % 2 === 0)
// Need: [...arr].reverse()
Python
arr = [0, 1, 2, 3, 4, 5, 6, 7]

arr[2:5]             # [2, 3, 4]
arr[:3]              # [0, 1, 2]
arr[-2:]             # [6, 7]
arr[:]               # shallow copy

# Step argument (third parameter!)
arr[::2]             # [0, 2, 4, 6] — every 2nd
arr[::-1]            # [7,6,5,4,3,2,1,0] — reversed!

The mental model: Python slicing is [start:stop:step]. Omit start for the beginning, omit stop for the end, and the step parameter is the superpower JS doesn't have. [::-1] to reverse anything is a Python classic.

Negative Indices

Both languages support negative indices, but Python makes them feel native.

JavaScript
const arr = ["a", "b", "c", "d", "e"];

arr.at(-1);       // "e" (ES2022)
arr.at(-2);       // "d"
arr.slice(-3);    // ["c", "d", "e"]

// Before .at(), needed:
arr[arr.length - 1]; // "e"
Python
arr = ["a", "b", "c", "d", "e"]

arr[-1]           # "e" (always worked!)
arr[-2]           # "d"
arr[-3:]          # ["c", "d", "e"]

# Works with assignment too
arr[-1] = "z"     # ["a", "b", "c", "d", "z"]

Tuples — Frozen Arrays

A tuple is an immutable list. Think of it as Object.freeze([1, 2, 3]) in JavaScript — you can read it, but you can never add, remove, or change its elements.

JavaScript
// Closest equivalent: frozen array
const point = Object.freeze([10, 20]);
point[0];         // 10
point.push(30);   // TypeError!

// No real tuple in JS
// Often use arrays as tuples:
const [lat, lng] = [40.7, -74.0];

// Or objects for clarity:
const point = { x: 10, y: 20 };
Python
# Tuples use parentheses (or nothing)
point = (10, 20)
point[0]          # 10
# point.append(30) — AttributeError!

# Parentheses are optional:
point = 10, 20    # also a tuple
lat, lng = 40.7, -74.0

# Single-element tuple needs trailing comma
single = (42,)    # tuple
not_tuple = (42)  # just the number 42!

Gotcha: Single-element tuples. (42) is just 42 with parentheses (like math grouping). (42,) is a tuple. The trailing comma is what makes it a tuple. This trips up everyone coming from JS.

When to use tuples vs lists: Use tuples for fixed data that shouldn't change — coordinates, RGB values, database rows, function return values. Use lists when you need to add/remove items. Tuples are also faster and can be used as dictionary keys (lists can't).

Dicts — Python's Objects (for data)

Python dict is what you use when you'd reach for a JavaScript object as a key-value store. The syntax is almost identical.

JavaScript
const user = {
  name: "Trevor",
  age: 38,
  skills: ["JS", "React"]
};

// Access
user.name;              // "Trevor"
user["age"];            // 38
user.email;             // undefined
user?.email;            // undefined (safe)

// Modify
user.email = "t@b.com";
delete user.age;

// Iterate
Object.keys(user);
Object.values(user);
Object.entries(user);
Python
user = {
    "name": "Trevor",
    "age": 38,
    "skills": ["JS", "React"]
}

# Access
user["name"]            # "Trevor"
user["age"]             # 38
# user["email"]         # KeyError!
user.get("email")       # None (safe)

# Modify
user["email"] = "t@b.com"
del user["age"]

# Iterate
user.keys()
user.values()
user.items()  # like Object.entries()

No dot notation for dicts! In JavaScript, obj.key and obj["key"] are interchangeable. In Python, dict.key does NOT work — you must use dict["key"] or dict.get("key"). Dot notation is reserved for object attributes and methods. You'll see dot notation later with Django models.

Dict Tricks

JavaScript
// Default value if key missing
const val = obj.key ?? "default";

// Merge objects
const merged = { ...a, ...b };

// Check key exists
"name" in obj;
obj.hasOwnProperty("name");

// Iterate entries
for (const [key, val] of Object.entries(obj)) {
  console.log(key, val);
}
Python
# Default value if key missing
val = d.get("key", "default")

# Merge dicts
merged = {**a, **b}        # or a | b (3.9+)

# Check key exists
"name" in d                # checks keys by default

# Iterate entries
for key, val in d.items():
    print(key, val)

Translation rule: ?? for defaults → .get(key, default). {...a, ...b} → {**a, **b} or a | b. Object.entries() → .items(). "key" in obj works the same in both languages.

Sets

Sets work almost identically in both languages. A set is a collection of unique values with no duplicates.

JavaScript
const tags = new Set(["news", "sports", "news"]);
// Set {"news", "sports"}

tags.add("politics");
tags.has("news");        // true
tags.delete("sports");
tags.size;               // 2

// Dedupe an array
const unique = [...new Set(arr)];

// No built-in intersection/union
// Need libraries or manual code
Python
tags = {"news", "sports", "news"}
# {"news", "sports"}

tags.add("politics")
"news" in tags           # True
tags.discard("sports")
len(tags)                # 2

# Dedupe a list
unique = list(set(arr))

# Built-in set operations!
a & b      # intersection
a | b      # union
a - b      # difference

Key difference: Python sets use curly braces {}, not new Set(). But watch out: {} alone creates an empty dict, not a set. Use set() for an empty set. Python's built-in set operators (&, |, -) are a joy — no library needed.

For Loops — Python's Bread and Butter

Python has one kind of for loop: for item in iterable. No C-style for (let i = 0; ...). Everything is iteration over a collection.

JavaScript
// Basic iteration
for (const fruit of fruits) {
  console.log(fruit);
}

// With index
fruits.forEach((fruit, i) => {
  console.log(i, fruit);
});

// C-style (counting)
for (let i = 0; i < 10; i++) {
  console.log(i);
}

// Iterate object entries
for (const [key, val] of Object.entries(obj)) {
  console.log(key, val);
}
Python
# Basic iteration
for fruit in fruits:
    print(fruit)


# With index — enumerate()
for i, fruit in enumerate(fruits):
    print(i, fruit)


# Counting — range()
for i in range(10):
    print(i)


# Iterate dict entries
for key, val in obj.items():
    print(key, val)

range() — Python's Counter

range() generates a sequence of numbers. It's what replaces the C-style for (let i = 0; i < n; i++).

range(5)         # 0, 1, 2, 3, 4        — start defaults to 0
range(2, 7)      # 2, 3, 4, 5, 6        — start, stop (exclusive)
range(0, 10, 2)  # 0, 2, 4, 6, 8        — start, stop, step
range(10, 0, -1) # 10, 9, 8, ... 1      — count backwards

The mental model: range(start, stop, step) mirrors slice syntax [start:stop:step]. Same three-argument pattern, same "stop is exclusive" rule. Learn it once, use it everywhere.

enumerate() — forEach with index

JavaScript
const colors = ["red", "green", "blue"];

colors.forEach((color, index) => {
  console.log(`${index}: ${color}`);
});
// 0: red
// 1: green
// 2: blue
Python
colors = ["red", "green", "blue"]

for index, color in enumerate(colors):
    print(f"{index}: {color}")

# 0: red
# 1: green
# 2: blue

# Start at 1 (great for display)
for num, color in enumerate(colors, 1):
    print(f"{num}. {color}")  # 1. red ...

zip() — Iterate Multiple Arrays Together

Python's zip() has no direct JavaScript equivalent. It pairs up elements from multiple iterables, like a zipper joining two sides.

JavaScript
const names = ["Alice", "Bob", "Charlie"];
const scores = [95, 87, 92];

// Manual pairing
names.forEach((name, i) => {
  console.log(`${name}: ${scores[i]}`);
});

// Or: names.map((n, i) => [n, scores[i]])
Python
names = ["Alice", "Bob", "Charlie"]
scores = [95, 87, 92]

# zip pairs them up automatically
for name, score in zip(names, scores):
    print(f"{name}: {score}")


# Create a dict from two lists
grade_book = dict(zip(names, scores))
# {"Alice": 95, "Bob": 87, "Charlie": 92}

Translation rule: Whenever you're using index-based access to sync two arrays in JS (scores[i]), that's a sign you want zip() in Python. It's cleaner, more readable, and impossible to get an off-by-one error.

While Loops

JavaScript
let count = 5;
while (count > 0) {
  console.log(count);
  count--;
}

// break and continue work the same
for (const item of items) {
  if (item === "skip") continue;
  if (item === "stop") break;
  process(item);
}
Python
count = 5
while count > 0:
    print(count)
    count -= 1  # no count-- in Python

# break and continue work the same
for item in items:
    if item == "skip": continue
    if item == "stop": break
    process(item)

# Python bonus: for/else (runs if no break)
for item in items:
    if item == "target":
        break
else:
    print("target not found")

Python has no ++ or --. Use count += 1 and count -= 1. Python's for/else runs the else block only if the loop completes without hitting break. It's niche but elegant for search patterns.

Conditionals — if / elif / else

JavaScript
// Standard if/else
if (status === "published") {
  show();
} else if (status === "draft") {
  edit();
} else {
  hide();
}

// Ternary
const label = count === 1 ? "item" : "items";

// Switch (verbose)
switch (action) {
  case "save": save(); break;
  case "delete": remove(); break;
  default: log("unknown");
}
Python
# Standard if/elif/else
if status == "published":
    show()
elif status == "draft":
    edit()
else:
    hide()


# Conditional expression (ternary)
label = "item" if count == 1 else "items"

# match/case (Python 3.10+ — like switch)
match action:
    case "save": save()
    case "delete": remove()
    case _: log("unknown")

Ternary order is different! JS: condition ? true_val : false_val. Python: true_val if condition else false_val. The value comes first in Python. This will trip you up for the first week, then it'll feel natural because you read "give me X if Y" like English.

Comprehensions — Python's Superpower

List comprehensions are the single most important syntax to learn in Python. They replace .map(), .filter(), and often both — in a single, readable line. Once you internalize the pattern, you'll reach for them constantly.

List Comprehensions

JavaScript
const nums = [1, 2, 3, 4, 5, 6, 7, 8];

// map
const doubled = nums.map(n => n * 2);

// filter
const evens = nums.filter(n => n % 2 === 0);

// map + filter (chained)
const doubledEvens = nums
  .filter(n => n % 2 === 0)
  .map(n => n * 2);

// Nested: flatMap
const matrix = [[1,2],[3,4],[5,6]];
const flat = matrix.flatMap(row => row);
Python
nums = [1, 2, 3, 4, 5, 6, 7, 8]

# map → [expression for item in iterable]
doubled = [n * 2 for n in nums]

# filter → add "if" at the end
evens = [n for n in nums if n % 2 == 0]

# map + filter → one expression
doubled_evens = [
    n * 2 for n in nums if n % 2 == 0
]

# Nested: flatten
matrix = [[1,2],[3,4],[5,6]]
flat = [x for row in matrix for x in row]

The pattern: [EXPRESSION for ITEM in ITERABLE if CONDITION]. Read it like English: "Give me [expression] for each [item] in [collection] if [condition]." The expression goes first (what you want), the loop goes second (where from), the filter goes last (which ones).

Dict Comprehensions

JavaScript
// Create object from array
const scores = { };
students.forEach(s => {
  scores[s.name] = s.score;
});

// Or with Object.fromEntries
const scores = Object.fromEntries(
  students.map(s => [s.name, s.score])
);

// Filter object keys
const filtered = Object.fromEntries(
  Object.entries(obj)
    .filter(([k, v]) => v > 50)
);
Python
# Create dict from list
scores = {
    s["name"]: s["score"] for s in students
}


# Same idea — key: value expression
word_lengths = {
    word: len(word) for word in words
}

# Filter dict
filtered = {
    k: v for k, v in obj.items() if v > 50
}

Set Comprehensions

JavaScript
// Unique values with transformation
const unique = new Set(
  names.map(n => n.toLowerCase())
);
Python
# Set comprehension — just use { } instead of [ ]
unique = {
    name.lower() for name in names
}

Comprehension cheat sheet: List = [expr for x in iter]. Dict = {key: val for x in iter}. Set = {expr for x in iter}. Generator = (expr for x in iter) (lazy, memory efficient). Same pattern, different brackets.

Unpacking — Python's Destructuring

JavaScript destructuring maps directly to Python unpacking. Same idea, slightly different syntax.

JavaScript
// Array destructuring
const [a, b, c] = [1, 2, 3];

// Skip elements
const [first, , third] = [1, 2, 3];

// Rest
const [head, ...tail] = [1, 2, 3, 4];
// head = 1, tail = [2, 3, 4]

// Object destructuring
const { name, age } = person;

// Rename
const { name: fullName } = person;

// Nested
const { address: { city } } = person;
Python
# Tuple/list unpacking
a, b, c = [1, 2, 3]

# Skip elements (convention: _)
first, _, third = [1, 2, 3]

# Rest (star expression)
head, *tail = [1, 2, 3, 4]
# head = 1, tail = [2, 3, 4]

# No direct dict destructuring
name, age = person["name"], person["age"]

# Or use .values() if order is guaranteed
# name, age, city = person.values()

# Works anywhere — even in loops!
for name, score in [("Alice", 95), ("Bob", 87)]:
    print(f"{name}: {score}")

The mental model: Python's *rest = JavaScript's ...rest. One star, not three dots. It works on the left side of assignment (unpacking) and in function definitions (*args). The star can go anywhere: first, *middle, last = [1,2,3,4,5] gives you middle = [2,3,4].

No dict destructuring! Python has no equivalent of JavaScript's const { name, age } = obj. You access dict values individually. This is a common source of frustration for JS devs. However, Python dataclasses and named tuples give you attribute access which feels similar.

Useful Built-in Functions

Python has a rich set of built-in functions that replace common array method chains in JavaScript.

JavaScript
// Sum
const sum = nums.reduce((a, b) => a + b, 0);

// Min / Max
Math.min(...nums);
Math.max(...nums);

// All / Some
nums.every(n => n > 0);   // all positive?
nums.some(n => n > 0);    // any positive?

// Sort (mutates!)
nums.sort((a, b) => a - b);

// Sort copy
const sorted = [...nums].sort((a, b) => a - b);
Python
# Sum (built-in!)
total = sum(nums)

# Min / Max (built-in!)
min(nums)
max(nums)

# All / Any (built-in!)
all(n > 0 for n in nums)  # all positive?
any(n > 0 for n in nums)  # any positive?

# Sort (mutates!)
nums.sort()

# Sort copy (returns new list)
result = sorted(nums)

Translation rule: .reduce(sum) → sum(). Math.min() → min(). .every() → all(). .some() → any(). [...arr].sort() → sorted(arr). Python's built-ins replace entire chains of JavaScript methods with a single function call.

Sorting with Keys

JavaScript
// Sort by property
users.sort((a, b) => a.age - b.age);

// Sort by string property
users.sort((a, b) =>
  a.name.localeCompare(b.name)
);

// Reverse sort
users.sort((a, b) => b.age - a.age);
Python
# Sort by property — key function
users.sort(key=lambda u: u["age"])

# Sort by string property
users.sort(key=lambda u: u["name"])


# Reverse sort
users.sort(key=lambda u: u["age"], reverse=True)

Python's key function is elegant: Instead of a comparison function that returns -1/0/1, you provide a function that extracts the value to sort by. Python figures out the ordering. Less code, fewer bugs.

Check Your Understanding

Quick Quiz

1. What Python syntax replaces arr.filter(x => x > 10).map(x => x * 2)?

[x * 2 for x in arr if x > 10] — A single list comprehension handles both filter and map. The condition goes at the end.

2. How do you get both the index and value when iterating a list (like forEach with index)?

for i, val in enumerate(my_list): — enumerate() gives you (index, value) pairs. You can even start at a different number: enumerate(my_list, 1).

3. What's the difference between (42) and (42,) in Python?

(42) is just the integer 42 (parentheses are grouping, like in math). (42,) is a tuple containing one element. The trailing comma makes it a tuple.

4. How do you iterate over two lists simultaneously in Python?

for a, b in zip(list_a, list_b): — zip() pairs up elements from multiple iterables. It stops at the shortest one. You can also dict(zip(keys, values)) to create a dict.

5. What does {} create in Python — an empty set or an empty dict?

An empty dict. To create an empty set, use set(). This is because dicts had {} syntax first, and sets were added later. It's a gotcha that bites everyone once.
