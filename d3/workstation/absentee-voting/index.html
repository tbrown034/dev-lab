Mail Voting in America: Who Relies on It and What's at Stake

A data-driven investigation into absentee and mail-in voting across all 50 states. Five visualizations built with D3.js, exploring how Americans vote by mail, how COVID-19 reshaped voting habits, and which populations depend most on this method.

By Trevor Brown · Data Visualization Workstation · 2026

Section 01

How America Votes by Mail

A state-by-state look at mail voting's share of total ballots. Oregon and Washington vote almost entirely by mail, while most excuse-required states remain below 10%.

viz1-choropleth-map.jsD3.js v7 + TopoJSON

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • d3.geoAlbersUsa() — the built-in projection for US maps. It repositions Alaska and Hawaii as insets. No need to set center or rotate.
    • 
            
  • d3.geoPath(projection) — the path generator. Takes a GeoJSON feature and returns an SVG path d attribute string. Think of it as a "translator" from lat/lng to SVG coordinates.
    • 
            
  • TopoJSON → GeoJSON: topojson.feature(us, us.objects.states) converts the compressed TopoJSON back to GeoJSON features that D3 can render.
    • 
            
  • d3.scaleSequential(d3.interpolateYlOrRd) — maps a continuous domain [0, 100] to a color ramp. Sequential scales are perfect for "low to high" data like percentages.
    • 
            
  • Data lookup pattern: Create a Map keyed by FIPS code, then use it in .attr("fill", d => colorScale(lookup.get(d.id))). This is the standard choropleth pattern.
    • 
            
  • Linked highlighting: A shared highlightState(abbr) function that both the map and bar chart call. This is how newsroom graphics sync multiple views.
    • 
          

          
Mental model: A D3 map is just SVG <path> elements that happen to look like geography. Each state is one path. The projection is the "lens" that converts 3D coordinates to 2D screen positions. geoPath turns those positions into SVG path commands (M, L, C, etc.).

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: U.S. Election Assistance Commission (EAVS) reports for 2016, 2020, and 2024. State law classifications from the National Conference of State Legislatures (NCSL), 2025 update.

          
Methodology: Mail vote share = (mail/absentee ballots cast) / (total ballots cast) x 100. State boundaries from US Atlas TopoJSON (10m resolution). Albers USA projection for standard journalistic layout with Alaska/Hawaii insets.

          
Color scale choice: Sequential yellow-orange-red ramp chosen for intuitive "cool to hot" reading. States with no data in our subset render as the baseline color.

          
Limitations: Not all 50 states are represented in our curated dataset. EAVS data completeness varies by state. Some 2024 figures are estimated from early reports and news coverage.

viz2-ranked-bars.jsD3.js v7

        

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • d3.scaleBand() for the y-axis — each state gets an equal vertical slice. .padding(0.25) adds gaps between bars.
    • 
            
  • d3.scaleLinear() for the x-axis — domain [0, 100] maps to [0, chartWidth] since we're showing percentages.
    • 
            
  • Sort + rebind: When the year changes, re-sort the data, update the scaleBand domain, then transition bars to new positions. This is the .data().join() update pattern in action.
    • 
            
  • Filter pattern: Don't remove bars on filter — instead, set opacity or display. This preserves the DOM elements for smooth transitions when toggling filters off.
    • 
            
  • .transition().duration(600) on both width and y makes bars smoothly re-sort and resize when the year or filter changes.
    • 
          

          
Mental model: A horizontal bar chart is a scaleBand (categories on y) crossed with a scaleLinear (values on x). When data changes, you update both scales' domains and transition the bars. The "linked highlighting" is just a shared function that adds/removes a CSS class on both the map paths and bar groups.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: Same EAVS data as Section 01. State rule classifications from NCSL.

          
Why horizontal bars: With 36+ states, horizontal bars are more readable than vertical. State names serve as natural labels. Sorting by value makes comparison instant.

          
Color encoding: Rule type is encoded as bar color (pink = all-mail, purple = no-excuse, orange = excuse-required). This dual encoding (position + color) helps readers see clustering of policy types at different mail vote levels.

          
Linked highlighting: Hovering a state on the map highlights the corresponding bar, and vice versa. This "brushing and linking" technique is standard in data journalism dashboards.

viz3-slope-chart.jsD3.js v7

        

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • Slope chart = simplified line chart with only 3 x-positions (2016, 2020, 2024). Each state gets a polyline connecting its three data points.
    • 
            
  • d3.scalePoint() for the x-axis — like scaleBand but for points. Perfect when you have discrete categories (years) that need even spacing.
    • 
            
  • d3.line() path generator — takes an array of [x, y] pairs and returns an SVG path string. The same generator used for time series, just with fewer data points.
    • 
            
  • Annotation pattern: Hand-placed text labels using .append("text") with specific coordinates. In newsrooms, annotations are often the most time-consuming part.
    • 
            
  • Hover interaction on overlapping lines: Use stroke-width expansion on hover, plus bring the hovered line to front with this.parentNode.appendChild(this).
    • 
          

          
Mental model: A slope chart is just a line chart with the "time" axis compressed to a few columns. The slope (angle) of each line segment shows direction and magnitude of change. Steep up = big increase. The visual pattern of lines crossing each other tells the ranking story.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: EAVS 2016, 2020, 2024 reports. State totals cross-referenced with news coverage for 2024 estimates.

          
Why a slope chart: Slope charts are ideal for showing change across a small number of time periods. They reveal three patterns simultaneously: (1) starting level, (2) magnitude of change, (3) direction of change. The COVID spike in 2020 and partial rebound in 2024 are immediately visible.

          
Key findings: All-mail states (OR, WA, CO) show flat lines near the top — mail voting was already universal. Swing states like PA, MI, and WI show dramatic spikes in 2020 followed by significant drops in 2024, suggesting the pandemic expansion was temporary. Excuse-required states barely moved.

          
Limitations: Three data points can't capture the full trend. State-level aggregation masks county-level variation.

viz4-scatterplot.jsD3.js v7

        

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • Three visual encodings: x-position (mail share %), y-position (mail volume), size (total votes), color (rule type). Scatterplots can encode 4+ dimensions.
    • 
            
  • d3.scaleSqrt() for dot radius — use sqrt scale, not linear, for area-based encoding. A state with 4x the votes should have a dot with 4x the area, not 4x the radius (which would be 16x the area).
    • 
            
  • Crosshair tooltip: Draw dashed lines from the hovered dot to both axes. Use line elements that update position on mousemove.
    • 
            
  • d3.format(",") and d3.format(".1f") — D3's number formatting. Commas for thousands separators, .1f for one decimal place.
    • 
            
  • Quadrant annotation: A dashed rectangle or shaded region highlighting the "high share + high volume" quadrant adds editorial meaning to the scatterplot.
    • 
          

          
Mental model: A scatterplot reveals relationships between two variables. Each dot is a state plotted at (x, y). If dots cluster along a diagonal, the variables are correlated. Size adds a third dimension. Color adds a fourth. The "danger zone" quadrant is a journalistic annotation that tells the reader what pattern to look for.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: EAVS 2024 data. Mail ballot volume and total turnout by state.

          
Why a scatterplot: This chart answers a specific editorial question: "If mail voting were restricted, which states would be most affected?" States in the top-right quadrant have both high reliance (share) and high volume, meaning millions of voters use this method. California alone accounted for ~14.4M mail ballots in 2024.

          
Bubble size: Encoded using scaleSqrt to ensure perceptually accurate area comparison. Domain based on total votes cast (smallest to largest state).

          
Editorial note: This visualization intentionally frames mail voting access as a potential policy vulnerability. It does not take a position on whether restrictions are warranted.

viz5-national-summary.jsD3.js v7

        

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • d3.stack() — takes tabular data and computes baseline and topline values for each category. Returns arrays of [y0, y1] pairs that you feed to d3.area() or use for stacked bars.
    • 
            
  • Stacked bar pattern: Each "layer" is a group of rectangles. The y-position of each rect is d[0] (baseline) and the height is d[1] - d[0] (the segment).
    • 
            
  • d3.scaleBand() for the year axis (3 bars: 2016, 2020, 2024). d3.scaleLinear() for the percentage axis [0, 100].
    • 
            
  • IntersectionObserver to trigger the stat card animations when they scroll into view. Each card has a transition-delay for staggered entrance.
    • 
          

          
Mental model: A stacked bar chart shows part-to-whole relationships across categories. Each bar sums to 100%. The d3.stack() generator does the math of calculating where each segment starts and ends. Think of it as "stacking blocks" — D3 calculates the shelf height for each block.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: National totals compiled from EAVS reports, EAC summaries, and MIT Election Lab. Voting method breakdown (mail, early in-person, Election Day) from EAVS surveys and CRS reports.

          
Methodology: National breakdown percentages: 2016 (21% mail, 19% early, 60% Election Day), 2020 (42% mail, 26% early, 32% Election Day), 2024 (31% mail, 35% early, 34% Election Day). Early in-person and Election Day splits for 2024 are estimated from preliminary state reports.

          
Key insight: The 2020 election saw mail voting double from 21% to 42%. By 2024, it partially reverted to 31%, but early in-person voting grew to 35%, suggesting voters who adopted non-Election Day voting in 2020 shifted methods rather than returning to traditional Election Day voting.

What This Can and Cannot Tell You


      

        
What this data CAN show

        
    
          
  • How reliant each state is on mail/absentee voting as a share of total votes
    • 
          
  • How mail voting usage changed across the 2016, 2020, and 2024 elections
    • 
          
  • Which states have policies that encourage or restrict mail voting
    • 
          
  • The raw volume of voters who depend on this voting method
    • 
          
  • How the COVID-19 pandemic temporarily reshaped voting behavior
    • 
        

      


      

        
What this data CANNOT show

        
    
          
  • Whether mail voting restrictions will pass in any given state legislature
    • 
          
  • Whether mail voting increases or decreases fraud (this data is about volume, not security)
    • 
          
  • How future turnout would change if mail voting were restricted
    • 
          
  • Partisan breakdown of mail voters (this data is method-only, not party affiliation)
    • 
          
  • Individual voter preferences or motivations for choosing mail over in-person
    • 
        

      


      

        
Data limitations

        
    
          
  • EAVS data varies in completeness; not all states report all fields consistently
    • 
          
  • 2024 data includes estimates where final certified results were not yet available
    • 
          
  • State law classifications can change between elections; NCSL categories shown are as of 2025
    • 
          
  • This dataset covers 36 states that represent the range of mail voting policies and volumes, not all 50+DC
