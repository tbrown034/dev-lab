For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • d3.interpolateNumber(a, b) — returns a function that interpolates between two numbers. Used inside transition().tween() to animate the counter from 0 to 21.3M.
    • 
            
  • IntersectionObserver — not D3, but the standard way to trigger animations on scroll. We observe each viz section and fire the animation when it enters the viewport.
    • 
            
  • Waffle chart pattern: Use d3.range(100) to generate 100 indices, then lay them out as a 10x10 grid using i % cols for x and Math.floor(i / cols) for y. Color 9 of them differently to represent 9%.
    • 
            
  • d3.scaleLinear() for the horizontal bar chart. Domain is [0, max value], range is [0, chart width].
    • 
            
  • d3.scaleBand() for the bar positions on the y-axis. Each category gets an equal slice.
    • 
            
  • Staggered transitions: .delay((d, i) => i * 100) makes bars animate sequentially rather than all at once.
    • 
          

          
Mental model: The waffle chart is just a grid of squares where some are highlighted. Think of it like a 10x10 Excel grid with conditional formatting. The animated counter is a "number tween" — D3 can interpolate anything, not just positions.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: Brennan Center for Justice, "Citizens Without Proof" (updated 2024). Survey-based estimate of citizenship document access among voting-age Americans.

          
Methodology: The 21.3 million figure represents US citizens who report lacking ready access to documents that could prove citizenship (birth certificate, passport, naturalization certificate). The 9% proportion is visualized via a unit chart (waffle) where each square = 1% of voting-age citizens.

          
Why this approach: The animated counter creates emotional impact (journalism convention for large numbers). The waffle chart makes the proportion tangible — 9 out of 100 is easier to grasp than "21.3 million out of 240 million." The horizontal bars show sub-categories for deeper understanding.

          
Limitations: Survey-based estimates have margins of error. "Ready access" is self-reported. The 69M women with name mismatch may overlap with other categories.

viz2-voter-id-map.jsD3.js v7 + TopoJSON

        

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • d3.geoAlbersUsa() — the projection built specifically for US maps. It repositions Alaska and Hawaii into insets. No need to set center/rotate; it just works for US state data.
    • 
            
  • d3.geoPath(projection) — the path generator. Takes a GeoJSON feature and returns an SVG path string. Think of it as a "translator" from latitude/longitude coordinates to SVG d attribute values.
    • 
            
  • TopoJSON vs GeoJSON: TopoJSON is compressed (shared borders stored once). We use topojson.feature() to convert back to GeoJSON for D3. The CDN file is ~200KB vs ~2MB for GeoJSON.
    • 
            
  • Choropleth pattern: Create a lookup object mapping state names to categories, then use it in the fill: .attr("fill", d => colorMap[stateCategory[d.properties.name]]).
    • 
            
  • Tooltip on geographic shapes: Use mousemove with event.clientX/Y for positioning. For maps, the tooltip needs to follow the cursor because states have irregular shapes.
    • 
          

          
Mental model: A D3 map is just SVG paths that happen to look like geography. Each state is a <path> element. The projection is the "lens" that converts 3D globe coordinates to 2D screen coordinates. geoPath takes those 2D coordinates and turns them into SVG path commands.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: National Conference of State Legislatures (NCSL), "Voter Identification Requirements by State" (2025 update).

          
Methodology: States are classified into 5 tiers by the NCSL's established taxonomy: Strict Photo ID, Non-strict Photo ID, Strict Non-photo ID, Non-strict Non-photo ID, and No ID Required. "Strict" means voters without acceptable ID must cast a provisional ballot that is not counted unless they return with ID.

          
Technical implementation: US Atlas TopoJSON (10m resolution) for state boundaries. Albers USA projection for standard journalistic US map layout. Sequential color scale from green (no requirement) to red (strict photo ID) chosen for intuitive severity mapping.

          
Editorial note: The SAVE Act would effectively move all states to a requirement stricter than any current "Strict Photo ID" state, since it requires documentary proof of citizenship (passport, birth certificate) rather than just a state-issued photo ID.

For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • Unit chart pattern: d3.range(n) generates an array of indices. Lay them out in a grid. Each unit represents a fixed number (here, 153,000 votes per dot). This is the "one dot = X" convention from data journalism.
    • 
            
  • d3.scaleLinear() for the magnified inset bar chart. Important concept: using two different scales for the same data to show both the big picture and the detail.
    • 
            
  • Text annotations: svg.append("text") is used heavily here. In data journalism, annotations ARE the story. D3 makes it easy to position text at exact data-driven coordinates.
    • 
            
  • d3.format(",") and d3.format(".4%") — D3's number formatting is incredibly useful for journalism. Commas for large numbers, percentage formatting, SI prefixes, etc.
    • 
          

          
Mental model: This is a "scale comparison" viz. The whole point is that when you put 153 million and 9,824 on the same scale, you literally can't see the smaller number. The magnified inset is the classic journalism trick — "let's zoom in on that tiny sliver."

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Sources: Total votes from Federal Election Commission 2024 general election results. State audit data from Texas Secretary of State (2,724), Virginia Department of Elections (6,303), Ohio Secretary of State (499), Iowa Secretary of State (277), Michigan Secretary of State (16), North Carolina State Board of Elections (5).

          
Methodology: Direct comparison of total legitimate votes cast against all documented noncitizen voter findings from the states that have conducted audits. Combined total of all findings: 9,824 out of 153,000,000 votes = 0.0064%. The unit chart uses 1,000 dots where each dot = 153,000 votes; the combined noncitizen total wouldn't fill 7% of a single dot.

          
Why this approach: Log scales would compress the difference. The unit chart with a magnified inset preserves the visceral sense of proportion while still making the smaller numbers readable. This is a standard technique in investigative data journalism (used by ProPublica, NYT Upshot).

          
Caveat: These are confirmed findings from states that chose to audit. Not all states have conducted audits. However, every audit conducted has found rates well below 0.1%.

viz4-demographics.jsD3.js v7

        

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • d3.scaleBand() — perfect for categorical axes. The .padding() method controls spacing between bands. For grouped bars, we nest a second scaleBand within each category band.
    • 
            
  • Grouped bar pattern: Each category gets a group <g> element positioned with the outer scaleBand. Within each group, bars are positioned using an inner scaleBand for the sub-categories ("Lack access" vs "Lack entirely").
    • 
            
  • d3.axisLeft() on a band scale automatically creates labels for each category. Use .tickSize(0) to remove the tick marks, keeping just the labels.
    • 
            
  • Grid lines: Create a second axis call with .tickSize(-width) and .tickFormat(""). This draws lines across the chart without labels. Style them with low opacity.
    • 
            
  • .attr("fill", (d, i) => colors[i]) — the index parameter i tells us which sub-bar we're drawing within the group.
    • 
          

          
Mental model: A grouped bar chart is really a "chart within a chart." The outer chart positions groups, the inner chart positions bars within each group. Two levels of scaleBand, nested.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Source: Brennan Center for Justice survey data, supplemented by NPR/Marist poll on document access by demographic group.

          
Methodology: Survey respondents were asked whether they could readily produce documents proving U.S. citizenship. Results are broken down by race/ethnicity, household income, and age group. Two metrics are shown: "Lack ready access" (cannot quickly produce a document) and "Lack entirely" (do not possess one at all).

          
Why grouped bars: The two-bar grouping allows direct comparison between "lack access" and "lack entirely" within each demographic, while the horizontal layout makes category labels readable and enables easy comparison across demographics.

          
Editorial context: The disparate impact across demographics raises civil rights concerns. The 15% rate among 18-29 year olds is nearly double the rate for those 50+, suggesting age-related barriers to document access (e.g., born in hospitals that have since closed, reliance on parents for vital records).

viz5-audit-lollipop.jsD3.js v7

        

      


      

        
For Me — D3 Learning Notes

        

          
Key D3 concepts used:

          
    
            
  • Lollipop chart pattern: Combine a line (the stick) and a circle (the dot) for each data point. It's cleaner than a bar chart for very small values because the dot draws the eye to the actual data point.
    • 
            
  • d3.scaleLinear() for the x-axis (percentage). Setting the domain to [0, 1] gives us a reference line at 1% for context.
    • 
            
  • d3.scaleBand() for the y-axis (state names). Using .padding(0.4) gives comfortable spacing between lollipops.
    • 
            
  • Reference lines: svg.append("line") at a specific x-scale value. Add a text label next to it. This is a core data journalism technique — "here's a benchmark for comparison."
    • 
            
  • Annotations with text elements: Placing the percentage value at the end of each lollipop stick. Use text-anchor: "start" and offset by a few pixels from the dot.
    • 
          

          
Mental model: A lollipop chart is a bar chart that went on a diet. The "lollipop" (dot + stick) communicates the same data as a bar but with less visual weight. It's perfect when your values are all very small and you want the dots to be the focus, not heavy bars.

        

      


      

        
Technical Brief — For Interviewers & Editors

        

          
Data Sources: Official audit reports from each state's Secretary of State or Board of Elections. Texas (2024 audit), Virginia (2022-2024 cumulative), Ohio (2024), Iowa (2024), Michigan (2024), North Carolina (2024).

          
Methodology: Each state's noncitizen findings are expressed as a percentage of total registered voters in that state. The 1% reference line provides scale context — all findings are at least an order of magnitude below 1%.

          
Why lollipop over bar: With percentages this small (max 0.103%), bars would be nearly invisible. The lollipop's dot provides a visible anchor point, and the stick connects it to the axis for readability. The explicit percentage labels ensure precision.

          
Key finding: Virginia shows the highest rate (0.103%), yet this represents removals over a 2-year period, not a single election. The median rate across all audited states is 0.013%. North Carolina's rate is so low (0.00007%) it's barely distinguishable from zero.
