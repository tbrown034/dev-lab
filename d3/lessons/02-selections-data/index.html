02 How D3 Turns Numbers Into Pixels

← Back to lessons

Part 1: The Problem

In Lesson 01, we wrote things like this:

.attr("width", d => d.pop / 100)

That / 100 is embarrassing. We just eyeballed a number to shrink the population values down to something that fits on screen. What if the data changes? What if the screen is a different size? It'll look wrong.

Every chart has this problem: your data lives in one world, your screen lives in another.

  • Populations are in the millions. Your SVG is 600 pixels wide.
  • Dates span years. Your chart area is 400 pixels.
  • Donation amounts range from $200 to $5,000,000. Your bars need to be 20-350 pixels tall.

You need a translator. That's what a scale is.

Part 2: A Scale is a Translator

Imagine you're a journalist writing for both US and UK audiences. You need to convert Fahrenheit to Celsius. The formula is always the same:

// A temperature "scale"
input:  32°F   →   output:  0°C
input: 212°F   →   output: 100°C
input: 98.6°F  →   output:  37°C

A D3 scale works the same way, except instead of Fahrenheit→Celsius, it's Data→Pixels:

// A D3 scale
input:  0 (population)      →   output:   0px
input:  39,538,000 (CA pop) →   output: 500px
input:  20,000,000 (NY pop) →   output: 253px  // proportional!

A scale is a function. You give it a data value, it gives back a pixel value. That's the whole concept. D3 just gives you convenient ways to create these translator functions.

Creating a scale in D3

// Two things to tell the scale:
// 1. DOMAIN = the range of your data (input world)
// 2. RANGE  = the range of your pixels (output world)

const scale = d3.scaleLinear()
  .domain([0, 39_538_000])   // data goes from 0 to 39.5M
  .range([0, 500]);           // pixels go from 0 to 500

// Now use it like a function:
scale(39_538_000)   // → 500    (max data = max pixels)
scale(20_000_000)   // → 253    (proportional)
scale(0)            // → 0      (min data = min pixels)

Part 3: Different Data, Different Scales

Not all data is the same. D3 has different scale types for different situations. You only need 3-4 for most work:

1 scaleLinear — for continuous numbers

Population, money, temperature, percentages. Any number in a continuous range.

const y = d3.scaleLinear()
  .domain([0, 100])     // voter turnout: 0% to 100%
  .range([height, 0]);  // NOTE: flipped! 0% at bottom, 100% at top

Why is the range flipped? Remember SVG's Y-axis goes top-down. A value of 100% should be at the TOP of the chart (y=0 in SVG). A value of 0% should be at the BOTTOM (y=height). So we flip the range: [height, 0]. You'll do this every single time for Y-axes.

2 scaleBand — for categories (bar charts)

State names, district labels, candidate names. Each gets its own "band" of space.

const x = d3.scaleBand()
  .domain(["CA", "TX", "FL", "NY", "IL"])  // your categories
  .range([0, width])                        // spread across width
  .padding(0.2);                            // 20% gap between bars

x("CA")            // → 0     (start of first band)
x("TX")            // → 124   (start of second band)
x.bandwidth()      // → 99    (how wide each bar should be)

scaleBand is specifically for bar charts. It divides space evenly among categories and gives you .bandwidth() — the width each bar should be. No more manually calculating bar widths.

3 scaleTime — for dates

const x = d3.scaleTime()
  .domain([new Date("2020-01-01"), new Date("2024-12-31")])
  .range([0, width]);

4 scaleSequential — for color

Data value → color instead of data value → pixel. Essential for choropleth maps.

const color = d3.scaleSequential()
  .domain([0, 100])
  .interpolator(d3.interpolateBlues);  // light → dark blue

color(0)    // → very light blue
color(100)  // → dark blue

Part 4: Axes Are Just Labels for Your Scale

You have a scale translating data to pixels. But your viewer doesn't know that. They need labels. An axis draws tick marks and labels that show what the scale is doing.

  • Without axis: "That bar is 253 pixels tall. So what?"
  • With axis: "That bar represents 20 million people."
// An axis is created FROM a scale
const xAxis = d3.axisBottom(xScale);  // ticks below the line
const yAxis = d3.axisLeft(yScale);    // ticks left of the line

// Render it by "calling" it on a <g> group
svg.append("g")
  .attr("transform", `translate(0, ${height})`)  // push to bottom
  .call(xAxis);                       // D3 draws ticks + labels

What does .call() mean? It passes the selection to a function. group.call(xAxis) is the same as xAxis(group). D3 uses this because the axis function creates multiple child elements (line, ticks, text labels) inside the group.

Part 5: The Margins Convention

Axes take up space. Labels need room. The margins convention is D3's standard pattern for this. Every professional D3 chart starts with it:

// Step 1: Define margins
const margin = { top: 20, right: 30, bottom: 40, left: 50 };

// Step 2: Calculate inner chart dimensions
const width = 700 - margin.left - margin.right;   // 620
const height = 400 - margin.top - margin.bottom;  // 340

// Step 3: SVG at full size, group pushed inward
const svg = d3.select("#chart")
  .append("svg")
  .attr("viewBox", `0 0 700 400`)
  .append("g")
  .attr("transform", `translate(${margin.left}, ${margin.top})`);

Live: Margins visualized

Orange = your chart area. Gray = margins where axes and labels live.

Copy-paste this pattern into every chart. It's boilerplate. Margin → width/height → SVG → group. Once set up, everything goes inside the group and margins handle themselves.

Part 6: All Together — A Real Chart

Here's the complete pattern: margins + scales + axes + bars. This is what you'll use for every bar chart:

Lesson 01 (magic numbers)
.attr("x", (d,i) => i * 100)
.attr("y", d => 300 - d.pop/150)
.attr("width", 80)
.attr("height", d => d.pop / 150)

// Breaks if data changes
// No labels for viewer
Lesson 02 (scales)
.attr("x", d => x(d.state))
.attr("y", d => y(d.pop))
.attr("width", x.bandwidth())
.attr("height", d => height-y(d.pop))

// Scales handle the math
// Axes label it

Part 7: Useful Extras

.nice() — clean axis numbers

// Without: axis shows 0, 7907, 15815, 23722, 39538 (messy)
// With:    axis shows 0, 10000, 20000, 30000, 40000 (clean)
y.domain([0, d3.max(data, d => d.pop)]).nice();

.tickFormat() — format labels

d3.axisLeft(y).tickFormat(d => `${d / 1000}k`)     // 10000 → 10k
d3.axisBottom(x).tickFormat(d3.format("$,.0f"))     // 1200000 → $1,200,000

viewBox — responsive charts

// The chart scales to fit its container, like an image.
svg.attr("viewBox", "0 0 700 400")  // define coordinate space
// No fixed width/height needed. Responsive by default.

Check Your Understanding

1. In your own words, what does a D3 scale do?

A scale is a translator function. It converts values from your data world (populations, dollars, dates) into your screen world (pixels, colors). You define the input range (domain) and output range (range), and it handles all the proportional math.

2. Why do we write .range([height, 0]) for Y scales instead of .range([0, height])?

SVG Y-axis goes top-down. Small values (0%) should be at the bottom (large Y pixel), large values (100%) at the top (small Y pixel). Flipping the range does this automatically: domain 0 → pixel height (bottom), domain max → pixel 0 (top).

3. When would you use scaleBand vs scaleLinear?

scaleBand for categories (state names, labels) — divides space into equal bands. scaleLinear for continuous numbers (population, money) — maps proportionally. Text labels → band. Numbers → linear.

4. What does the margins convention give you?

Room for axes and labels around the chart area. You create the SVG at full size, push a group inward by the margins. Everything you draw goes in the group, axes fit in the margins. It's the standard boilerplate for every D3 chart.

5. What does .call(xAxis) do?

It passes the current selection (a <g> group) to the axis generator function. The axis function then creates all the axis elements (line, ticks, text labels) inside that group. It's equivalent to xAxis(selection).

Recap

  1. Scales are translators — data world → pixel world (or color world)
  2. Domain = input range, Range = output range
  3. scaleLinear for numbers, scaleBand for categories, scaleTime for dates
  4. Y range is always flipped [height, 0]
  5. Axes = labels for your scale, drawn with .call()
  6. Margins convention = the boilerplate every chart starts with
