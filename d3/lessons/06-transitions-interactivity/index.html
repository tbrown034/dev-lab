06 Axes & The Margins Convention

← Back to lessons

Mental Model: An axis is a "name tag" for your scale. It shows the viewer what the pixel positions mean. The margins convention is like the matting inside a picture frame — it creates space around the art for labels.

The Problem

You built a bar chart with scales, but the viewer sees unlabeled bars. They do not know that 300 pixels tall means 20 million people. An axis renders tick marks and labels that translate pixels back to human-readable data values.

Axes Are Generated FROM Scales

This is the key insight: you do not create axes from scratch. You create them from an existing scale. The axis reads the scale's domain and range to figure out where to put ticks and what to label them.

// Step 1: Create a scale (you already know this)
const yScale = d3.scaleLinear()
  .domain([0, 40_000_000])
  .range([height, 0]);

// Step 2: Create an axis FROM the scale
const yAxis = d3.axisLeft(yScale);

// Step 3: Render it into a <g> group
svg.append("g").call(yAxis);

Four axis orientations

  • d3.axisBottom(scale) — ticks below the line (for X-axis at bottom)
  • d3.axisTop(scale) — ticks above the line
  • d3.axisLeft(scale) — ticks to the left (for Y-axis)
  • d3.axisRight(scale) — ticks to the right

What does .call() do?

// These are equivalent:
svg.append("g").call(yAxis);
yAxis(svg.append("g"));

// .call() passes the selection to a function.
// The axis function creates lines, ticks, and labels
// inside that <g> group.

Why use .call()? It keeps the chain going. Without .call(), you would need to save the group to a variable, pass it to the axis function, then continue. With .call(), it is one fluent chain: svg.append("g").attr("transform", ...).call(yAxis).

Customizing Axes

d3.axisLeft(yScale)
  .ticks(5)                          // suggest ~5 ticks
  .tickFormat(d => `$${d / 1000}k`)  // custom labels
  .tickSize(-width)                  // extend ticks = grid lines!
  .tickValues([0, 25, 50, 75, 100]) // exact tick positions

Grid lines are just long ticks. Setting .tickSize(-width) extends each tick line across the full chart width. That is all grid lines are. No separate grid system needed.

The Margins Convention

Axes take up space. The chart area needs to be smaller than the SVG to leave room for labels. The margins convention is the standard pattern every professional D3 chart uses:

// Step 1: Define margins
const margin = { top: 20, right: 30, bottom: 40, left: 50 };

// Step 2: Calculate inner dimensions
const width = 700 - margin.left - margin.right;   // 620
const height = 400 - margin.top - margin.bottom;  // 340

// Step 3: SVG at full size, group pushed inward
const svg = d3.select("#chart")
  .append("svg")
  .attr("viewBox", `0 0 700 400`)
  .append("g")
  .attr("transform", `translate(${margin.left}, ${margin.top})`);

// Now (0,0) in the group = top-left of chart area
// Scales use width and height (inner dimensions)
// Axes fit in the margin space

Live: Margins visualized

Orange = chart area where data goes. Gray = margins where axes and labels live.

Copy-paste this pattern into every chart. It is boilerplate. Every lesson from here on starts with it. margin → width/height → SVG → group. Once set up, everything you draw goes inside the group, and margins handle themselves.

Complete Bar Chart: Everything Together

Margins + scales + axes + data binding + bars. This is the template for every bar chart you will ever make:

Live: Median household income by state (complete chart)

// The complete pattern:
// 1. Margins
const margin = { top: 20, right: 20, bottom: 50, left: 60 };
const width = 700 - margin.left - margin.right;
const height = 380 - margin.top - margin.bottom;

// 2. SVG + group
const svg = d3.select("#chart")
  .append("svg").attr("viewBox", "0 0 700 380")
  .append("g").attr("transform", `translate(${margin.left},${margin.top})`);

// 3. Scales
const x = d3.scaleBand().domain(data.map(d => d.state)).range([0, width]).padding(0.2);
const y = d3.scaleLinear().domain([0, d3.max(data, d => d.income)]).nice().range([height, 0]);

// 4. Axes
svg.append("g").attr("transform", `translate(0,${height})`).call(d3.axisBottom(x));
svg.append("g").call(d3.axisLeft(y).tickFormat(d => `$${d/1000}k`));

// 5. Grid lines (just long ticks)
svg.append("g").call(d3.axisLeft(y).tickSize(-width).tickFormat(""))
  .selectAll("line").attr("stroke", "#1a1a1a");

// 6. Bars
svg.selectAll("rect").data(data).join("rect")
  .attr("x", d => x(d.state)).attr("y", d => y(d.income))
  .attr("width", x.bandwidth()).attr("height", d => height - y(d.income))
  .attr("fill", "#f97316");

Check Your Understanding

1. Where does an axis get its tick positions and labels from?

From the scale you pass to it. d3.axisLeft(yScale) reads the scale's domain and range to determine where ticks go and what values to display. This is why you create the scale first, then the axis from it.

2. What does .call(axisGenerator) actually do?

It passes the current selection (a <g> group) to the axis generator function. The axis function then creates multiple child elements inside that group: a <path> for the axis line, <line> elements for ticks, and <text> elements for labels. It is equivalent to axisGenerator(selection).

3. How do you create grid lines in D3?

Grid lines are just extended ticks. Create an axis with .tickSize(-width) (for horizontal grid lines on a left axis) or .tickSize(-height) (for vertical grid lines on a bottom axis). Use .tickFormat("") to hide the labels since you only want the lines. Then style them with a subtle color.

4. What does the margins convention give you?

Room for axes and labels around the chart area. The SVG is created at full size, then a <g> group is pushed inward by the margins using translate(). All drawing happens inside the group. Axes fit in the margin space. Scales use the inner width/height, not the full SVG dimensions.

5. Why do we position the X-axis with translate(0, height)?

Because d3.axisBottom() draws ticks below the axis line, but the group starts at (0,0) which is the top-left of the chart area. We need to push the axis down to the bottom of the chart area, which is at y = height. Without the translate, the X-axis would appear at the top.

6. How do you format Y-axis labels as currency (e.g., "$50k")?

Use .tickFormat() on the axis generator: d3.axisLeft(y).tickFormat(d => "$" + d/1000 + "k"). Or use D3's built-in formatter: .tickFormat(d3.format("$,.0f")). The .tickFormat() method receives each tick value and returns the string to display.
