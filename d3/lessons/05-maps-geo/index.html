05 Scales — The Translator

← Back to lessons

Mental Model: A scale is a translator between two languages. Your data speaks "millions of people" or "dollars." Your screen speaks "pixels." A scale converts one to the other, proportionally.

The Problem

In Lesson 04, we wrote things like d.pop / 150 to convert population to bar height. That is fragile. Change the data or the chart size and everything breaks. A scale solves this permanently.

// Bad: magic numbers
.attr("width", d => d.pop / 100)

// Good: scale handles the math
.attr("width", d => xScale(d.pop))

Domain and Range: The Two Settings

Every scale needs two things:

  • Domain = the input range (data world). "My data goes from 0 to 40 million."
  • Range = the output range (pixel world). "My chart area is 0 to 600 pixels wide."
const scale = d3.scaleLinear()
  .domain([0, 40_000_000])   // data: 0 to 40M
  .range([0, 600]);           // pixels: 0 to 600

// Now it's a function:
scale(40_000_000)   // → 600   (max data = max pixels)
scale(20_000_000)   // → 300   (half data = half pixels)
scale(0)            // → 0     (min data = min pixels)

A scale is just a function. You call it with a data value, it returns a pixel value. That is the entire concept. d3.scaleLinear() creates the function; .domain() and .range() configure it.

Scale Types: Pick the Right Translator

1 scaleLinear — continuous numbers

Population, money, temperature, percentages. Any number in a continuous range.

const y = d3.scaleLinear()
  .domain([0, 100])     // voter turnout: 0% to 100%
  .range([height, 0]);  // FLIPPED: 0% at bottom, 100% at top

Why flip the range? SVG Y-axis goes top-down (0 = top). We want high values at the top of the chart. Flipping the range ([height, 0]) makes the scale put 0% at the bottom (large Y pixel) and 100% at the top (small Y pixel). You will do this for every Y-axis scale.

2 scaleBand — categories (bar charts)

State names, category labels, anything discrete. Each item gets its own "band" of space.

const x = d3.scaleBand()
  .domain(["CA", "TX", "FL", "NY", "PA"])
  .range([0, width])
  .padding(0.2);          // 20% gap between bars

x("CA")            // → 0     (left edge of CA's band)
x("TX")            // → 124   (left edge of TX's band)
x.bandwidth()      // → 99    (width of each bar)

scaleBand is specifically for bar charts. It divides space evenly, gives you .bandwidth() for bar width, and handles padding. No more manually calculating i * barWidth.

3 scaleTime — dates

const x = d3.scaleTime()
  .domain([new Date("2020-01-01"), new Date("2024-12-31")])
  .range([0, width]);

4 scaleSequential — data to colors

Instead of data → pixels, it maps data → colors. Essential for choropleth maps and heatmaps.

const color = d3.scaleSequential()
  .domain([0, 100])
  .interpolator(d3.interpolateBlues);

color(0)    // → very light blue
color(100)  // → dark blue

5 scaleSqrt — area-proportional sizing

// For bubble charts: use sqrt so AREA scales linearly
const r = d3.scaleSqrt()
  .domain([0, 40_000_000])
  .range([2, 40]);  // min/max bubble radius

6 scaleOrdinal — categories to discrete values

// Map categories to specific colors
const color = d3.scaleOrdinal()
  .domain(["Democrat", "Republican", "Independent"])
  .range(["#3b82f6", "#ef4444", "#a78bfa"]);

Essential Scale Methods

// .nice() — round domain to clean numbers
y.domain([3.2, 97.8]).nice()  // → domain becomes [0, 100]

// .invert() — reverse: pixel → data (linear/time only)
xScale.invert(300)            // → 20,000,000

// .clamp(true) — prevent out-of-range outputs
scale.clamp(true)
scale(50_000_000)             // → 600 (capped at max range)

Check Your Understanding

1. In your own words, what does a D3 scale do?

A scale is a function that converts values from one world (your data: populations, dollars, dates) to another world (your screen: pixels or colors). You tell it the input range (domain) and output range (range), and it handles all the proportional math for you.

2. Why do we write .range([height, 0]) for Y scales instead of .range([0, height])?

Because SVG Y-axis goes top-down (0 = top of SVG). Small data values should map to the bottom of the chart (large Y pixel value) and large data values to the top (small Y pixel value). Flipping the range does this automatically. You will always do this for Y-axis scales.

3. When would you use scaleBand vs scaleLinear?

scaleBand for categories (state names, labels). It divides space into equal bands and gives you .bandwidth(). scaleLinear for continuous numbers (population, money). It maps proportionally. If your domain is strings, use band. If it is numbers, use linear.

4. You need to size circles proportionally to population data. Why use scaleSqrt instead of scaleLinear?

Because circle area = pi * r^2. If you linearly map population to radius, a city 4x larger gets a circle 16x larger (by area), which is visually misleading. scaleSqrt takes the square root, so the visible area scales linearly with the data value.

5. What does .nice() do on a scale?

It rounds the domain to "nice" values. If your data ranges from 3.2 to 97.8, .nice() extends the domain to [0, 100]. This makes axis tick labels cleaner and the chart more readable. Always call it after setting the domain.

6. How would you create a color scale for a choropleth map showing poverty rates from 5% to 25%?

const color = d3.scaleSequential(d3.interpolateReds).domain([5, 25]); Then use it: color(15) returns a medium-red color. You can also use d3.interpolateBlues, d3.interpolateViridis (colorblind-safe), or any D3 color interpolator.

Recap

  1. Scales are translator functions — data world → pixel/color world
  2. Domain = input range, Range = output range
  3. scaleLinear for numbers, scaleBand for categories, scaleTime for dates
  4. Y range is always flipped: [height, 0]
  5. .nice() cleans up domain boundaries
  6. scaleSequential maps to colors, scaleSqrt for proportional area
