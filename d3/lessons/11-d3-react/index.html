Mental Model: D3 events are just callbacks, same as React's onClick. Instead of <div onClick={handleClick}>, it is .on("click", handleClick). Same concept, different syntax.

Mouse Events in D3 v7

// D3 v7 pattern: event is the FIRST argument
selection.on("mouseenter", function(event, d) {
  console.log(event);   // native DOM event (has pageX, pageY, etc.)
  console.log(d);        // bound data (the __data__ sticky note)
  console.log(this);     // the DOM element (use d3.select(this))
});

// Common events:
.on("mouseenter", (event, d) => { })   // hover start
.on("mousemove",  (event, d) => { })   // cursor tracking
.on("mouseleave", (event, d) => { })   // hover end
.on("click",      (event, d) => { })   // click

D3 v7 breaking change: In older D3 (v5 and earlier), events were accessed via the global d3.event. In v7, the event is passed as the first argument to the handler. If you see d3.event in old code on the web, know it will not work in v7. This comes up in interviews.

Building Tooltips

Tooltips are HTML divs positioned near the cursor. Use an HTML div (not SVG text) because divs support rich formatting, word wrap, and are easier to style.

// Create tooltip div outside the SVG
const tooltip = d3.select("body").append("div")
  .style("position", "fixed")
  .style("display", "none")
  .style("background", "#1a1a1a")
  .style("border", "1px solid #f97316")
  .style("padding", "0.5rem 0.75rem")
  .style("border-radius", "4px")
  .style("pointer-events", "none");

// Attach to chart elements
bars
  .on("mouseenter", function(event, d) {
    d3.select(this).attr("opacity", 0.8);
    tooltip.style("display", "block")
      .html(`<strong>${d.name}</strong><br>${d.value}`);
  })
  .on("mousemove", (event) => {
    tooltip
      .style("left", (event.clientX + 12) + "px")
      .style("top", (event.clientY - 28) + "px");
  })
  .on("mouseleave", function() {
    d3.select(this).attr("opacity", 1);
    tooltip.style("display", "none");
  });

Transitions

// Insert .transition() to animate any attribute change
selection
  .transition()
  .duration(800)               // animation length in ms
  .delay((d, i) => i * 50)    // stagger by index
  .ease(d3.easeCubicInOut)     // easing curve
  .attr("height", d => yScale(d.value))
  .attr("fill", "#f97316");

// Chain transitions (second waits for first)
selection
  .transition().duration(500).attr("height", h)
  .transition().duration(300).attr("fill", "green");

Easing Functions

  • d3.easeCubicInOut — natural, smooth acceleration/deceleration (default-ish feel)
  • d3.easeLinear — constant speed (use for progress bars)
  • d3.easeBounce — bouncy ending (use sparingly)
  • d3.easeElastic — springy overshoot (playful UIs only)
  • d3.easeCubicOut — fast start, slow end (good for entrance animations)

Responsive Charts with viewBox

// The key insight: viewBox makes SVG responsive like an image
const svg = d3.select("#chart")
  .append("svg")
  .attr("viewBox", `0 0 700 400`)  // virtual 700x400 canvas
  // Do NOT set width/height attrs - let CSS handle it
  .style("width", "100%")
  .style("max-width", "800px");

// The SVG scales to fill its container while maintaining
// the 700:400 aspect ratio. All coordinates stay the same.

viewBox is almost always sufficient. It handles 90% of responsive needs. Only reach for ResizeObserver when you need layout changes at different sizes (e.g., switching bar chart orientation on mobile).

Accessibility Basics

  • Add role="img" and aria-label="Description" to the SVG
  • Add <title> and <desc> elements inside the SVG
  • Make interactive elements focusable with tabindex="0"
  • Add keyboard event handlers (keydown) alongside mouse events
  • Never rely on color alone — add patterns, labels, or tooltips

Check Your Understanding

1. In D3 v7, how do you access the mouse event in an event handler?

The event is the first argument to the handler: .on("click", (event, d) => { ... }). In older versions (v5 and earlier), the global d3.event was used instead. This is one of the most common breaking changes between D3 versions.

2. Why use an HTML <div> for tooltips instead of SVG <text>?

HTML divs support rich formatting (bold, line breaks, HTML), automatic word wrapping, easy CSS styling, and position: fixed for cursor-following. SVG text does not wrap, cannot contain HTML, and gets clipped by the SVG viewport. Always place the tooltip div outside the SVG.

3. What does .delay((d, i) => i * 50) do in a transition?

It creates a staggered animation. Each element waits i * 50 milliseconds before starting its transition, where i is its index. So the first element starts immediately, the second after 50ms, the third after 100ms, etc. This creates a cascading "wave" effect.

4. What does the viewBox attribute do?

viewBox defines a virtual coordinate system. viewBox="0 0 700 400" means you draw as if the canvas is 700x400, regardless of the actual rendered size. The browser scales everything to fit the element's CSS dimensions while maintaining the aspect ratio. This is the simplest way to make D3 charts responsive.

5. How do you chain two transitions sequentially?

Call .transition() twice in the chain: selection.transition().duration(500).attr("height", h).transition().duration(300).attr("fill", "green"). The second transition automatically starts after the first completes. You can also use .on("end", fn) for more complex sequencing.

6. Name two accessibility improvements you should add to every D3 chart.

1. Add role="img" and aria-label to the SVG for screen readers. 2. Never rely on color alone: add text labels, patterns, or tooltips so the data is accessible without color perception. Other good practices: <title>/<desc> inside SVG, tabindex="0" on interactive elements, keyboard event handlers.
