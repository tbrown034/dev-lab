09 Arcs, Pies & Radial Charts

← Back to lessons

Mental Model: Two tools, two jobs. d3.pie() computes the angles (divides the circle). d3.arc() draws the slices (generates path strings from angles). They always work together.

The Two-Step Process

// Step 1: pie() transforms data into angles
const pie = d3.pie()
  .value(d => d.amount)    // which field to use for proportions
  .sort(null);             // keep original order (don't re-sort)

const arcsData = pie(data);
// Each item now has: { startAngle, endAngle, data, value }

// Step 2: arc() generates SVG path strings from angles
const arc = d3.arc()
  .innerRadius(0)          // 0 = pie, > 0 = donut
  .outerRadius(150);

// Step 3: Combine them
svg.selectAll("path")
  .data(arcsData)          // data is the pie-computed angles
  .join("path")
  .attr("d", arc)          // arc() reads startAngle/endAngle
  .attr("fill", (d, i) => colors[i]);

pie() does NOT draw anything. It only computes angles. arc() does NOT know about your data. It only reads angles. Separating computation from drawing is a D3 design principle — it gives you full control over layout and appearance independently.

Donut vs Pie

// Pie chart: innerRadius = 0
d3.arc().innerRadius(0).outerRadius(150)

// Donut chart: innerRadius > 0
d3.arc().innerRadius(70).outerRadius(150)

// The center space is great for summary text

Centroid for Labels

// arc.centroid() returns [x, y] of the midpoint of the arc
// Perfect for positioning labels
const labelArc = d3.arc()
  .innerRadius(outerRadius * 0.7)
  .outerRadius(outerRadius * 0.7);

svg.selectAll("text")
  .data(arcsData)
  .join("text")
  .attr("transform", d => `translate(${labelArc.centroid(d)})`)
  .text(d => d.data.label);

Check Your Understanding

1. What does d3.pie() return when you call it with data?

An array of objects, each with startAngle, endAngle, data (original datum), value, index, and padAngle. The angles are in radians and sum to 2*pi. These angle objects are what d3.arc() reads to draw slices.

2. How do you make a donut chart instead of a pie chart?

Set innerRadius to a value greater than 0 in the arc generator: d3.arc().innerRadius(70).outerRadius(150). When innerRadius = 0, you get a full pie. When it is > 0, you get a donut with a hole in the center.

3. Why pass .sort(null) to d3.pie()?

By default, d3.pie() sorts slices by descending value. .sort(null) tells it to keep the original data order. This is important when the data has a meaningful order (e.g., chronological, or matching a legend) or when you want consistent positioning across updates.

4. How do you position labels at the center of each arc slice?

Use arc.centroid(d) which returns an [x, y] array for the midpoint of the arc. Create a label-specific arc generator with radius set to where you want labels (e.g., 70% of outer radius), then use translate(${labelArc.centroid(d)}) to position text elements.

5. What does .padAngle(0.02) do?

It adds a small gap (in radians) between adjacent slices, making them visually distinct. Combined with .cornerRadius(4), it gives the donut chart a polished, modern look. The padding is subtracted from each slice proportionally.

6. How do you animate arcs growing from 0 to their final angle?

Use attrTween("d", ...) with a custom interpolator. Start each arc's endAngle at its startAngle (0 size), then interpolate to the final endAngle. The arc generator is called at each frame of the transition to produce the intermediate path string.

Exercise: Build a Donut Chart from Donation Data

Create a donut chart showing campaign donation sources:

const donations = [
  { source: "Individual <$200", amount: 1_250_000 },
  { source: "Individual $200-$2900", amount: 2_800_000 },
  { source: "PAC", amount: 950_000 },
  { source: "Party Committee", amount: 400_000 },
  { source: "Self-Funded", amount: 500_000 },
];
  1. Use d3.pie().value(d => d.amount) to compute angles
  2. Use d3.arc() with innerRadius > 0 for donut style
  3. Add percentage labels using arc centroid positioning
  4. Put the total raised amount in the center
  5. Add a legend below the chart
  6. Animate the arcs appearing one by one with delay

Recap

  1. d3.pie() computes angles from data values
  2. d3.arc() draws SVG paths from angles
  3. innerRadius = 0 is pie, > 0 is donut
  4. arc.centroid() gives the midpoint for label positioning
  5. .padAngle() and .cornerRadius() for polish
  6. attrTween enables smooth arc animations
