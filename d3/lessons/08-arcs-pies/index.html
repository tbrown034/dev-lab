08 Lines, Areas & Time Series

← Back to lessons

Mental Model: Path generators are cookie cutters. You give them data (the dough), and they stamp out an SVG path string (the shape). d3.line() makes a line shape; d3.area() makes a filled shape. You never write path commands by hand.

d3.line() — One Path From Many Points

// A line generator is a function factory
const line = d3.line()
  .x(d => xScale(d.date))     // how to get X from each data point
  .y(d => yScale(d.value));    // how to get Y from each data point

// Feed it data, get back a path string
line(data)  // → "M0,150 L50,120 L100,80 L150,90..."

// Use it to set the "d" attribute of a <path>
svg.append("path")
  .datum(data)                  // .datum() not .data()!
  .attr("d", line)              // line(data) runs automatically
  .attr("fill", "none")
  .attr("stroke", "#f97316")
  .attr("stroke-width", 2);

.datum() vs .data()

Key distinction: .data(array) = one element per array item (bars, circles). .datum(array) = one element from the entire array (a single line or area path). A line chart is ONE <path> element that uses ALL the data points, so you bind with .datum().

d3.area() — Filled Region

const area = d3.area()
  .x(d => xScale(d.date))
  .y0(height)                   // bottom edge (baseline)
  .y1(d => yScale(d.value))     // top edge (data line)
  .curve(d3.curveMonotoneX);

svg.append("path")
  .datum(data)
  .attr("d", area)
  .attr("fill", "#f9731622")    // semi-transparent fill
  .attr("stroke", "none");

Curve Types

The .curve() method controls how points are connected:

  • curveLinear — straight line segments (default, most accurate)
  • curveMonotoneX — smooth, preserves monotonicity (best for time series)
  • curveStep — staircase pattern (good for discrete values)
  • curveBasis — very smooth but doesn't pass through points (aesthetic only)

For journalism, use curveLinear or curveMonotoneX. Basis and cardinal curves create values between data points that never existed in the data. For factual reporting, that is misleading. curveMonotoneX is the safe "smooth" option — it passes through every data point.

Check Your Understanding

1. What is the difference between .datum(data) and .data(data)?

.datum(data) binds the entire dataset to a single element (used for line/area paths). .data(data) binds each array item to a separate element (used for bars, circles). A line chart is one path element using all points, so it needs .datum().

2. What does d3.line() actually return?

A function. When you call that function with an array of data, it returns an SVG path string (the d attribute). The .x() and .y() accessors tell it how to extract coordinates from each data point.

3. In d3.area(), what do .y0() and .y1() define?

.y0() defines the bottom edge of the filled area (usually the baseline at height). .y1() defines the top edge (your data values). The area is the filled region between y0 and y1. For a confidence band, both y0 and y1 can be data-driven.

4. Why should journalists prefer curveMonotoneX over curveBasis?

curveMonotoneX passes through every data point and preserves monotonicity (no false peaks). curveBasis creates a smooth line that does NOT pass through the data points — it can show values that never existed. For factual reporting, creating phantom data points is unacceptable.

5. How do you create an animated "line drawing" effect?

Use SVG's stroke-dasharray and stroke-dashoffset. Set both to the path's total length (via path.node().getTotalLength()), making the line invisible. Then transition stroke-dashoffset to 0, which "draws" the line from start to end.

6. You need to show the difference between two data series (e.g., projected vs actual). Which generator works best?

d3.area() with .y0() set to one series and .y1() set to the other. This fills the gap between the two lines, visually highlighting the difference. This is commonly used for confidence bands, margin of error, and projected vs. actual comparisons.

Recap

  1. d3.line() takes data, returns a path string
  2. d3.area() fills between two boundaries (y0 and y1)
  3. .datum() binds entire array to one element (vs .data() for many)
  4. curveMonotoneX is the safe smooth curve for journalism
  5. Animated drawing uses stroke-dasharray/dashoffset trick
