04 Data Binding — The ONE Concept

← Back to lessons

Mental Model: Data binding is a forEach loop with superpowers. Think of it as attaching a sticky note (data) to each DOM element, then using that sticky note to control the element's appearance.

The Problem Data Binding Solves

Say you have population data for 5 states and you want a bar for each. Without data binding, you would write:

const data = [
  { state: "CA", pop: 39538 },
  { state: "TX", pop: 29146 },
  { state: "FL", pop: 21538 },
  { state: "NY", pop: 20201 },
  { state: "PA", pop: 13002 },
];

// The forEach approach (what you know)
data.forEach((d, i) => {
  svg.append("rect")
    .attr("x", i * 100)
    .attr("y", 300 - d.pop / 150)
    .attr("width", 80)
    .attr("height", d.pop / 150)
    .attr("fill", "#f97316");
});

This works, but it has a problem: there is no link between the data and the elements. If the data changes, you have to manually find, update, or remove bars. D3's data binding creates that permanent link.

What .data() Actually Does

When you call .data(array), D3 does something invisible but powerful: it takes each item in the array and physically attaches it to a DOM element using a hidden property called __data__.

// After data binding, each rect has a hidden property:
rect.__data__  // → { state: "CA", pop: 39538 }

// You can see it! Open DevTools, select a rect, and type:
$0.__data__    // → the data object bound to that element

This is the sticky note. Each DOM element literally carries its data around. When you write .attr("width", d => d.pop), that d parameter is the sticky note glued to that specific element. D3 just reads the __data__ property and passes it to your function.

The Pattern: selectAll → data → join

svg.selectAll("rect")   // 1. Select all existing rects (probably 0)
  .data(data)            // 2. Bind the data array
  .join("rect")          // 3. Create/update/remove rects to match data
  .attr("x", (d, i) => i * 100)  // 4. Use bound data to set attributes
  .attr("width", 80)
  .attr("height", d => d.pop / 150)
  .attr("fill", "#f97316");

1 selectAll("rect") — declare intent

This usually selects zero elements on first run. That is fine. You are telling D3: "I want rects to represent my data." It is a declaration of intent, not a selection of existing elements.

Why select things that don't exist? This is the most confusing part of D3 for newcomers. Think of it like React's return <div>{items.map(...)}</div>. You are declaring what the DOM should look like, not finding what's there now. D3 uses the empty selection as a starting point to figure out what needs to be created.

2 .data(data) — match data to elements

D3 compares the data array to the existing elements by index. 5 data items, 0 rects = 5 new rects needed.

3 .join("rect") — create, update, or remove

This is the magic. .join() creates missing elements, keeps existing ones, and removes extras. After .join("rect"), you have exactly one rect per data item, each with its __data__ attached.

4 Accessor functions: (d, i) => ...

Any method that takes a value can also take a function. D3 calls it once per element:

  • d = the data bound to this element (the sticky note)
  • i = the index in the data array

Side by Side: forEach vs D3

forEach (manual)
data.forEach((d, i) => {
  svg.append("rect")
    .attr("x", i * 100)
    .attr("y", 300 - d.pop/150)
    .attr("width", 80)
    .attr("height", d.pop/150)
    .attr("fill", "#f97316");
});
// No data link to elements
// Can't easily update later
D3 data binding
svg.selectAll("rect")
  .data(data)
  .join("rect")
  .attr("x", (d,i) => i * 100)
  .attr("y", d => 300 - d.pop/150)
  .attr("width", 80)
  .attr("height", d => d.pop/150)
  .attr("fill", "#f97316");
// Data glued to elements
// Re-run with new data = auto-update

The superpower: If you call the D3 version again with different data, it automatically updates existing bars, creates new ones, and removes old ones. The forEach version would just keep appending duplicates. This is what makes D3's data binding fundamentally different.

Check Your Understanding

1. What does .data() actually do to a DOM element?

It attaches each item from the data array to a DOM element via a hidden __data__ property. This creates a persistent link between the data and the element. You can verify this in DevTools: select an element and check $0.__data__.

2. Why does selectAll("rect") select zero elements, and why is that OK?

On first render, there are no rects yet. The empty selection is a declaration of intent: "I want rects to represent my data." .data() compares 5 data items to 0 existing elements and determines 5 need to be created. .join("rect") then creates those 5 rects. This is the "enter" phase.

3. In .attr("width", d => d.pop / 100), where does d come from?

d is the data item bound to the current element. D3 reads the element's __data__ property and passes it as the first argument to your accessor function. The second argument i is the index. So for the 3rd bar, d is data[2] and i is 2.

4. What happens if you call the data binding pattern again with different data?

.join() handles it automatically: existing elements get updated with new data (update), new elements are created if the new array is longer (enter), and extra elements are removed if the new array is shorter (exit). This is why D3's data binding is more powerful than a simple forEach loop.

5. What is the difference between a static value and an accessor function in .attr()?

A static value like .attr("width", 80) sets the same value for every element. An accessor function like .attr("width", d => d.pop / 100) is called once per element, receiving the bound data, so each element can have a different value. Use static values for shared properties (like color), functions for data-driven properties (like bar width).

6. How would you verify that data is correctly bound to a DOM element?

Open DevTools, select the element in the Elements panel, then in the Console type $0.__data__. This shows the data object D3 attached. You can also use d3.select(element).datum() to access the bound data from code. This is an essential debugging technique.

Exercise: Build a Bar Chart from Data

Using the data binding pattern, create a horizontal bar chart from this data:

const data = [
  { state: "California", pop: 39538, abbr: "CA" },
  { state: "Texas", pop: 29146, abbr: "TX" },
  { state: "Florida", pop: 21538, abbr: "FL" },
  { state: "New York", pop: 20201, abbr: "NY" },
  { state: "Pennsylvania", pop: 13002, abbr: "PA" },
  { state: "Illinois", pop: 12812, abbr: "IL" },
  { state: "Ohio", pop: 11799, abbr: "OH" },
  { state: "Georgia", pop: 10711, abbr: "GA" },
];
  1. Use selectAll("rect").data(data).join("rect") to create bars
  2. Use accessor functions to set bar height based on d.pop
  3. Add text labels using a second selectAll("text").data(data).join("text")
  4. Show the state abbreviation below each bar
  5. Bonus: Add value labels above each bar

Practice in your browser console on this page, or create a new file. The key is to write the pattern from memory: selectAll → data → join → attrs.

Recap

  1. .data() attaches data to DOM elements via the hidden __data__ property
  2. selectAll → data → join is the core pattern
  3. Accessor functions (d, i) => ... make each element data-driven
  4. Empty selectAll is a declaration of intent, not a bug
  5. .join() handles creation and cleanup automatically
  6. Re-running with new data updates the chart — no manual DOM management
