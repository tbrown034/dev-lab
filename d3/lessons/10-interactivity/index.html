Mental Model: A projection is just another scale. Scales translate data to pixels; projections translate latitude/longitude to pixels. d3.geoPath() is just another path generator, like d3.line() but for geography.

The Pipeline

  1. Geographic data (GeoJSON/TopoJSON) — shapes as coordinate arrays
  2. Projection — flattens the globe (like peeling an orange flat)
  3. Path generator — converts projected coordinates to SVG path strings
import * as topojson from "topojson-client";

// 1. Load geographic data
const us = await d3.json(
  "https://cdn.jsdelivr.net/npm/us-atlas@3/states-10m.json"
);
const states = topojson.feature(us, us.objects.states);

// 2. Create projection (like a scale for coordinates)
const projection = d3.geoAlbersUsa()
  .fitSize([width, height], states);

// 3. Create path generator (like d3.line() for geography)
const path = d3.geoPath(projection);

// 4. Draw!
svg.selectAll("path")
  .data(states.features)
  .join("path")
  .attr("d", path)                // path generator creates "d" attribute
  .attr("fill", d => color(getData(d)))
  .attr("stroke", "#333");

GeoJSON vs TopoJSON

  • GeoJSON = standard format. Each feature has coordinates. Simple but large files (2MB+ for US states).
  • TopoJSON = compressed format. Shares borders between adjacent features. 80-90% smaller (~200KB). Convert to GeoJSON in the browser with topojson.feature().

Projections

  • d3.geoAlbersUsa() — US map with Alaska/Hawaii insets. Use this for US maps.
  • d3.geoMercator() — Google Maps projection. Distorts area near poles.
  • d3.geoEqualEarth() — Modern world projection. Equal area.

.fitSize() is your best friend. Instead of manually calculating scale and translate, call projection.fitSize([width, height], geojsonData). It automatically scales and centers the map to fit your SVG dimensions.

Choropleth Pattern

// Create a lookup map: FIPS code → data value
const dataMap = new Map(data.map(d => [d.fips, d.value]));

// Color scale
const color = d3.scaleSequential(d3.interpolateYlOrRd)
  .domain(d3.extent(data, d => d.value));

// Color each state by its data value
svg.selectAll("path")
  .data(states.features)
  .join("path")
  .attr("d", path)
  .attr("fill", d => {
    const val = dataMap.get(d.id);
    return val ? color(val) : "#333";
  });

Check Your Understanding

1. What does topojson.feature(us, us.objects.states) do?

It converts TopoJSON data to GeoJSON format. TopoJSON is a compressed format that shares boundaries between adjacent features. topojson.feature() expands it into standard GeoJSON FeatureCollection that D3's path generator can render.

2. Why use d3.geoAlbersUsa() for US maps?

It is specifically designed for US maps: includes insets for Alaska and Hawaii, uses an equal-area projection so state sizes are proportional, and is centered on the US. geoMercator severely distorts Alaska's size and does not handle the insets.

3. How is a projection similar to a scale?

Both are translator functions. A scale maps data values (domain) to pixels (range). A projection maps geographic coordinates (latitude/longitude) to pixel coordinates (x/y). Both take input from one "world" and produce output in the pixel world. .fitSize() is like setting domain and range at once.

4. What is a choropleth map?

A thematic map where geographic areas are colored according to a data variable. For example, coloring states by median income or voter turnout. The color intensity represents the data magnitude. The standard pattern is: data lookup map + sequential color scale + fill each path by its data value.

5. What does topojson.mesh(us, us.objects.states, (a, b) => a !== b) do?

It generates a single path for all internal state borders (where two states share a boundary). The filter function (a, b) => a !== b excludes exterior borders. This gives you clean, non-overlapping border lines without the visual artifacts of per-state strokes.

6. How do you make a state map responsive?

Use viewBox on the SVG element and .fitSize() on the projection. The viewBox creates a virtual coordinate system that scales with the container. .fitSize([width, height], geojson) ensures the map fills that space. No fixed pixel dimensions needed.

  1. Projection = a scale for coordinates (lat/lng to pixels)
  2. d3.geoPath() = path generator for geography
  3. TopoJSON is smaller, convert with topojson.feature()
