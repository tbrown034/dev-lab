03 D3 Selections Deep Dive

← Back to lessons

Mental Model: D3 selections are like jQuery but for SVG. If you ever used $(".class").css("color", "red"), you already understand the core idea. D3 just does the same thing for SVG attributes.

The Core Idea

In React, you describe what the UI should look like and React figures out how to update the DOM. D3 is the opposite: you directly tell the DOM what to do, step by step.

A selection is a wrapper around one or more DOM elements that gives you a chainable API for manipulating them. Think of it as a "handle" you grab onto elements with.

jQuery (you know this)
$("circle")
  .css("fill", "orange")
  .attr("r", 20);
D3 (same pattern)
d3.selectAll("circle")
  .style("fill", "orange")
  .attr("r", 20);

Every D3 method returns the selection itself. That is how method chaining works. .attr() sets the attribute AND returns the selection so you can call .style() next. This is the same pattern as jQuery, Lodash chains, or even array methods like .filter().map().

d3.select() vs d3.selectAll()

// Select ONE element (first match)
d3.select("#my-chart")      // by ID
d3.select("svg")            // first SVG on the page
d3.select(".highlight")     // first element with class

// Select ALL matching elements
d3.selectAll("circle")      // every circle in the SVG
d3.selectAll(".bar")        // every element with class "bar"
d3.selectAll("rect, circle")// CSS multi-selector

These use standard CSS selectors, exactly like document.querySelector() and document.querySelectorAll(). The difference is that D3 wraps the result in a selection object with its chainable API.

Scoped selections

You can chain selections to scope them, just like element.querySelector():

// Select circles inside #chart only
d3.select("#chart").selectAll("circle")

// This is equivalent to:
document.querySelectorAll("#chart circle")

The Essential Methods

.attr() — Set SVG attributes

This is the workhorse of D3. SVG elements are controlled by attributes.

d3.select("rect")
  .attr("x", 10)
  .attr("y", 20)
  .attr("width", 200)
  .attr("height", 50)
  .attr("fill", "#f97316")
  .attr("rx", 4);           // corner radius

.style() — Set CSS styles

d3.select("rect")
  .style("opacity", 0.8)
  .style("cursor", "pointer");

.attr("fill") vs .style("fill"): Both work for properties like fill and stroke, but .attr() sets the SVG attribute directly (appears in the markup), while .style() sets an inline CSS style (appears in the style="" attribute). CSS styles override SVG attributes. For SVG-specific properties like x, width, r, d, you must use .attr(). For CSS-only properties like cursor, opacity, use .style(). When in doubt, use .attr() for SVG.

.text() and .html()

d3.select("text").text("Hello");       // set text content
d3.select("div").html("<b>Bold</b>"); // set inner HTML (divs only, not SVG)

.property() — for form elements

d3.select("input").property("value", "42");
d3.select("input").property("checked", true);

.classed() — toggle CSS classes

d3.select("rect").classed("active", true);   // add class
d3.select("rect").classed("active", false);  // remove class

// Toggle based on condition
d3.selectAll("rect").classed("highlighted", d => d.value > 50);

.append() and .remove()

// Add a new element as the last child
const circle = d3.select("svg")
  .append("circle")         // creates <circle> inside the SVG
  .attr("cx", 100)
  .attr("cy", 50)
  .attr("r", 20);

// Remove elements
d3.selectAll(".old").remove();

.each() — side effects

d3.selectAll("circle").each(function(d, i) {
  console.log(i, this);  // index, DOM node
  // "this" is the raw DOM element
});

Inspection methods

const sel = d3.selectAll("circle");
sel.empty()    // true if no elements matched
sel.size()     // number of elements in selection
sel.node()     // first raw DOM element
sel.nodes()    // array of all raw DOM elements

Check Your Understanding

1. What is the difference between d3.select() and d3.selectAll()?

d3.select() selects the first matching element. d3.selectAll() selects all matching elements. Both use CSS selectors. Think of it as querySelector vs querySelectorAll.

2. Why does method chaining work in D3?

Because almost every selection method (.attr(), .style(), .text(), etc.) returns the selection itself after modifying it. This lets you write sel.attr("x", 10).attr("y", 20).style("fill", "red") instead of calling each method on a separate line.

3. When should you use .attr("fill", "red") vs .style("fill", "red")?

.attr() sets the SVG attribute directly in the element's markup. .style() sets an inline CSS style. CSS styles override SVG attributes, so if both are set, the .style() wins. For SVG-only attributes like x, width, r, d, you must use .attr(). In practice, most D3 code uses .attr() for SVG drawing properties.

4. What does .append("circle") return?

A new selection containing the newly created <circle> element. This is important: after .append(), subsequent chained methods operate on the new child element, not the parent. So svg.append("circle").attr("r", 10) sets the radius on the new circle, not on the SVG.

5. How do you check if a selection matched any elements?

Use selection.empty(). It returns true if the selection contains no elements. You can also use selection.size() to get the count. These are inspection methods that return values instead of the selection, so they break the chain.

6. You need to add a CSS class to all rects with a value above 50. How?

d3.selectAll("rect").classed("high-value", d => d.value > 50). The .classed() method can take a function as the second argument. When the function returns true, the class is added; when false, it is removed. This works because each rect has data bound to it (which we will learn in the next lesson).

Recap

  1. Selections wrap DOM elements with a chainable API
  2. d3.select() = first match, d3.selectAll() = all matches
  3. .attr() for SVG attributes, .style() for CSS properties
  4. .append() creates children, .remove() destroys elements
  5. Method chaining works because every method returns the selection
  6. .classed() for toggling CSS classes
  7. .empty(), .size(), .node(), .nodes() for inspecting selections
