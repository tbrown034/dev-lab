This is not a course you watch. It's a learning environment you build inside of. Every lesson, game, sandbox, and AI tutor here was designed around one question: how does my brain actually learn?

The rule: mental models first, syntax second. You won't memorize API signatures here. You'll learn the why behind every function so you can derive the how on the fly.

What You'll Be Able to Do

After 12 lessons, you should be able to:

Read any D3 code like English

See a block of D3 and immediately understand what it draws, how data flows through it, and why each line exists.

Build standard charts from memory

Bar, line, scatter, area, pie, donut, map — without looking anything up.

Build broadcast-ready dashboards

Election maps, weather overlays, census visualizations — the kind of thing that goes on TV.

Integrate D3 into React

Know when D3 should own the DOM, when React should, and how to make them coexist.

How Lessons Work

Each lesson follows the same structure:

  1. Mental model first. An analogy or concept that makes the new idea click before you see any code.
  2. Core concepts. The 2–3 things you need to understand deeply. Explained through comparisons to what you already know (React, JS, journalism).
  3. Code examples. Short, focused blocks with inline explanations. Every code block has an "Explain This" button.
  4. Live demos. Interactive SVG right in the lesson — see the output, tweak the values.
  5. Quiz. Quick checks: "Can you explain this back?" Click to reveal answers.
  6. Exercise. Something to build. Small enough to finish in one sitting, meaty enough to prove understanding.

Every page also has an AI Tutor in the corner. It knows which lesson you're on, what concepts you've covered, and can explain things differently if you're stuck.

The Mental Models

D3 is hard to learn because the API is huge and the docs assume you already understand the concepts. These analogies are the shortcut:

  • SVG is a coordinate system — y goes down, not up. Everything is placed by (x, y).
  • Selections are jQuery for SVG — grab elements, chain methods, modify attributes.
  • Data binding is a wedding seating chart — match each data point to a seat (DOM element). New guests get added, departed guests get removed.
  • Scales are translators — they convert data values (0–100) into visual values (0px–500px).
  • Axes are labels for scales — they show the human-readable ticks along the edge.
  • Projections are ways of flattening a globe onto a table — every method distorts something.
  • Transitions are CSS animations for SVG attributes — same idea, different syntax.

Prerequisites

You should be comfortable with:

  • HTML & CSS — you know what a <div> is, how classes work, basic layout.
  • JavaScript fundamentals — variables, functions, arrays, objects, arrow functions, .map(), .filter().
  • The browser dev tools — you can inspect elements and read the console.

You do not need prior experience with D3, SVG, or data visualization. That's what we're here for.
