name: The Scroll design philosophy
description: Core design principles for The Scroll feed — passive ADHD-friendly learning, no interactive mechanics, good vibes only
type: feedback
---

The Scroll should be PASSIVE learning — designed for ADHD brains that trick themselves into learning through doomscrolling. No interactive post types (except polls). The user just reads and absorbs.

**Why:** Trevor has ADHD and wants to channel the doomscroll dopamine loop into learning. Interactive elements break the flow. The learning happens through exposure, personality, and comment threads where disagreements lead to correct answers.

**How to apply:**
- No code challenges, fill-in-the-blank, or graded exercises in the feed
- Polls are OK (passive vote, see results)
- All "spot the bug" posts should have the answer in the comments — user reads, doesn't submit
- Persona disagreements ALWAYS lead to the right answer (or both valid approaches if there's no single answer)
- Snarky is fine, mean is not. Good vibes. Spark joy.
- Personas should look like real people at first glance, then you notice the pun name and smile
- Easter eggs based loosely on real dev culture (xkcd, SO, Reddit) but never directly attacking anyone
- Content base: large static JSON (~100 posts/track) + 3-5 AI-generated fresh posts per visit
