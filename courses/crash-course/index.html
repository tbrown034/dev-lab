The Election Night Pipeline — Django + D3 Working Together

This is the single most impressive workflow to understand for the interview. If they ask "how do Django and D3 work together?" — walk them through this.

1
AP Election Feed
External data source — Associated Press
AP gathers results from the Secretary of State, county offices, and stringers on the ground. Structured JSON with race IDs, candidates, vote counts, precincts reporting.
|
2
Django Management Command
python manage.py import_election_results
Runs on a cron job every 2–5 minutes on election night. Fetches AP data. Uses update_or_create pattern — updates existing records, creates new ones. No duplicates.
|
3
PostgreSQL Database
Django Models: Race, Candidate, CountyResult
Django's ORM maps Python classes to database tables. Query like Python: Race.objects.filter(state='TX', called=False). Migrations keep schema in sync.
|
4
DRF JSON API
Django REST Framework — Serializers + ViewSets
Turns database objects into JSON endpoints. GET /api/v1/races/ returns all races. Handles pagination, filtering, caching. This is the contract between backend and frontend.
|
5
D3 Frontend + Maps
D3.js + GeoJSON + geoAlbers projection
Fetches JSON from the API. Binds data to SVG paths (one per county). Colors by party using scaleOrdinal. Enter-update-exit handles live updates — counties smoothly transition as results arrive.
|
6
Reader Sees Live Results
apps.texastribune.org — hosted on AWS S3
Page polls the API every 30–60 seconds. Map and tables update in place. Readers can enter their address to see specific races. Mobile responsive. Accessible tabular fallbacks.
Say this

"What I find elegant is the separation of concerns. The backend team can fix a data ingestion bug without touching the frontend. The frontend team can redesign the map without touching the API. And the Django admin is the emergency fallback — if the automated pipeline breaks at 11pm, an editor can manually update results."

AI + Elections — Where AI Can Help the Data Visuals Team

The job listing specifically mentions "AI experimentation." Here are concrete examples of how AI/LLMs could help a team like the Tribune's — especially around elections. This is your competitive edge.

Automated Anomaly Detection in Election Results

An LLM or ML model could flag unusual patterns as results stream in on election night: "County X is showing a 30% swing from the primary — verify this isn't a data error." Catches AP data glitches or genuinely newsworthy shifts faster than a human scanning spreadsheets.

Your pitch: "I've built AI tools that process and compare large datasets. Applying that to election night data — flagging anomalies, catching errors before they go live — feels like a natural application."
Natural Language Data Summaries

Instead of just showing numbers, an LLM could generate plain-English summaries: "Republicans have flipped 3 border counties that haven't voted red since 1972." Auto-generated context blurbs for each race, updated as results change. Helps reporters write faster and gives readers instant narrative.

Your pitch: "As a journalist, I know what a good lede sounds like. I could help design AI prompts that generate useful narrative summaries, not generic ones — because I understand what makes a data point a story."
Accessibility — Alt Text and Descriptions for Charts

AI can auto-generate meaningful alt text for D3 visualizations: not just "a bar chart" but "Bar chart showing Travis County voter turnout at 62%, the highest in the state, compared to the statewide average of 41%." Makes every graphic accessible to screen reader users.

Your pitch: "Accessibility is one of the most practical AI applications for data journalism. Instead of generic alt text, we could generate descriptions that actually convey the insight the chart is showing."
Data Cleaning and Standardization

Election data comes from 254 Texas counties with inconsistent formatting. LLMs can help standardize names, fix encoding issues, match records across datasets (e.g., matching AP candidate IDs to FEC campaign finance records). Saves hours of manual cleaning.

Your pitch: "My Second Brain project processes 200k+ records. I understand the data cleaning problem at scale. AI assistants for entity resolution and format standardization could save the team significant time."
Story Discovery in Large Datasets

Feed a campaign finance dataset into an LLM: "What are the most unusual donation patterns?" "Which candidates received the largest out-of-state contributions?" AI can surface leads that would take a reporter days to find manually.

Your pitch: "I've spent 15 years finding stories in data manually. AI doesn't replace that instinct — it accelerates it. The journalist still decides what's a story. AI helps you look in more places, faster."
Conversational Data Explorer — They're Already Doing This

The Tribune built an AI chatbot on their 2025 water crisis coverage that answers reader questions based on Tribune reporting. This is not hypothetical — they're already experimenting. Imagine extending this to election data, school ratings, or campaign finance: "How did my county vote?" "Compare school ratings in Austin vs. Dallas." Readers ask questions in natural language, the system queries the API and returns insights.

Your pitch: "I saw the AI chatbot on the water crisis coverage — that's exactly the kind of experiment I'm excited about. I built an AI news aggregator and a model comparison tool. Extending conversational AI across Tribune datasets is a natural next step."
Say this in the interview

"The listing mentions AI experimentation, and I think the biggest opportunity is using AI to make the team's existing data more accessible and discoverable — auto-generating alt text for charts, flagging anomalies in election data, helping readers ask natural language questions about datasets the Tribune already maintains. AI amplifies the journalism, it doesn't replace it."
