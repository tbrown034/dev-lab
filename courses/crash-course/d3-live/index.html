D3 Mental Model

See D3 In Action

Live, interactive examples of the core D3 concepts. These aren't tutorials — they're visual proof of how D3 thinks, so you can talk about it with confidence.

1. Data Binding — The Core Concept

D3 binds data to shapes. Each data point controls one shape. Change the data, and the shapes update.

Mental model: Think of a spreadsheet. Each row of data gets one bar. The value in the cell controls the bar's width. D3 automates that connection.
// The D3 pattern: select → bind data → join shapes
svg.selectAll("rect")
  .data(countyData, d => d.name)  // bind data by county name
  .join("rect")                 // enter + update + exit
  .attr("width", d => xScale(d.votes)) // width driven by data
  .attr("fill", d => colorScale(d.party)) // color driven by data

Click the buttons to see enter-update-exit in action. New counties animate in (enter), existing ones update smoothly (update), and removed ones fade out (exit). This is how election results update live on election night.

2. Scales — The Bridge Between Data and Pixels

A scale is a function that converts data values to visual values. Domain = your data. Range = pixels (or colors).

Mental model: A scale is like a translator. "Population 4,800,000" means nothing to SVG. The scale converts it to "bar width: 450 pixels" or "color: dark blue." Different data types need different scale types.
// scaleLinear: continuous numbers → continuous pixels
const x = d3.scaleLinear()
  .domain([0, 4800000])  // data range (population)
  .range([0, 700])     // pixel range (chart width)

x(2400000) // → 350 (halfway)

The three scales you'll talk about most: scaleLinear (number to pixels), scaleBand (categories to even spacing), scaleOrdinal (categories to colors, like party → red/blue).

3. Maps — How the Tribune Draws Texas

D3 draws maps from GeoJSON boundary data + a projection + a path generator. Each county is one SVG path element, colored by data.

Mental model: A GeoJSON file is like a coloring book outline — it defines the shape of each county as coordinates. The projection converts lat/lng to screen pixels. geoPath converts those coordinates to SVG drawing instructions. You color each county based on election results. That's a choropleth.
// The Tribune's Texas map in ~10 lines
const projection = d3.geoAlbers().fitSize([width, height], texasGeo)
const path = d3.geoPath(projection)

svg.selectAll("path")
  .data(texasGeo.features)         // one feature per county
  .join("path")
  .attr("d", path)                // draw the county shape
  .attr("fill", d => colorByParty(d)) // color by election data

This simplified Texas map uses randomly generated "election results" to demonstrate the choropleth pattern. In production, the Tribune merges real AP election data with county GeoJSON using FIPS codes as the key. Hover/tap counties to see the concept.

4. Putting It Together — A Complete D3 Chart

Every D3 chart follows the same pattern: set up margins, create scales, binds data, add axes. Here's a scatter plot of (fake) Texas county election data.

// The universal D3 pattern:
// 1. Margin convention (space for axes)
const margin = { top: 20, right: 20, bottom: 40, left: 50 }
// 2. Create scales (data → pixels)
const x = d3.scaleLinear().domain([...]).range([...])
// 3. Bind data to shapes
svg.selectAll("circle").data(counties).join("circle")
// 4. Add axes
svg.append("g").call(d3.axisBottom(x))

Every chart — bar, line, scatter, map — follows this same skeleton. The only things that change are the shape elements (rect vs circle vs path) and the specific scales and generators. Once you see the pattern, you see it everywhere.
