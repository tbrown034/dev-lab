# Development Log

A chronological record of development sessions and significant changes.

## 2026-05-31 — Production fix, rebrand to Dev Lab, new tracks, Connections feature

**Session Summary:**
- Fixed the AI in production: `/api/chat` only existed as Vite dev-server middleware, so the tutor, skill test, and Scroll replies 404'd in prod. Added a Vercel serverless function (`api/chat.js`) and extracted shared prompts to `lib/system-prompts.js`.
- Stripped the Texas Tribune / newsroom job-prep apparatus (deleted `/jobs/` and all of `/courses/crash-course/`, removed nav links, fixed the post-login redirect, removed orphaned crash-course audio). Reframed all AI tutor personas to neutral, general-purpose.
- Split Python into its own `/python/` track (5 lessons moved out of `/django/`); slimmed `/django/` to 7 lessons, renumbered 01–07.
- Added R and Pandas as tracks (landing pages with live AI tutors; lessons in progress).
- Retaxonomized the homepage: **Main Tracks** (Python, JavaScript, SQL, HTML/CSS, R) + an **All Courses** catalog, with TypeScript / PostgreSQL / Tailwind as specifics pointing to their parent tracks.
- Redesigned the homepage hero into a compact personal launchpad ("What do you want to work on?" + jump chips); removed the marketing hero, pull-quote, and "Learning Tools" grid. Fixed a left-edge alignment bug and the grey empty-cell card-grid bug.
- Fixed The Scroll: its content never loaded in prod (build gap — moved `feed/content/` to `public/`). Generated 100 new connection-focused posts (10/track) via a 10-agent workflow; ~829 posts now render.
- Removed the robotic audio "read the site" TTS mode (kept the Pod).
- AI cost/abuse policy: switched all features to Haiku (`claude-haiku-4-5`); replaced the per-instance in-memory rate limit with a Postgres-backed fixed-window limiter (reuses Neon `DATABASE_URL`); added a same-origin gate on `/api/chat`.
- Added the **Connections** page (`/connections/`) — the north-star feature: 12 core concepts shown as the same task side by side across JavaScript/Python/SQL/R/Pandas with a "Connection" insight each. Content generated by a 12-agent workflow, assembled via `tools/gen-connections.mjs`.
- SEO + security: `robots.txt`, generated `sitemap.xml` (152 routes), homepage OG/Twitter meta + per-page descriptions, and `vercel.json` security headers.
- Added a custom Dev Lab favicon at the site root (`public/favicon.svg`) so it's detectable.

**Key Decisions:**
- One model (Haiku) for everything to keep costs low.
- Declined to auto-generate formal R/Pandas/Tailwind lessons — the north star is *less* lesson-heavy, *more* explaining how things connect (Connections page embodies that).
- Generation work (Scroll posts, Connections content) done with multi-agent workflows, then validated and assembled into static files.

**Notable Changes:**
- Repo moved to `github.com/tbrown034/dev-lab` (now public); prod at `learn-coding-iota.vercel.app`, Vercel project `dev-lab`.
- "No Production Deployment" dashboard label was caused by Vercel Deployment Protection (Standard) blocking the screenshot bot — resolved by turning Vercel Authentication off in project settings (user action). Site was always live.
- 8,241 tests passing. All work shipped to `main` and verified live.
