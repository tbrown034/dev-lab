--- LESSON CONTENT ---

The Problem (Revisited)

Magic numbers like d.pop / 150 are fragile. Change the data or chart size and everything breaks. Scales solve this permanently by creating reusable translator functions configured with domain (data range) and range (pixel range).

Domain and Range: The Two Settings

Domain = input range (data world). "My data goes from 0 to 40 million."
Range = output range (pixel world). "My chart area is 0 to 600 pixels wide."

A scale is just a function. Call it with a data value, get a pixel value back. d3.scaleLinear() creates the function; .domain() and .range() configure it.

Scale Types: Pick the Right Translator

1. scaleLinear -- continuous numbers (population, money, temperature, percentages). For Y-axis, range is FLIPPED: .range([height, 0]) because SVG Y goes top-down.

2. scaleBand -- categories (bar chart X-axis). Divides space evenly. Provides .bandwidth() for bar width. .padding() for gaps between bars.

3. scaleTime -- dates. Maps Date objects to pixels. Used for time series X-axes.

4. scaleSequential -- data to colors. Maps numbers to colors using interpolators. Essential for choropleth maps and heatmaps. d3.interpolateBlues goes light-to-dark blue. d3.interpolateViridis is colorblind-safe.

5. scaleSqrt -- area-proportional sizing. For bubble charts: use sqrt so AREA scales linearly with data. If you linearly map population to radius, a city 4x larger gets a circle 16x larger (by area), which is visually misleading.

6. scaleOrdinal -- categories to discrete values. Map party names to specific colors: Democrat -> blue, Republican -> red.

7. scaleLog -- huge ranges (income distribution). Logarithmic scaling for data spanning many orders of magnitude.

Essential Scale Methods

.nice() -- rounds domain to clean numbers. Domain [3.2, 97.8] becomes [0, 100]. Always call after setting domain.

.invert() -- reverse translator. Given a pixel position, returns the data value. Only works on linear/time scales. Essential for mouse interaction (converting cursor position to data coordinates).

.clamp(true) -- prevents out-of-range outputs. Input beyond the domain still returns values within the range.

Quick Reference Table:
- scaleLinear: numbers (pop, money, %) -> continuous pixels
- scaleBand: categories -> pixel bands (with .bandwidth())
- scaleTime: dates -> continuous pixels
- scaleSequential: numbers -> colors
- scaleSqrt: numbers -> continuous (area-proportional)
- scaleOrdinal: categories -> discrete values (colors)
- scaleLog: huge numeric ranges -> continuous pixels

--- KEY CONCEPTS TO EMPHASIZE ---

1. Choose scale type based on data type: numbers=linear, categories=band, dates=time
2. scaleSqrt for proportional AREA (bubble charts) -- because area = pi * r^2
3. scaleSequential for color mapping in choropleth maps
4. .nice() for clean axis boundaries, .invert() for mouse-to-data conversion
5. Y range is ALWAYS flipped: [height, 0]

--- COMMON PITFALLS ---

- Using scaleLinear for bubble radius instead of scaleSqrt (creates visually misleading size comparisons)
- Forgetting .nice() and getting ugly axis labels like 0, 7907, 15815
- Not flipping the Y range (bars grow downward instead of upward)
- Trying to use .invert() on a scaleBand (it only works on continuous scales)
- Using the wrong color interpolator for accessibility (interpolateViridis is colorblind-safe, many others aren't)

--- BROADCAST TV CONNECTION ---

- Election dashboards use scaleOrdinal for party colors (Democrat blue, Republican red, Independent purple)
- Weather maps use scaleSequential with temperature interpolators (blue cold, red hot)
- Census bubble maps use scaleSqrt so population circles are proportional by area
- Economic timelines use scaleTime for the X-axis
- Income inequality visualizations use scaleLog to handle ranges from $20k to $10M
- .invert() enables the "hover over the chart to see exact values" interaction used in broadcast graphics
