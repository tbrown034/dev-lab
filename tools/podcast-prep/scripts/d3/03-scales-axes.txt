--- LESSON CONTENT ---

The Core Idea

In React, you describe what the UI should look like and React figures out how to update the DOM. D3 is the opposite: you directly tell the DOM what to do, step by step. A selection is a wrapper around one or more DOM elements that gives you a chainable API for manipulating them.

jQuery comparison:
$("circle").css("fill", "orange").attr("r", 20);
D3 equivalent:
d3.selectAll("circle").style("fill", "orange").attr("r", 20);

Every D3 method returns the selection itself. That is how method chaining works. .attr() sets the attribute AND returns the selection so you can call .style() next. Same pattern as jQuery, Lodash chains, or array methods.

d3.select() vs d3.selectAll()

select() picks the FIRST matching element. selectAll() picks ALL matching elements. Both use standard CSS selectors (like querySelector and querySelectorAll). Scoped selections work by chaining: d3.select("#chart").selectAll("circle") scopes to circles inside #chart.

The Essential Methods

.attr(name, value) -- the workhorse. SVG elements are controlled by attributes: x, y, width, height, fill, r, d, etc.

.style(name, value) -- sets CSS inline styles. CSS styles override SVG attributes when both are set. Use .attr() for SVG-specific properties (x, width, r, d). Use .style() for CSS-only properties (cursor, opacity).

.text(value) -- sets text content of elements.

.html(value) -- sets inner HTML (works on HTML divs, not SVG elements).

.property(name, value) -- for form element properties like value, checked.

.classed(name, boolean) -- toggles CSS classes. Can take a function: .classed("highlighted", d => d.value > 50).

.append(tag) -- creates a new child element and returns a selection of the NEW child (important: subsequent chained methods operate on the child, not the parent).

.remove() -- removes elements from the DOM.

.each(function(d, i) {...}) -- runs a function for each element in the selection. Inside the function, "this" is the raw DOM element.

Inspection methods (these BREAK the chain because they return values, not selections):
- .empty() -- true if no elements matched
- .size() -- count of elements
- .node() -- first raw DOM element
- .nodes() -- array of all raw DOM elements

Key distinction: .attr("fill") vs .style("fill") -- both work for properties like fill and stroke. .attr() sets the SVG attribute directly (appears in the markup). .style() sets an inline CSS style. CSS styles override SVG attributes. For SVG-specific properties like x, width, r, d, you MUST use .attr(). When in doubt, use .attr() for SVG.

--- KEY CONCEPTS TO EMPHASIZE ---

1. A selection is a wrapper/handle for DOM elements with a chainable API
2. d3.select() = first match, d3.selectAll() = all matches (standard CSS selectors)
3. .attr() for SVG attributes, .style() for CSS properties -- .attr() is the workhorse
4. .append() returns the NEW child element, not the parent
5. Method chaining works because every method returns the selection (except inspection methods)

--- COMMON PITFALLS ---

- Confusing .attr("fill") with .style("fill") -- CSS style overrides SVG attribute when both set
- Forgetting that .append() changes what you're chaining on (it returns the child, not parent)
- Using .html() inside SVG (it doesn't work -- SVG text needs <text> elements)
- Using arrow functions with .each() when you need "this" to refer to the DOM element (arrow functions don't have their own "this")
- Calling .empty() or .size() mid-chain and wondering why the chain broke

--- BROADCAST TV CONNECTION ---

- Building broadcast graphics requires precise control over every SVG element -- selections give you that control
- Election night updates: select all state shapes, change their fill colors as results come in
- Weather overlays: select temperature labels and update their text content as forecasts change
- Breaking-news lower thirds: append new text elements for headlines, remove old ones
- The chainable API lets you write compact, readable code for rapid iteration during live broadcasts
