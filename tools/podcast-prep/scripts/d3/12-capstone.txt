--- LESSON CONTENT ---

Dashboard Architecture

The capstone dashboard uses a simple but effective architecture:

1. Shared state: Two global variables control the dashboard:
   - currentRegion = "all" (filter: all regions, or Northeast/South/Midwest/West)
   - sortByValue = true (bar chart sort order: by population or alphabetically)

2. Data filtering function: getFilteredData() reads the current state and returns the appropriate subset of data, sorted as requested.

3. Individual chart update functions: updateBars(data) and updateDonut(data) each accept filtered data and re-render their chart using the enter/update/exit pattern.

4. Master update function: updateAll() calls getFilteredData() and passes the result to both chart update functions. This is called whenever a filter changes.

5. Event binding: The region dropdown, sort buttons, and map interactions all update the shared state and call updateAll().

This is the "general update pattern" from Lesson 07 scaled up to a multi-chart dashboard. One state change triggers coordinated re-renders across all panels.

Components Used

US State Choropleth Map (Lesson 10):
- Loads TopoJSON from CDN, converts with topojson.feature()
- geoAlbersUsa projection with .fitSize()
- States colored by region using an ordinal color scheme
- Hover tooltips with state name and region
- Border highlighting on hover using topojson.mesh()
- Region color legend in the map corner

Filterable/Sortable Bar Chart (Lessons 06, 07, 11):
- Margins convention for proper axis spacing
- scaleBand for X (state abbreviations), scaleLinear for Y (population)
- Enter/update/exit with key function (d => d.state)
- Enter: bars grow from bottom with staggered delays
- Update: bars slide to new positions and heights with transitions
- Exit: bars shrink and fade out
- Hover tooltips showing full state name, population, and region
- Animated axes that smoothly update when data changes

Regional Donut Chart (Lesson 09):
- d3.pie() with .sort(null) to preserve region order
- d3.arc() with innerRadius > 0 for donut style
- attrTween for smooth arc transitions when data changes
- Centroid-based labels (two-letter region abbreviation)
- Center text showing total population
- .padAngle() and .cornerRadius() for polish

Shared Interactions (Lesson 11):
- HTML tooltip div with position: fixed, shared across all charts
- Hover highlights (brighter fill, border stroke) on bars and map states
- viewBox on all SVGs for responsive sizing
- role="img" and aria-label for accessibility

Filter Controls:
- Region dropdown filters data to one region or all regions
- "By Value" and "A-Z" sort buttons reorder the bar chart
- All controls call updateAll() to synchronize all charts

--- COMMON PITFALLS ---

- Not using key functions in the bar chart's .data() call -- without keys, filtering and sorting animations look chaotic
- Forgetting to update axes when data changes -- axes need their own .transition().call(axis) chain
- Tooltip positioning issues across multiple panels -- use event.clientX/clientY (viewport coordinates) with position: fixed
- Donut chart transitions breaking when the number of slices changes -- need proper enter/update/exit with attrTween
- Not removing center text before re-rendering -- old text elements accumulate without cleanup
- Map data loading failures (async/network issues) -- always handle errors gracefully with try/catch
