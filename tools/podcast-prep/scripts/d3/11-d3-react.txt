--- LESSON CONTENT ---

Mouse Events in D3 v7

D3's event pattern is callbacks, same concept as React. Instead of <div onClick={handleClick}>, it is .on("click", handleClick).

In D3 v7, the event is the FIRST argument to the handler function:

selection.on("mouseenter", function(event, d) {
  console.log(event);   // native DOM event (pageX, pageY, etc.)
  console.log(d);        // bound data (the __data__ property)
  console.log(this);     // the DOM element -- use d3.select(this)
});

Common events: mouseenter (hover start), mousemove (cursor tracking), mouseleave (hover end), click.

IMPORTANT: In older D3 versions (v5 and earlier), events were accessed via the global d3.event object. In D3 v7, the event is passed as the first argument. If you see d3.event in old tutorials and Stack Overflow answers, it will NOT work in v7. This is one of the most common interview questions about D3 version differences.

Building Tooltips

Tooltips are HTML <div> elements positioned near the cursor. Use HTML divs (not SVG <text>) because divs support rich formatting, automatic word wrap, and are easier to style with CSS.

Pattern:
1. Create a div outside the SVG with position: fixed, display: none, and pointer-events: none
2. On mouseenter: show the div, set its content with .html()
3. On mousemove: update position with event.clientX and event.clientY
4. On mouseleave: hide the div

The highlight effect on hover: change the bar's fill to a brighter color and add a stroke border using d3.select(this). Use a short transition for smoothness.

Transitions

Insert .transition() between a selection and attribute changes to animate the change:

selection
  .transition()
  .duration(800)              // animation length in milliseconds
  .delay((d, i) => i * 50)   // stagger each element by index
  .ease(d3.easeCubicInOut)    // easing curve
  .attr("height", d => yScale(d.value));

Chaining transitions: call .transition() twice. The second transition automatically starts after the first completes:

selection
  .transition().duration(500).attr("height", h)
  .transition().duration(300).attr("fill", "green");

Easing Functions

Easing controls the acceleration profile of animations:

d3.easeCubicInOut -- smooth acceleration and deceleration. Natural feel. Best general-purpose easing.
d3.easeLinear -- constant speed. Good for progress bars and continuous motion.
d3.easeBounce -- bouncy ending. Use very sparingly.
d3.easeElastic -- springy overshoot. Only for playful UIs.
d3.easeCubicOut -- fast start, slow end. Good for entrance animations (elements arriving).

Responsive Charts with viewBox

The simplest way to make D3 charts responsive:

const svg = d3.select("#chart")
  .append("svg")
  .attr("viewBox", "0 0 700 400")  // virtual 700x400 canvas
  .style("width", "100%")
  .style("max-width", "800px");

viewBox defines a virtual coordinate system. You draw everything as if the canvas is 700x400 pixels. The browser scales the entire SVG to fit its container while maintaining the aspect ratio. All coordinates stay the same regardless of actual rendered size.

viewBox handles 90% of responsive needs. Only use ResizeObserver when you need layout changes at different sizes (e.g., switching from horizontal to vertical bar orientation on mobile).

Accessibility Basics

Every D3 chart should include:
- role="img" and aria-label="Description" on the SVG element
- <title> and <desc> elements inside the SVG for screen readers
- tabindex="0" on interactive elements to make them keyboard-focusable
- Keyboard event handlers (keydown) alongside mouse events
- Never rely on color alone -- add patterns, labels, or tooltips for colorblind users

--- KEY CONCEPTS TO EMPHASIZE ---

1. D3 v7 events: (event, d) -- event is the first argument, not d3.event (old pattern)
2. Tooltips are HTML divs positioned with event.clientX/clientY, not SVG text
3. .transition().duration().delay().ease() is the transition chain
4. viewBox makes SVG responsive without ResizeObserver
5. Accessibility: role="img", aria-label, never rely on color alone

--- COMMON PITFALLS ---

- Using d3.event (D3 v5 pattern) instead of the event argument (D3 v7) -- this is a breaking change that comes up in interviews
- Placing tooltips inside the SVG -- they should be HTML divs outside the SVG for proper formatting and positioning
- Forgetting pointer-events: none on tooltip divs (causes flickering when cursor moves over the tooltip)
- Using arrow functions with .each() or .on() when you need "this" to refer to the DOM element -- arrow functions don't bind "this"
- Applying transitions to selections that are being removed -- transitions on exiting elements need .transition() before .remove()
- Not using viewBox and instead setting fixed width/height (chart won't resize with the container)

--- BROADCAST TV CONNECTION ---

- Broadcast graphics always have hover/click interactions on their web companions -- this lesson teaches exactly that
- The staggered bar entrance animation is a standard on-air effect -- bars grow up one by one during a reveal
- Smooth transitions between datasets (FY 2022 vs FY 2023) is exactly what a broadcast data team does when switching between data views during a live show
- Tooltips showing detailed values are essential for broadcast web apps -- the viewer watches the simple graphic on TV and explores the details online
- viewBox responsiveness ensures the same chart works on TV (1920x1080), desktop browsers, and mobile devices
- Easing functions control the "feel" of broadcast animations -- too bouncy looks unprofessional; easeCubicOut is the safe default
- Accessibility compliance is a legal requirement for broadcast network websites
