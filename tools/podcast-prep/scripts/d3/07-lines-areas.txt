--- LESSON CONTENT ---

Why This Matters

In Lesson 04, .join("rect") handled everything automatically. But what happens under the hood when data changes? Understanding enter/update/exit gives you control over animations, transitions, and complex data updates -- the difference between a basic chart and a polished one.

The Three Groups

When you call .data(newData), D3 compares the new data to existing elements and sorts everything into three groups:

Enter: Data items that have no matching element. Need to create new elements. (Green flash -- new items appearing)

Update: Data items that match an existing element. May need to change attributes. (Orange -- existing items repositioning)

Exit: Elements that have no matching data item. Need to be removed. (Red flash -- items disappearing)

.join() with Three Callbacks

svg.selectAll("rect")
  .data(newData, d => d.id)     // key function!
  .join(
    enter => enter.append("rect")
      .attr("fill", "#4ade80")   // green for new
      .attr("y", height).attr("height", 0)  // start from bottom
      .call(el => el.transition().duration(500)
        .attr("y", d => y(d.value))
        .attr("height", d => height - y(d.value))),

    update => update
      .attr("fill", "#f97316")   // orange for existing
      .call(el => el.transition().duration(500)
        .attr("y", d => y(d.value))
        .attr("height", d => height - y(d.value))),

    exit => exit
      .attr("fill", "#ef4444")   // red for leaving
      .call(el => el.transition().duration(300)
        .attr("y", height).attr("height", 0).remove())
  );

Key Functions: Name Tags for Data

By default, D3 matches data to elements by index (position). First data item goes with first element. This works for static charts but breaks for animations.

Without key: .data(newData) matches by index. If you remove the first item, D3 thinks EVERY element changed (indices shifted).

With key: .data(newData, d => d.id) matches by identity. D3 knows each element is the same item, just in a new position. This enables smooth, meaningful animations.

Keys are like React's key prop. Without keys, both D3 and React default to matching by position. With keys, both can track individual items through data changes.

.join("rect") vs .join(enter, update, exit)

The string form .join("rect") is shorthand: enter appends the element, update does nothing special, exit removes immediately. The three-callback form gives full control: animate entering elements, transition updates, fade out exiting elements.

The General Update Pattern

Write one function that handles any data change: filtering, sorting, adding, removing. Call it whenever data changes. The enter/update/exit pattern handles DOM updates automatically. Buttons, sliders, and filters all just call the update function with new data.

The lesson demonstrates a filterable, sortable bar chart where you can filter by region (South, West, Northeast) and sort by value or alphabetically. Same update function handles everything.

--- KEY CONCEPTS TO EMPHASIZE ---

1. Enter = new data needs new elements, Update = existing data may change, Exit = old elements need removal
2. Key functions (d => d.id) match by identity, not position -- like React's key prop
3. .join(enter, update, exit) gives full animation control over each transition group
4. The general update pattern: one reusable function handles all data changes
5. Always remove exit elements to prevent memory leaks

--- COMMON PITFALLS ---

- Not using key functions and getting jumbled animations when items are reordered
- Forgetting to call .remove() on exit elements (they stay in the DOM as invisible orphans)
- Using .join("rect") when you need animation control (it skips to the three-callback form)
- Not understanding that .join() returns the MERGED enter + update selection
- Mixing up which callback is which (enter = new, update = existing, exit = leaving)

--- BROADCAST TV CONNECTION ---

- Election night: new precincts report in (enter), existing precincts update their counts (update), invalidated results disappear (exit) -- all with smooth animations
- Weather dashboards: forecast cities appear and disappear as the user pans the map
- Breaking-news bar charts that smoothly animate as new data points arrive
- The filter-and-sort demo is exactly what a broadcast data team builds: filter states by region, sort by population, and the bars animate into their new positions
- Key functions ensure that when you reorder bars (sort by value vs. alphabetically), each bar slides to its new position rather than morphing into a different state
