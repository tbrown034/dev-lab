--- LESSON CONTENT ---

The Pipeline

Drawing a map in D3 follows a three-step pipeline:

1. Geographic data (GeoJSON or TopoJSON) -- shapes represented as arrays of coordinates (latitude/longitude pairs)

2. Projection -- a function that flattens the 3D globe onto a 2D surface. Like peeling an orange and laying the skin flat. Different projections make different tradeoffs about shape, area, and distance distortion.

3. Path generator -- d3.geoPath(projection) converts the projected coordinates into SVG path strings, just like d3.line() converts data points into line path strings.

import * as topojson from "topojson-client";

// 1. Load geographic data
const us = await d3.json("https://cdn.jsdelivr.net/npm/us-atlas@3/states-10m.json");
const states = topojson.feature(us, us.objects.states);

// 2. Create projection
const projection = d3.geoAlbersUsa()
  .fitSize([width, height], states);

// 3. Create path generator
const path = d3.geoPath(projection);

// 4. Draw
svg.selectAll("path")
  .data(states.features)
  .join("path")
  .attr("d", path)           // path generator creates "d" attribute
  .attr("fill", d => color(getData(d)))
  .attr("stroke", "#333");

GeoJSON vs TopoJSON

GeoJSON is the standard format. Each geographic feature has its own complete set of coordinates. Simple to understand but large files -- a US states file can be 2MB or more.

TopoJSON is a compressed format. It shares border coordinates between adjacent features -- California and Nevada share a border, so why store it twice? TopoJSON files are 80-90% smaller (around 200KB for US states). You convert to GeoJSON in the browser using topojson.feature().

topojson.mesh() generates a single path for all internal borders. The filter function (a, b) => a !== b excludes exterior borders, giving you clean, non-overlapping border lines without the visual artifacts of per-state strokes.

Projections

d3.geoAlbersUsa() -- the projection for US maps. Includes insets for Alaska and Hawaii. Uses equal-area projection so state sizes are proportional. This is the standard for election maps, census maps, and any US geographic visualization.

d3.geoMercator() -- the Google Maps projection. Familiar but severely distorts area near the poles. Alaska appears much larger than it actually is relative to the lower 48 states.

d3.geoEqualEarth() -- a modern world map projection. Preserves area accurately. Good for global visualizations.

.fitSize() is Essential

Instead of manually calculating the scale and translate values to position your map, call projection.fitSize([width, height], geojsonData). It automatically scales and centers the map to fit your SVG dimensions. This is like setting domain and range on a regular scale -- but for geographic coordinates.

The Choropleth Pattern

A choropleth map colors geographic regions according to data values. The standard pattern:

1. Create a data lookup map: const dataMap = new Map(data.map(d => [d.fips, d.value]))
2. Create a sequential color scale: d3.scaleSequential(d3.interpolateYlOrRd).domain([min, max])
3. Color each path: .attr("fill", d => { const val = dataMap.get(d.id); return val ? color(val) : "#333"; })

The lookup map uses FIPS codes (federal geographic identifiers) to match geographic features to data values.

Making Maps Responsive

Use viewBox on the SVG element and .fitSize() on the projection. The viewBox creates a virtual coordinate system that scales with the container. .fitSize() ensures the map fills that space. No fixed pixel dimensions needed.

Accessibility

Color-only encoding is problematic for colorblind users. In production, always add tooltips with actual values, ARIA labels for screen readers, and consider colorblind-safe color schemes like d3.interpolateViridis.

The lesson demonstrates a US state choropleth map showing median household income by state, with hover tooltips, highlighted borders on hover, a gradient color legend, and state name display.

--- KEY CONCEPTS TO EMPHASIZE ---

1. A projection is just another scale -- it translates lat/lng to pixels, just like scaleLinear translates data values to pixels
2. d3.geoPath() is just another path generator -- like d3.line() but for geographic shapes
3. TopoJSON is compressed GeoJSON -- convert with topojson.feature(), 80-90% smaller files
4. geoAlbersUsa() is THE projection for US maps -- includes Alaska/Hawaii insets
5. .fitSize() auto-scales the projection to fill your SVG -- no manual calculation needed
6. The choropleth pattern: data lookup map + sequential color scale + fill each path

--- COMMON PITFALLS ---

- Using geoMercator for US maps (Alaska appears massive; no Hawaii inset)
- Not converting TopoJSON to GeoJSON before passing to d3.geoPath() (the path generator expects GeoJSON features)
- Manually setting projection scale and translate instead of using .fitSize()
- Forgetting that geographic features have ID properties (usually FIPS codes) for matching data
- Not handling missing data gracefully (some features may not have matching data -- provide a fallback color)
- Using per-state strokes instead of topojson.mesh() for borders (creates visual artifacts at shared borders)
