--- LESSON CONTENT ---

Path Generators: Cookie Cutters for Data

A path generator is a function factory. d3.line() creates a function. You configure that function with .x() and .y() accessors that tell it how to extract coordinates from each data point. When you call the resulting function with an array of data, it returns an SVG path string -- the "d" attribute for a <path> element.

const line = d3.line()
  .x(d => xScale(d.date))
  .y(d => yScale(d.value));

line(data)  // returns "M0,150 L50,120 L100,80..."

You never write path commands by hand. The generator handles all the M (move to), L (line to), and other SVG path commands automatically.

.datum() vs .data(): The Critical Distinction

This is one of the most important distinctions in D3:

.data(array) = one element per array item. Used for bar charts (one rect per data point), scatter plots (one circle per data point). Each element gets one array item bound to it.

.datum(array) = one element for the ENTIRE array. Used for line charts and area charts. A line chart is ONE <path> element that uses ALL the data points to draw the line. You bind the whole array to that single path.

svg.append("path")
  .datum(data)          // entire array bound to this ONE path
  .attr("d", line)      // line generator receives the array
  .attr("fill", "none")
  .attr("stroke", "#f97316");

When you write .attr("d", line), D3 calls line(data) automatically -- passing the bound datum (the entire array) to the line generator, which returns the path string.

d3.area(): Filled Regions

An area generator works like a line generator but defines TWO boundaries:

.y0() = the bottom edge (baseline), usually the chart height
.y1() = the top edge, driven by data values

The filled region between y0 and y1 creates the area chart. For a confidence band (showing uncertainty), both y0 and y1 can be data-driven -- y0 maps to the lower bound, y1 maps to the upper bound.

const area = d3.area()
  .x(d => xScale(d.date))
  .y0(height)                // bottom baseline
  .y1(d => yScale(d.value))  // top edge from data
  .curve(d3.curveMonotoneX);

Curve Types: How Points Connect

The .curve() method controls how points are connected. This matters for journalism:

curveLinear -- straight line segments between points. Default. Most accurate representation of the data.

curveMonotoneX -- smooth curve that preserves monotonicity. Passes through every data point. The safe "smooth" option for journalism because it does not create false peaks or valleys between data points.

curveStep -- staircase pattern. Good for showing discrete values that change at specific points (like interest rate changes).

curveBasis -- very smooth but does NOT pass through data points. It creates values between data points that never existed in the actual data. For factual reporting, this is misleading. Use only for aesthetic purposes where data accuracy is not critical.

For journalism, use curveLinear or curveMonotoneX. Basis and cardinal curves create phantom data points that never existed. On a broadcast graphic showing approval ratings, creating a false peak between two survey dates is editorially indefensible.

The Animated Line Drawing Effect

The stroke-dasharray trick creates the illusion of a line being drawn on screen:

1. Get the total length of the path: path.node().getTotalLength()
2. Set stroke-dasharray and stroke-dashoffset to the total length (makes the line invisible)
3. Transition stroke-dashoffset to 0 (gradually reveals the line)

This is a pure SVG trick -- no D3-specific API needed. It works by creating a dash pattern as long as the entire line, then sliding the offset from "fully hidden" to "fully visible."

Combining Line and Area

A common pattern layers an area fill underneath a line:

1. Draw the area first (so it appears behind)
2. Draw the line on top
3. Optionally add dots at each data point using .data().join("circle")

The area uses semi-transparent fill (e.g., "#f9731622") so it provides subtle context without overpowering the line.

--- KEY CONCEPTS TO EMPHASIZE ---

1. d3.line() and d3.area() are function factories -- they create functions that return path strings
2. .datum() binds the entire array to ONE element (line/area); .data() binds each item to a SEPARATE element (bars/circles)
3. curveMonotoneX is the safe smooth curve for journalism -- it passes through every data point
4. The stroke-dasharray trick creates animated line-drawing effects
5. d3.area() defines a region between two boundaries (y0 and y1) -- useful for confidence bands

--- COMMON PITFALLS ---

- Confusing .datum() and .data() -- using .data() for a line chart creates multiple broken paths instead of one continuous line
- Using curveBasis for factual data -- it creates phantom values between data points
- Forgetting to set fill to "none" on line paths -- by default, SVG paths fill with black, creating weird shapes
- Not understanding that d3.line() returns a FUNCTION, not a path string -- you call the function with data to get the string
- Forgetting that the path "d" attribute is what d3.line() generates -- .attr("d", line) calls line(datum) automatically
