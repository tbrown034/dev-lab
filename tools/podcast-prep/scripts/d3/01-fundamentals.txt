--- LESSON CONTENT ---

What is SVG?

SVG stands for Scalable Vector Graphics. It is a set of HTML tags for drawing shapes. Just as standard HTML has <div>, <p>, and <img> for content, SVG adds tags like <rect>, <circle>, <line>, <text>, and <path> for visual shapes. An SVG element acts as a container (like a <div>) and you put shapes inside it.

Example: An <svg> with width="300" and height="100" containing a <rect> at position x="10" y="10" with width="200" height="80" and fill="orange" draws an orange rectangle.

The key insight: This is the ENTIRE foundation of D3. Every bar chart, line chart, map, and visualization that D3 produces is just SVG elements like this. D3 doesn't use canvas or WebGL by default -- it writes these exact HTML tags into the page.

The Coordinate System (The One Gotcha)

SVG's coordinate system has one critical difference from math class:
- (0, 0) is the TOP-LEFT corner
- X increases going right (normal)
- Y increases going DOWN (opposite of math class)

This is why, when making a bar chart, taller bars need SMALLER Y values (to reach higher on screen). This feels backwards at first but becomes natural. D3's scales handle this later, but understanding WHY is essential.

Every Shape You Need to Know

RECT -- Rectangles (bars in bar charts): Uses x, y for the top-left corner position, width and height for size, fill for color, and rx for optional rounded corners.

CIRCLE -- Circles (dots in scatter plots): Uses cx, cy for the CENTER position (not top-left!), r for radius, and fill for color. The positioning difference between rect (top-left) and circle (center) trips people up constantly.

LINE -- Lines: Uses x1, y1 for the start point and x2, y2 for the end point. Lines use stroke for color (not fill!) and stroke-width for thickness. A line with no stroke is invisible.

TEXT -- Labels: The x, y position refers to where the text sits on its baseline (like where a letter rests on a ruled line), not the top-left corner like rect. The key feature is text-anchor which controls horizontal alignment relative to the x-position:
- text-anchor="start" -- text flows right from the point
- text-anchor="middle" -- text centers on the point
- text-anchor="end" -- text flows left from the point

The pattern: text-anchor="middle" for labels centered on bars in vertical charts. text-anchor="end" for labels beside bars in horizontal charts. These two cover almost every chart labeling scenario.

G + TRANSFORM -- "Move everything as a group": The <g> element groups shapes together. Adding transform="translate(x, y)" slides everything inside the group x pixels right and y pixels down. All children draw from (0, 0) of the group's new position. This is like position: relative in CSS. In D3, you rarely position each shape individually -- D3 creates a <g> for each data point, translates it into position, then draws shapes inside at (0, 0).

The Margins Convention: Every D3 chart uses a pattern where the outer SVG is the full canvas, and a single <g> group is translated inward to create margins for axes and labels. This is the #1 D3 pattern: create SVG, define margins, create a <g> translated by the margins. All chart code draws inside that group. Axes sit in the margin space around it.

PATH -- Complex Shapes: The <path> element draws anything using a mini drawing language in its d attribute. M = Move to (pick up the pen), L = Line to (draw a line). A line chart in D3 is a single <path> element. D3's d3.line() generator writes the d attribute string from your data array. You never write path commands manually.

Quick Reference:
- rect: x, y, width, height, fill, rx -- bars, backgrounds, heatmap cells
- circle: cx, cy, r, fill -- scatter plot dots, bubble charts
- line: x1, y1, x2, y2, stroke -- grid lines, reference lines
- text: x, y, text-anchor, fill, font-size -- labels, axis ticks, annotations
- path: d, fill, stroke -- lines, areas, arcs, map outlines
- g: transform -- grouping, positioning, margins

Fill vs Stroke: fill is the inside color. stroke is the border/line color. Lines and paths NEED stroke to be visible. opacity controls transparency (0 = invisible, 1 = solid).

Build exercises: The lesson includes hand-coding a bar chart and scatter plot in pure HTML with no JavaScript. The bar chart uses rect and text elements with text-anchor for alignment. The scatter plot uses circle elements where cx represents one variable and cy represents another.

The Inspect Trick: Right-click any D3 visualization on the web and click "Inspect" to see the raw SVG elements. There is no magic layer between D3's code and what appears on screen. This is why SVG fluency matters -- when you debug D3, you're debugging SVG.

--- KEY CONCEPTS TO EMPHASIZE ---

1. SVG is just HTML tags for shapes -- rect, circle, line, text, path, g cover 95% of D3 output
2. The Y-axis is inverted: (0,0) is top-left, Y increases downward -- taller bars need smaller Y values
3. Rect positions from top-left corner; circle positions from center -- a common source of positioning bugs
4. The <g> + translate pattern is the foundation of all D3 layout, especially the margins convention
5. text-anchor controls text alignment relative to the x position -- essential for labeling charts

--- COMMON PITFALLS ---

- Forgetting that circle uses cx/cy (center) while rect uses x/y (top-left corner)
- Setting fill on a line (it needs stroke to be visible)
- Not understanding why taller bars need smaller Y values (the inverted coordinate system)
- Positioning text without text-anchor, causing labels to be misaligned
- Trying to put HTML content inside SVG (you can't -- use <text> for words in SVG)

--- BROADCAST TV CONNECTION ---

- Every election night dashboard with colored state maps, vote tally bars, and percentage labels is made of these exact SVG elements
- Weather map overlays use <path> elements for state and county boundaries
- Breaking-news lower thirds with animated bars and numbers are SVG rects and text elements
- The margins convention creates the structured layout that broadcast graphics require -- precise placement of chart area vs. labels vs. titles
- Census data visualizations layer circles (population dots) and paths (geographic boundaries) on the same SVG canvas
- The coordinate system precision matters on TV: every pixel counts on a 1920x1080 broadcast frame
