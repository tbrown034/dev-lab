LESSON CONTENT

LISTS — Python's Arrays

A Python list is JavaScript's Array. Same concept, same bracket syntax, slightly different method names.

Creating: nums = [1, 2, 3, 4, 5] — identical to JS except no const/let.

Method name translations:
- .push(6) -> .append(6)
- .pop() -> .pop() (same!)
- .unshift(0) -> .insert(0, 0)
- .splice(1, 1) -> del nums[1]
- .concat([7, 8]) -> nums + [7, 8] or nums.extend([7, 8])
- .indexOf(3) -> .index(3)
- .includes(3) -> 3 in nums (operator, not method!)
- .length -> len(nums) (function, not property!)
- .reverse() -> .reverse() (same!)
- .sort() -> .sort() (same! But key function instead of comparator)

Slicing — Python's Supercharged .slice():
Python's slice syntax [start:stop:step] is far more powerful than JS's .slice(). The step parameter is the superpower JS doesn't have.
- arr[2:5] gives [2, 3, 4] — same as JS .slice(2, 5)
- arr[:3] gives first 3 elements
- arr[-2:] gives last 2 elements
- arr[::2] gives every 2nd element — no JS equivalent without .filter()
- arr[::-1] reverses the whole thing — the Python classic

Mental model: [start:stop:step] mirrors range(start, stop, step). Same three-argument pattern, same "stop is exclusive" rule.

Negative indices work natively: arr[-1] gives the last element (always worked in Python, JS only got .at() in ES2022).

TUPLES — Frozen Arrays

A tuple is an immutable list. Like Object.freeze([1, 2, 3]) in JavaScript. You can read it but never add, remove, or change elements.

point = (10, 20) — parentheses, or even just: point = 10, 20

Gotcha: Single-element tuples need a trailing comma. (42) is just the number 42. (42,) is a tuple. The comma makes it a tuple.

When to use tuples vs lists: Use tuples for fixed data that shouldn't change — coordinates, RGB values, database rows, function return values. Lists when you need to add/remove items. Tuples are faster and can be used as dictionary keys.

DICTS — Python's Objects (for data)

Python dict is what you'd use when you reach for a JavaScript object as a key-value store. Almost identical syntax.

JS: const user = { name: "Trevor", age: 38 }; user.name // "Trevor"
Python: user = {"name": "Trevor", "age": 38}; user["name"] # "Trevor"

CRITICAL DIFFERENCE: No dot notation for dicts! In JavaScript, obj.key and obj["key"] are interchangeable. In Python, dict.key does NOT work. You must use dict["key"] or dict.get("key"). Dot notation is reserved for object attributes and methods.

Missing keys: JS returns undefined silently. Python raises KeyError! Use .get("key") for safe access — returns None if missing, or .get("key", "default") for a default value.

Dict tricks:
- ?? for defaults -> .get("key", "default")
- {...a, ...b} -> {**a, **b} or a | b (Python 3.9+)
- Object.entries() -> .items()
- "key" in obj -> "key" in d (same!)

SETS

Sets work almost identically. Unique values, no duplicates.
- new Set(arr) -> set(arr) or {val1, val2}
- .has() -> "in" operator
- .add() -> .add() (same!)
- .delete() -> .discard()
- .size -> len()

Python has built-in set operators: & for intersection, | for union, - for difference. JS needs libraries for these.

Gotcha: {} creates an empty dict, not a set. Use set() for an empty set.

FOR LOOPS

Python has ONE kind of for loop: for item in iterable. No C-style for (let i = 0; ...). Everything is iteration over a collection.

for fruit in fruits: — like JS for...of
for i, fruit in enumerate(fruits): — like .forEach with index
for i in range(10): — like for (let i = 0; i < 10; i++)
for key, val in obj.items(): — like for...of Object.entries()

range(): Generates a sequence. range(5) gives 0-4. range(2, 7) gives 2-6. range(0, 10, 2) gives 0, 2, 4, 6, 8.

enumerate(): Gives (index, value) pairs. Can start at a custom number: enumerate(colors, 1).

zip(): No direct JS equivalent. Pairs up elements from multiple lists. Instead of syncing arrays with index-based access (scores[i]), use zip(names, scores). Creates a dict from two lists: dict(zip(keys, values)).

While loops and break/continue work the same. Python bonus: for/else runs the else block only if no break was hit — niche but elegant for search patterns.

Python has no ++ or --. Use count += 1 and count -= 1.

CONDITIONALS

if/elif/else instead of if/else if/else. Colons, indentation, no parentheses needed around the condition.

Ternary order is different! JS: condition ? true_val : false_val. Python: true_val if condition else false_val. The VALUE comes first.

match/case (Python 3.10+) is like switch/case but with pattern matching superpowers.

COMPREHENSIONS — Python's Superpower

List comprehensions replace .map(), .filter(), and often both in a single readable line.

The pattern: [EXPRESSION for ITEM in ITERABLE if CONDITION]

JS .map(): nums.map(n => n * 2) -> [n * 2 for n in nums]
JS .filter(): nums.filter(n => n % 2 === 0) -> [n for n in nums if n % 2 == 0]
JS .map() + .filter(): nums.filter(n => n % 2 === 0).map(n => n * 2) -> [n * 2 for n in nums if n % 2 == 0]

Read it like English: "Give me [expression] for each [item] in [collection] if [condition]."

Dict comprehensions: {key: val for x in items} — like Object.fromEntries(arr.map(...))
Set comprehensions: {expr for x in items} — same pattern, curly braces
Generator expressions: (expr for x in items) — lazy, memory efficient

UNPACKING — Python's Destructuring

Array destructuring: const [a, b, c] = [1, 2, 3] -> a, b, c = [1, 2, 3]
Rest: const [head, ...tail] = arr -> head, *tail = arr (one star, not three dots)
Skip elements: const [first, , third] = arr -> first, _, third = arr (underscore convention)

No dict destructuring! Python has no equivalent of const { name, age } = obj. You access dict values individually.

*rest in Python = ...rest in JS. Star can go anywhere: first, *middle, last = [1,2,3,4,5] gives middle = [2,3,4].

USEFUL BUILT-IN FUNCTIONS

.reduce() for sum -> sum(nums) (built-in!)
Math.min() -> min(nums)
Math.max() -> max(nums)
.every() -> all(n > 0 for n in nums)
.some() -> any(n > 0 for n in nums)
[...arr].sort() -> sorted(arr) (returns new list)

Python's sort key function: Instead of a comparison function returning -1/0/1, you provide a function that extracts the value to sort by. users.sort(key=lambda u: u["age"]) — Python figures out the ordering.

------------------------------------------------------------
KEY CONCEPTS TO EMPHASIZE
------------------------------------------------------------

1. List comprehensions are THE Python idiom — once you internalize [expr for x in items if condition], you'll reach for it constantly. It replaces map + filter in one shot.
2. "in" is your Swiss Army knife — checks membership in lists, dicts, sets, strings. Way cleaner than .includes() or .has().
3. Dicts are NOT objects — no dot notation! Use bracket access or .get() for safety.
4. enumerate() and zip() are your loop power tools — they replace index-based array access patterns.
5. Tuples exist for a reason — fixed data, dictionary keys, function return values. Get comfortable with them now because Django returns tuples everywhere (e.g., choices fields).

------------------------------------------------------------
COMMON PITFALLS FOR JS DEVELOPERS
------------------------------------------------------------

- Using dot notation on dicts (user.name instead of user["name"]) — this will be a constant reflex for weeks
- Forgetting that {} is an empty dict, not an empty set — use set() for empty sets
- Getting the ternary order wrong — Python puts the value first: "yes" if True else "no"
- Trying to use .push() instead of .append()
- Using .includes() instead of the "in" operator
- Expecting dict["missing_key"] to return undefined — it raises KeyError, use .get()
- Writing (42) thinking it's a single-element tuple — need the trailing comma: (42,)
- Trying .length instead of len()
- Writing Object.entries(d) instead of d.items()
