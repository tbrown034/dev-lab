About This Platform

How I Learned D3.js by Building the Thing That Teaches It

This is a custom-built learning ecosystem created by Trevor Brown — an investigative journalist turned web developer who believes the best way to learn something is to build the platform that teaches it.

The Story

I spent 15+ years as an investigative journalist. Then I taught myself web development — React, Next.js, TypeScript, Tailwind — and discovered that building things scratched the same itch as breaking a story: find the truth in complex systems and present it clearly.

Now I'm building expert-level skills in D3.js, Python/Django, and SQL for a potential role at Tenga, a chain of broadcast TV networks where data visualization meets election dashboards, weather maps, census overlays, and breaking-news graphics.

This platform is how I learn. I don't watch tutorials. I don't follow along with textbooks. I build the thing that teaches me, and in doing so, I learn it more deeply than any course could offer. Every page you see here was designed, written, and coded by me with help from Claude Code.

The Philosophy

Every lesson follows a single rule: mental models first, syntax second. I don't memorize API signatures. I learn the why behind every function so I can derive the how on the fly.

Each concept is taught through analogies to tools I already know. D3 scales are translators between data and pixels. Enter/update/exit is a wedding seating chart. Projections are ways of flattening a globe onto a table. When you truly understand the concept, the syntax writes itself.

"I don't want to memorize syntax — I want to understand WHY things work so I can build anything."

One concept per lesson. Depth over breadth. The goal after two weeks: read any D3 code like English and build any standard chart from memory.
