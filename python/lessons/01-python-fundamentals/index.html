You already know how to program. This lesson translates the absolute basics from JavaScript into Python. Variables, types, strings, and operators — the atoms of every program.

Why Python?

Python and JavaScript are both high-level, dynamically-typed, interpreted languages. If JavaScript is the language of the browser, Python is the language of everything else: data science, machine learning, automation, backend web development, scripting.

Django (the web framework we're learning) is built on Python, just like Next.js is built on JavaScript/TypeScript. So step one is getting fluent in Python's syntax.

The mental model: Python is what JavaScript would look like if someone redesigned it to be as readable as possible. Less punctuation, more whitespace, one obvious way to do things. The "Zen of Python" literally says: "There should be one — and preferably only one — obvious way to do it."

Running Python

Just like you run node script.js or open the browser console, you run Python with:

JavaScript
// Run a file
node script.js

// Interactive REPL
node
Python
# Run a file
python script.py

# Interactive REPL
python

Pro tip: On macOS, you might need to use python3 instead of python. To check your version: python3 --version. You want Python 3.10 or newer.

Variables & Types

In JavaScript you declare variables with let, const, or var. In Python, you just assign. There's no declaration keyword.

JavaScript
// Variable declaration
let name = "Trevor";
const age = 38;
let isJournalist = true;
let salary = 85000.50;
let nothing = null;
let undef = undefined;
Python
# Just assign — no let/const/var
name = "Trevor"
age = 38
is_journalist = True
salary = 85000.50
nothing = None
# No undefined in Python

Key differences: Python uses True/False (capitalized), None instead of null, and has no undefined. Variable names use snake_case by convention, not camelCase.

No const in Python

Python has no const keyword. By convention, ALL_CAPS names signal "don't change this," but nothing enforces it.

JavaScript
// Enforced constant
const API_URL = "https://api.example.com";
API_URL = "nope"; // TypeError!

// Mutable variable
let count = 0;
count = 1; // fine
Python
# Convention only — not enforced
API_URL = "https://api.example.com"
API_URL = "still works" # No error!

# Everything is just assignment
count = 0
count = 1 # fine

Coming from TypeScript, this feels lawless. But Python's philosophy is "we're all consenting adults here." If a variable is named MAX_RETRIES, everyone agrees not to change it. No language feature needed.

Python's Type System

JavaScript has 7 primitive types. Python has a simpler set, but with a key difference: Python separates integers from floats (JavaScript lumps them all into number).

Concept JavaScript Python
Text "string" "str"
Whole numbers number int
Decimal numbers number float
Boolean true / false True / False
Nothing null / undefined None

Type Checking

JavaScript / TypeScript
typeof "hello"     // "string"
typeof 42          // "number"
typeof true        // "boolean"
typeof null        // "object" (bug!)

// TypeScript annotations
let name: string = "Trevor";
let age: number = 38;
Python
type("hello")      # <class 'str'>
type(42)           # <class 'int'>
type(True)         # <class 'bool'>
type(None)         # <class 'NoneType'>

# Type hints (like TypeScript, but optional)
name: str = "Trevor"
age: int = 38

Python has two number types: int (whole numbers) and float (decimals). JavaScript only has number. Python integers have unlimited precision — no BigInt needed. 2 ** 1000 just works.

Type Conversion

JavaScript
// Implicit coercion (dangerous!)
"5" + 3       // "53" (string!)
"5" - 3       // 2 (number!)

// Explicit conversion
Number("42")  // 42
String(42)    // "42"
parseInt("42px") // 42
Boolean(0)    // false
Python
# No implicit coercion (safe!)
"5" + 3       # TypeError!
# Python refuses to guess

# Explicit conversion
int("42")     # 42
str(42)       # "42"
int("42px")   # ValueError! (no guessing)
bool(0)       # False

Python refuses to silently coerce types. Where JavaScript guesses (and often guesses wrong — "5" + 3 = "53"), Python throws an error. This means fewer bugs. You explicitly convert with int(), str(), float(), bool().

Strings Deep Dive

Strings work almost identically in both languages, with some small syntax differences you need to internalize.

Creating Strings

JavaScript
// Single or double quotes
const a = 'hello';
const b = "hello";

// Template literal (backticks)
const greeting = `Hello, ${name}!`;

// Multi-line (backticks only)
const multi = `
  Line 1
  Line 2
`;
Python
# Single or double quotes (identical)
a = 'hello'
b = "hello"

# f-string (like template literal)
greeting = f"Hello, {name}!"

# Multi-line (triple quotes)
multi = """
  Line 1
  Line 2
"""

Translation rule: Backtick + dollar sign → f"" + curly braces. Same mental model, slightly different punctuation. The f prefix is like saying "this string has expressions in it."

String Methods

JavaScript
"Hello World".toLowerCase()   // "hello world"
"Hello World".toUpperCase()   // "HELLO WORLD"
"  hi  ".trim()               // "hi"
"hello".includes("ell")       // true
"hello".startsWith("hel")     // true
"hello".replace("l", "r")     // "herlo"
"a,b,c".split(",")            // ["a","b","c"]
["a","b","c"].join("-")        // "a-b-c"
"hello".length                 // 5
"hello"[0]                     // "h"
"hello".slice(1, 3)            // "el"
Python
"Hello World".lower()          # "hello world"
"Hello World".upper()          # "HELLO WORLD"
"  hi  ".strip()               # "hi"
"ell" in "hello"               # True
"hello".startswith("hel")      # True
"hello".replace("l", "r")     # "herro" (all!)
"a,b,c".split(",")            # ["a","b","c"]
"-".join(["a","b","c"])        # "a-b-c"
len("hello")                   # 5
"hello"[0]                     # "h"
"hello"[1:3]                   # "el"

Watch out: Python's replace() replaces ALL occurrences by default (JS only replaces the first). Python's .join() is called on the separator, not the array — "-".join(list) feels backwards at first. And len() is a function, not a property.

String Slicing

Python's slicing is more powerful than JavaScript's .slice(). The syntax is [start:stop:step].

text = "Hello, World!"

text[0:5]     # "Hello"     — from 0 to 5 (exclusive)
text[7:]      # "World!"    — from 7 to end
text[:5]      # "Hello"     — from start to 5
text[-6:]     # "World!"    — last 6 characters
text[::2]     # "Hlo ol!"   — every 2nd character
text[::-1]    # "!dlroW ,olleH" — reversed!

The [::-1] trick reverses any sequence. No need for .reverse() or .split('').reverse().join(''). This is one of Python's most elegant idioms and it works on strings, lists, and any other sequence.

Operators

Arithmetic

JavaScript
5 + 3      // 8
5 - 3      // 2
5 * 3      // 15
5 / 3      // 1.6666...
5 % 3      // 2 (remainder)
5 ** 3     // 125 (exponent)
// No integer division operator
Math.floor(5 / 3)  // 1
Python
5 + 3      # 8
5 - 3      # 2
5 * 3      # 15
5 / 3      # 1.6666... (always float!)
5 % 3      # 2 (remainder)
5 ** 3     # 125 (exponent)
5 // 3     # 1 (integer/floor division)

Python has floor division: // divides and rounds down to the nearest integer. JavaScript needs Math.floor(). Also note: Python's / ALWAYS returns a float, even 6 / 3 gives 2.0.

Comparison & Equality

JavaScript
// == does type coercion (avoid!)
1 == "1"    // true (wat?)

// === strict equality (use this)
1 === "1"   // false
1 === 1     // true

// Comparisons
5 > 3       // true
5 >= 5      // true
5 != 3      // true
Python
# == compares values (no coercion!)
1 == "1"    # False (sensible!)

# No === needed
1 == 1      # True
# is checks identity (same object)

# Comparisons
5 > 3       # True
5 >= 5      # True
5 != 3      # True

Good news: Python's == doesn't do the crazy type coercion that JavaScript's == does. You never need === because == already works correctly. Use is only for identity checks — mainly x is None.

Logical Operators

JavaScript
// Symbols
if (a && b) { ... }
if (a || b) { ... }
if (!a) { ... }

// Short-circuit
const name = user && user.name;
const fallback = value || "default";
Python
# English words!
if a and b: ...
if a or b: ...
if not a: ...

# Short-circuit works the same
name = user and user.name
fallback = value or "default"

Assignment Operators

JavaScript
let x = 10;
x += 5;    // 15
x -= 3;    // 12
x *= 2;    // 24
x /= 4;    // 6
x++;       // 7 (increment)
x--;       // 6 (decrement)
Python
x = 10
x += 5     # 15
x -= 3     # 12
x *= 2     # 24
x /= 4     # 6.0 (float!)
x += 1     # No ++ in Python
x -= 1     # No -- in Python

No ++ or -- in Python. Use x += 1 and x -= 1 instead. Also note that /= gives you a float. Use //= for integer floor division assignment.

Printing & Input

JavaScript
// Output
console.log("Hello");
console.log("Name:", name, "Age:", age);
console.error("Something broke");
console.warn("Watch out");

// Input (Node.js — complicated)
const readline = require('readline');
// ... 10 lines of boilerplate
Python
# Output
print("Hello")
print("Name:", name, "Age:", age)
print("Something broke", file=sys.stderr)
# No console.warn equivalent

# Input (built-in, simple!)
name = input("What's your name? ")
age = int(input("How old? "))  # convert to int

Python's print() auto-adds spaces between arguments and a newline at the end. You can customize: print("a", "b", sep="-", end="!") gives a-b!. And input() always returns a string — cast it if you need a number.

The Top 5 Gotchas

These are the things that will trip you up in the first hour coming from JavaScript. Memorize them now.

1. Indentation IS Syntax

# This works
if True:
    print("indented correctly")

# This BREAKS — IndentationError
if True:
print("not indented")    # IndentationError!

Coming from JS, this is the #1 gotcha. In JavaScript, { } defines blocks. In Python, indentation defines blocks. Mixing tabs and spaces will cause subtle bugs. Configure your editor to insert 4 spaces when you press Tab.

2. No Semicolons

# Python — no semicolons, ever
name = "Trevor"
age = 38
print(f"{name} is {age}")

3. No Curly Braces for Blocks

# Colons start blocks, indentation defines them
if score > 90:
    grade = "A"
    print("Great job!")     # still inside the if
print("Done")              # outside the if (less indented)

4. Comments Use #

JavaScript
// Single line comment

/*
   Multi-line
   comment
*/
Python
# Single line comment

# Python has no block comments
# Just use multiple # lines
# like this

5. Everything is an Object

In JavaScript, primitives like numbers and strings have wrapper objects. In Python, everything is genuinely an object — even integers and functions. This is why type(42) returns <class 'int'>. There are no primitives.

Check Your Understanding

Quick Quiz

1. What's the Python equivalent of const name = "Trevor"?

name = "Trevor" — No const or let. Just assign. If you want to signal it shouldn't change, use ALL_CAPS: NAME = "Trevor".

2. What does "5" + 3 do in Python?

TypeError! Python refuses to guess. You must explicitly convert: "5" + str(3) gives "53", or int("5") + 3 gives 8.

3. How do you write `Total: ${price * quantity}` in Python?

f"Total: {price * quantity}" — Use an f-string. The f prefix + curly braces replaces backtick + dollar sign.

4. What's 7 // 2 in Python?

3 — Floor division. It divides and rounds down. In JavaScript you'd need Math.floor(7 / 2).

5. How do you reverse a string in Python?

"hello"[::-1] gives "olleh". The slice [::-1] means "start to end, stepping backwards." In JS you'd need "hello".split('').reverse().join('').
