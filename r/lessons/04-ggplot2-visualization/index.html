04 Visualization with ggplot2

← Back to lessons

A ggplot2 chart is a sentence you build word by word. Instead of calling one big "make a chart" function with a wall of arguments, you start with data, declare which columns map to which visual properties, then add geometry, scales, labels, and themes as layers joined by +. If you have ever chained methods in pandas or stacked layers in D3, the grammar will feel familiar — just cleaner.

The Grammar of Graphics

Most plotting libraries are imperative: you issue commands one after another — draw axes, add bars, set a title. ggplot2 is declarative: you describe the chart as a structured object, and it figures out the drawing. Every plot is the same three ingredients.

ggplot(data = penguins) +                       # 1. the data frame
  aes(x = flipper_length_mm, y = body_mass_g) +  # 2. mapping: columns -> visuals
  geom_point()                                   # 3. geometry: how to draw it

The mental model: ggplot(data) is the canvas, aes(mapping) says "this column controls the x position, that one controls color," and each geom_*() is a layer of marks drawn from that mapping. You join the pieces with +, not the pipe. The same grammar produces a scatter plot, a bar chart, or a histogram — you just swap the geom.

Python (matplotlib, imperative)
import matplotlib.pyplot as plt

fig, ax = plt.subplots()
ax.scatter(df["flipper_length_mm"],
           df["body_mass_g"])
ax.set_xlabel("Flipper length (mm)")
ax.set_ylabel("Body mass (g)")
ax.set_title("Penguins")
plt.show()
# Each line mutates the figure in place.
R (ggplot2, declarative)
library(ggplot2)

ggplot(penguins,
       aes(x = flipper_length_mm,
           y = body_mass_g)) +
  geom_point() +
  labs(x = "Flipper length (mm)",
       y = "Body mass (g)",
       title = "Penguins")
# You describe the chart; ggplot draws it.

Where ggplot2 comes from. It is the most-used package in the tidyverse for charts, written by Hadley Wickham and built directly on Leland Wilkinson's book The Grammar of Graphics. The "gg" in the name is literally those two words. Load it with library(ggplot2), or get it for free with library(tidyverse).

aes() — Mapping Data to Visuals

The single idea that unlocks ggplot2: aes() connects a column in your data to a visual property (an "aesthetic") — position, color, size, shape, transparency. You never set those properties row by row. You declare the mapping once, and ggplot applies it across every row.

SQL (the columns you have)
SELECT
  flipper_length_mm,   -- goes on x
  body_mass_g,         -- goes on y
  species              -- becomes color
FROM penguins;
R (the mapping in aes)
ggplot(penguins,
  aes(
    x     = flipper_length_mm,
    y     = body_mass_g,
    color = species   # one color per species
  )) +
  geom_point()

Note what you did not write: no loop, no "if species == Adelie use blue." You said "color is driven by species," and ggplot assigned a distinct color to each value and drew a legend automatically.

Aesthetic What it controls Typical column type
x, y Position on the axes Numeric, or categorical
color / colour Outline / point / line color Categorical or numeric
fill Interior fill (bars, areas) Categorical or numeric
size Point size, line width Numeric
shape Point marker shape Categorical
alpha Transparency (0–1) Numeric

Both spellings work. ggplot2 accepts color and colour as aliases — a nod to its New Zealand author. Pick one and stay consistent. This lesson uses color.

Mapping vs Setting — The Single Biggest Gotcha

This trips up everyone once. There are two completely different ways to give a geom a color, and putting things in the wrong place is the most common ggplot2 bug.

  • Mapping (inside aes()): the property comes from a column. ggplot reads each row and picks the color/size/shape from the data.
  • Setting (outside aes(), as a plain geom argument): the property is a constant you choose for every mark.
Mapping — color from a column
# color is INSIDE aes(): driven by data.
ggplot(penguins,
  aes(x = flipper_length_mm,
      y = body_mass_g)) +
  geom_point(aes(color = species))

# Result: each species gets its own
# color + a legend appears.
Setting — a fixed color
# color is OUTSIDE aes(): a constant.
ggplot(penguins,
  aes(x = flipper_length_mm,
      y = body_mass_g)) +
  geom_point(color = "steelblue")

# Result: every point is the same
# blue. No legend.

The classic mistake: geom_point(color = "blue") inside aes() — as in aes(color = "blue") — does not make your points blue. ggplot treats "blue" as a one-level categorical column named "blue," maps it through the color scale, and you get a salmon-red point with a legend labeled "blue." If you want a literal color, set it outside aes(). If you want color to depend on data, map it inside aes().

Translation rule: "this visual depends on the data" → put it inside aes(). "this visual is a fixed style choice" → put it as a bare argument to the geom. In matplotlib you would write ax.scatter(..., c=df["species"]) for the first and ax.scatter(..., color="steelblue") for the second — ggplot makes the distinction explicit with the aes() wrapper.

geom_*() — The Layer Vocabulary

The geom decides what kind of mark gets drawn from your mapping. Same data, same aes(), different geom, different chart. Here are the four you will reach for constantly.

geom_point — scatter plots

ggplot(penguins, aes(x = flipper_length_mm, y = body_mass_g)) +
  geom_point()
# Two numeric columns -> a dot per row. The workhorse for correlations.

geom_line — trends over an ordered x

ggplot(stocks, aes(x = date, y = price)) +
  geom_line()
# Connects points in x order. Use for time series, not unordered categories.

geom_col and geom_bar — bars

geom_col — you give the height
# Data already has a y value per category.
ggplot(sales,
  aes(x = region, y = total)) +
  geom_col()
# Draws bar of height = total. Like
# SELECT region, SUM(total) ... already done.
geom_bar — ggplot counts for you
# No y: it counts rows per category.
ggplot(penguins,
  aes(x = species)) +
  geom_bar()
# Equivalent to a GROUP BY species,
# COUNT(*) under the hood.

geom_col vs geom_bar: geom_col() uses the y value you supply (you already aggregated). geom_bar() ignores y and counts rows itself (stat = "count"). If your data frame already has one row per bar with a total, use geom_col(). If you have raw rows and want a frequency count, use geom_bar().

geom_histogram — distribution of one numeric column

ggplot(penguins, aes(x = body_mass_g)) +
  geom_histogram(bins = 30)
# Buckets a numeric column and counts how many rows fall in each bin.
# Set bins (count) or binwidth (size). Always tune one of them.

The default 30 bins is rarely right. ggplot2 warns: "stat_bin() using bins = 30. Pick better value with binwidth." Always set bins = or binwidth = deliberately so the shape of your distribution is honest, not an artifact of the default.

Goal Geom Pandas / matplotlib analogue
Two numerics, relationship geom_point() df.plot.scatter(x, y)
Trend over ordered x geom_line() df.plot.line(x, y)
Pre-aggregated bars geom_col() df.plot.bar()
Count rows per category geom_bar() df["c"].value_counts().plot.bar()
Distribution of one numeric geom_histogram() df["x"].plot.hist()
Smoothed trend / fit line geom_smooth() seaborn regplot

Stacking Layers with +

The real payoff of the grammar: a plot can carry several geoms at once, and they share the same data and mapping. Want raw points and a trend line? Add both layers.

ggplot(penguins, aes(x = flipper_length_mm, y = body_mass_g)) +
  geom_point(aes(color = species)) +       # layer 1: the dots, colored by species
  geom_smooth(method = "lm", se = FALSE)   # layer 2: a linear fit line on top

If you know D3, this is the same idea. In D3 you select a layer, bind data, and append marks — then do it again for the next layer. ggplot2 hides the SVG plumbing: each + geom_*() is one more .append() of marks bound to the same data. The difference is you describe what the layer is, not how to draw every element.

Order matters for overlap. Layers draw bottom to top in the order you write them. Put geom_point() before geom_smooth() and the line sits on top of the dots; flip them and the dots cover the line. It is z-order, controlled by source order.

Scales — Controlling How Mappings Render

A scale governs how a mapped column turns into pixels, colors, or tick marks. ggplot picks sensible defaults, but scale_*() functions let you override them. The naming is regular: scale_ + the aesthetic + how it varies.

ggplot(penguins, aes(x = flipper_length_mm, y = body_mass_g, color = species)) +
  geom_point() +
  scale_x_continuous(limits = c(170, 240)) +        # zoom the x axis
  scale_y_continuous(labels = scales::comma) +      # 6,000 not 6000
  scale_color_brewer(palette = "Set2")              # swap the color palette
Function What it does
scale_x_continuous() Numeric x axis: limits, breaks, label format
scale_y_log10() Log-transform the y axis
scale_color_manual(values = ...) Pick exact colors by hand
scale_color_brewer(palette = ...) Use a ColorBrewer palette
scale_fill_viridis_c() Perceptually uniform fill for a continuous variable

The scales::comma trick formats axis labels. The scales package (a ggplot2 companion) gives you comma, dollar, and percent — so a y axis can read $1,500 or 25% instead of raw numbers. The scales:: prefix calls a function without attaching the whole package.

labs() — Titles and Labels

One function sets every text label on the plot: title, subtitle, axis names, legend names, and a caption. The legend label comes from the aesthetic it is tied to.

ggplot(penguins, aes(x = flipper_length_mm, y = body_mass_g, color = species)) +
  geom_point() +
  labs(
    title    = "Bigger flippers, heavier penguins",
    subtitle = "Palmer Archipelago, 2007 to 2009",
    x        = "Flipper length (mm)",
    y        = "Body mass (g)",
    color    = "Species",            # renames the legend title
    caption  = "Source: palmerpenguins"
  )

Translation rule: matplotlib spreads labels across set_title, set_xlabel, set_ylabel, legend(title=...). ggplot folds them all into one labs() call where the argument name is the aesthetic. Want to rename the color legend? Set color = "Species" inside labs() — the same name you used in aes().

facet_wrap() — Small Multiples

Faceting splits one plot into a grid of sub-plots, one per group, all sharing the same axes. It is the "small multiples" technique: instead of cramming three species into one cluttered scatter, give each its own panel for honest comparison.

SQL (the grouping idea)
-- One result set per species,
-- then you would chart each.
SELECT * FROM penguins
ORDER BY species;
# You'd loop and plot per group by hand.
R (one line does it)
ggplot(penguins,
  aes(x = flipper_length_mm,
      y = body_mass_g)) +
  geom_point() +
  facet_wrap(~ species)
# One panel per species, shared axes.

The ~ species is a formula: the tilde means "by." For a grid split across two variables, use facet_grid(rows ~ cols).

facet_wrap(~ species)              # one variable, panels wrap onto multiple rows
facet_wrap(~ species, ncol = 1)   # force a single column
facet_grid(island ~ species)      # 2D grid: islands down, species across
facet_wrap(~ year, scales = "free_y")  # let each panel scale its own y axis

The mental model: faceting in ggplot2 is df.groupby(...) followed by drawing the same chart per group — but as one declaration instead of a loop. In pandas you would iterate groups and call subplots; here facet_wrap(~ col) does the split, the layout, and the matching axes for you.

Themes — The Non-Data Look

Themes control everything that is not the data: background, gridlines, fonts, legend position, tick marks. Swapping a complete theme is one layer; tweaking a single element is theme().

ggplot(penguins, aes(x = flipper_length_mm, y = body_mass_g, color = species)) +
  geom_point() +
  theme_minimal() +                       # a clean, full preset theme
  theme(legend.position = "bottom")        # then override one detail
Theme Look
theme_gray() Default: gray panel, white gridlines
theme_minimal() No panel border, light gridlines
theme_bw() White background, thin black border
theme_classic() Axis lines only, no gridlines
theme_void() Nothing but the data (good for maps)

Apply the preset theme before your tweaks. A full theme like theme_minimal() resets every element, so it overwrites earlier theme() calls. Order it before a custom theme(...), or your one-off adjustments get wiped out.

Piping dplyr into ggplot

In real work you rarely chart raw data. You filter, group, and summarise first — then plot. dplyr uses the pipe (|> in base R, %>% from magrittr); ggplot2 layers use +. They meet at exactly one spot: the pipe hands the cleaned data frame into ggplot(), and from there you switch to +.

Python (pandas, then plot)
(df
  .query("body_mass_g.notna()")
  .groupby("species", as_index=False)
  .agg(avg_mass=("body_mass_g", "mean"))
  .plot.bar(x="species", y="avg_mass"))
R (dplyr pipe, then ggplot)
penguins |>
  filter(!is.na(body_mass_g)) |>
  group_by(species) |>
  summarise(avg_mass = mean(body_mass_g)) |>
  ggplot(aes(x = species, y = avg_mass)) +
    geom_col()

Watch the operator switch. Use |> to chain dplyr verbs, but the moment you reach ggplot(), every following layer joins with +. Writing ggplot(...) |> geom_col() fails, and filter(...) + group_by(...) fails. The pipe carries data; the plus stacks plot layers. Note that when data arrives via the pipe, you skip the first ggplot() argument and pass aes() directly.

Putting It All Together

A realistic chart that uses the whole grammar — clean the data, map several aesthetics, stack two geoms, format the scales, label everything, facet, and theme. This is the shape of a publication-ready figure.

library(tidyverse)   # dplyr + ggplot2 + scales, all at once

penguins |>
  # 1. Clean: drop rows missing the columns we plot
  filter(!is.na(flipper_length_mm), !is.na(body_mass_g)) |>

  # 2. Canvas + base mapping
  ggplot(aes(x = flipper_length_mm, y = body_mass_g)) +

  # 3. Layer one: the points, colored by sex (mapped inside aes)
  geom_point(aes(color = sex), alpha = 0.7) +   # alpha set, not mapped

  # 4. Layer two: a linear trend line per panel
  geom_smooth(method = "lm", se = FALSE, color = "grey30") +

  # 5. Small multiples: one panel per species
  facet_wrap(~ species) +

  # 6. Scales: comma-format the mass axis, choose a palette
  scale_y_continuous(labels = scales::comma) +
  scale_color_brewer(palette = "Dark2", na.translate = FALSE) +

  # 7. Every label in one place
  labs(
    title    = "Flipper length predicts body mass across species",
    subtitle = "Palmer penguins, split by species, trend line fitted per panel",
    x        = "Flipper length (mm)",
    y        = "Body mass (g)",
    color    = "Sex",
    caption  = "Source: palmerpenguins"
  ) +

  # 8. Look and feel last, so it does not get overwritten
  theme_minimal() +
  theme(legend.position = "bottom")

Read it top to bottom and it reads like a recipe: take the data, keep complete rows, set up the axes, drop the colored points, add a fit line, split into panels, format the numbers, write the labels, polish the theme. No single function had a dozen arguments. Each line did one job. That is the grammar of graphics paying off — the same eight moves describe almost any chart you will ever build in R.

Check Your Understanding

Quick Quiz

1. What are the three core ingredients of every ggplot2 chart, and what joins them?

A data frame in ggplot(data), a mapping in aes(...) that ties columns to visual properties, and one or more geom_*() layers that draw the marks. They are joined with +, not the pipe.

2. Why does geom_point(aes(color = "blue")) not produce blue points?

Because it is inside aes(), ggplot treats "blue" as a categorical column with one value, maps it through the color scale, and draws the default first color with a legend labeled "blue." For a literal color, set it outside aes(): geom_point(color = "blue"). Mapping = from data; setting = a constant.

3. When would you use geom_col() instead of geom_bar()?

Use geom_col() when your data already has a y value per bar (you aggregated, e.g. one row per region with a total). Use geom_bar() when you have raw rows and want ggplot to count them per category — it is a built-in GROUP BY with COUNT(*).

4. What does facet_wrap(~ species) do, and what is the pandas equivalent?

It splits the plot into small multiples — one panel per species, all sharing the same axes. The tilde formula means "by species." In pandas you would groupby("species") and draw the same chart per group in a loop; faceting does the split, layout, and matching axes in one declaration.

5. You chain dplyr verbs with the pipe, then want to plot. What operator change do you make, and why?

Keep using |> (or %>%) to pipe data through filter, group_by, summarise and into ggplot(). From ggplot() onward, switch to + to stack layers. The pipe carries the data frame forward; the plus adds plot layers. Mixing them up is an error.
