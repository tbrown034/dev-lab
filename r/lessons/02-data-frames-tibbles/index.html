02 Data Frames & Tibbles

← Back to lessons

A data frame is a table, and tables are where R earns its living. If you know a Pandas DataFrame or a SQL table, you already understand the shape of the thing: rows of observations, columns of variables. This lesson maps the R data frame — and its modern cousin, the tibble — onto the tools you already use, then walks through reading, inspecting, and reshaping real data.

What a Data Frame Actually Is

A data frame is a list of columns of equal length. Each column is a vector with a single type; different columns can have different types. That is the whole idea — a spreadsheet where every column is internally consistent.

The mental model: a data frame is a Pandas DataFrame and a SQL table at the same time. Columns are typed like SQL columns and addressable by name like Pandas Series. The difference from a matrix: a matrix is one type all the way through, while a data frame mixes types column by column.

Python (Pandas)
import pandas as pd

df = pd.DataFrame({
    "name": ["Ada", "Linus", "Grace"],
    "age": [36, 28, 45],
    "active": [True, False, True],
})

df.shape   # (3, 3)
df.columns # Index(['name','age','active'])
R
df <- data.frame(
  name   = c("Ada", "Linus", "Grace"),
  age    = c(36, 28, 45),
  active = c(TRUE, FALSE, TRUE)
)

dim(df)    # 3 3
names(df)  # "name" "age" "active"
nrow(df)   # 3
ncol(df)   # 3

Translation rule: pd.DataFrame({...}) → data.frame(...). Pandas uses a dict of column name to list; R uses named arguments where each argument is a c() vector. df.shape → dim(df). df.columns → names(df).

The Tibble — A Modern Data Frame

A tibble is the tidyverse's data frame. It behaves like a data.frame but fixes a handful of old annoyances: it prints cleanly, never silently converts strings to factors, and never partial-matches column names.

R (base data.frame)
df <- data.frame(
  name = c("Ada", "Linus"),
  age  = c(36, 28)
)

# Prints all rows, no type hints
df
R (tibble)
library(tibble)  # or library(tidyverse)

tb <- tibble(
  name = c("Ada", "Linus"),
  age  = c(36, 28)
)

# Prints first 10 rows + column types
tb
# # A tibble: 2 x 2
#   name    age
#   <chr> <dbl>
# 1 Ada      36
# 2 Linus    28

Why people prefer tibbles: the printout tells you the column type (<chr>, <dbl>, <int>, <lgl>) right under each name, and it shows only the first 10 rows so a big table will not flood your console. Under the hood a tibble is a data frame — class(tb) includes "data.frame" — so anything that takes a data frame will take a tibble.

Reading Data — read.csv vs read_csv

Base R ships with read.csv(). The tidyverse readr package adds read_csv() — note the underscore. They look almost identical but differ in defaults and output.

Python (Pandas)
import pandas as pd

df = pd.read_csv("people.csv")

# Common options
df = pd.read_csv(
    "people.csv",
    na_values=["NA", ""],
    dtype={"id": "Int64"},
)
R
# Base R, returns a data.frame
df <- read.csv("people.csv")

# readr, returns a tibble (preferred)
library(readr)
df <- read_csv("people.csv")

# read_csv guesses column types and
# prints a one-line spec of what it found
Concern read.csv() (base) read_csv() (readr)
Returns data.frame tibble
Strings to factors No (since R 4.0) Never
Speed on big files Slower Much faster
Column name cleanup Mangles spaces to dots Keeps names as-is
Type guessing Basic Reports a column spec

Watch the dot versus the underscore. read.csv is base R; read_csv is readr. They are different functions with different defaults. The underscore version is the tidyverse one and is what most modern R code uses. Picking the wrong one is a common copy-paste error.

Inspecting a Data Frame

Before you touch a dataset, you look at it. R gives you several lenses: a peek at the top rows, a structural summary, descriptive statistics, and the tidyverse glimpse().

Python (Pandas)
df.head()        # first 5 rows
df.head(3)       # first 3 rows
df.tail()        # last 5 rows

df.info()        # types + non-null counts
df.describe()    # numeric summary
df.dtypes        # type per column
R
head(df)         # first 6 rows
head(df, 3)      # first 3 rows
tail(df)         # last 6 rows

str(df)          # structure: types + samples
summary(df)      # per-column summary
glimpse(df)      # tidyverse: transposed view

Translation rule: df.head() → head(df) (R defaults to 6 rows, not 5). df.info() → str(df) for a structural look, or glimpse(df) for the cleaner tidyverse version. df.describe() → summary(df). R puts the data frame inside the function call; Pandas calls a method on it.

What Each One Shows You

# str() reports type and first few values per column
str(df)
# 'data.frame': 3 obs. of 3 variables:
#  $ name  : chr  "Ada" "Linus" "Grace"
#  $ age   : num  36 28 45
#  $ active: logi  TRUE FALSE TRUE

# summary() gives stats: min/median/mean for numbers,
# counts for factors, lengths for character
summary(df)

# glimpse() is str() reimagined: one row per column,
# type in angle brackets, values running across
library(dplyr)
glimpse(df)
# Rows: 3
# Columns: 3
# $ name   <chr> "Ada", "Linus", "Grace"
# $ age    <dbl> 36, 28, 45
# $ active <lgl> TRUE, FALSE, TRUE

Use glimpse() on wide tables. When a data frame has 30 columns, head() wraps and becomes unreadable. glimpse() lists one column per line, so you can scan every column and its type at a glance — the closest thing R has to Pandas df.info().

Column Access — $, [[ ]], and [ ]

This is where R diverges most from the languages you know. There are three bracket idioms and each returns something slightly different.

Python (Pandas)
# A single column as a Series
df["age"]
df.age          # attribute access

# A one-column DataFrame
df[["age"]]

# Multiple columns
df[["name", "age"]]

# Single cell
df.loc[0, "age"]
R
# A single column as a vector
df$age
df[["age"]]

# A one-column data frame
df["age"]

# Multiple columns (still a data frame)
df[c("name", "age")]

# Single cell: [row, column]
df[1, "age"]

The mental model: single brackets [ ] keep the data frame wrapper and return a data frame; double brackets [[ ]] and $ reach inside and pull out the bare column vector. It mirrors Pandas exactly: df["age"] in Pandas gives a Series (a bare column), and in R the equivalent bare column comes from df$age or df[["age"]]. R's single-bracket df["age"] is the analog of Pandas df[["age"]] — it keeps the table shape.

Row, Column, and Cell Selection

# df[rows, columns] — leave a side blank to keep all
df[1, ]            # first row, all columns
df[, "age"]        # all rows, age column (drops to vector)
df[1:2, c("name", "age")]  # rows 1-2, two columns

# Logical filtering — like Pandas boolean indexing
df[df$age > 30, ]          # rows where age > 30
df[df$active, "name"]      # names of active people

R is 1-based, and the comma matters. df[1, ] is the first row (R counts from 1, not 0). The comma separates rows from columns: df[rows, cols]. Forgetting the comma changes the meaning entirely — df[1] selects the first column, while df[1, ] selects the first row.

Adding and Removing Columns

Python (Pandas)
# Add a column
df["age_months"] = df["age"] * 12

# Add a constant column
df["source"] = "import"

# Remove a column
df = df.drop(columns=["source"])

# Rename
df = df.rename(columns={"name": "full_name"})
R
# Add a column
df$age_months <- df$age * 12

# Add a constant column (recycled)
df$source <- "import"

# Remove a column (assign NULL)
df$source <- NULL

# Rename via names()
names(df)[names(df) == "name"] <- "full_name"

Translation rule: assigning to a new df$col adds or overwrites a column, exactly like df["col"] = ... in Pandas. The R surprise is deletion: there is no drop in base R — you assign NULL to the column and it disappears. A scalar like "import" is recycled to fill every row, the same as broadcasting in Pandas.

The tidyverse way is cleaner. With dplyr you would write mutate(df, age_months = age * 12) to add a column and select(df, -source) to drop one, and rename(df, full_name = name) to rename. That is the subject of the next lesson; here we are staying in base R so you can see the machinery underneath.

Factors — R's Categorical Type

A factor stores categorical data: a fixed set of allowed values called levels. It is R's version of a Pandas Categorical or a SQL ENUM. Under the hood a factor is an integer vector plus a lookup table of labels.

Python (Pandas)
sizes = pd.Categorical(
    ["S", "L", "M", "S"],
    categories=["S", "M", "L"],
    ordered=True,
)

sizes.categories  # ['S','M','L']
sizes > "S"       # comparable when ordered
R
sizes <- factor(
  c("S", "L", "M", "S"),
  levels = c("S", "M", "L"),
  ordered = TRUE
)

levels(sizes)    # "S" "M" "L"
sizes > "S"      # comparable when ordered
as.integer(sizes) # 1 3 2 1 (the codes)

The mental model: a factor is two things glued together — an integer code per value and a vector of level labels. levels() shows the labels in their defined order, which also sets sort and plot order. Set ordered = TRUE when the categories have a rank (small/medium/large); leave it off for plain categories (red/green/blue).

The stringsAsFactors History

For most of R's life, data.frame() and read.csv() silently turned every character column into a factor. That default was stringsAsFactors = TRUE, and it surprised generations of beginners whose text columns behaved like locked-down categories.

# Before R 4.0 (old default, no longer the case):
df <- data.frame(name = c("Ada", "Linus"))
# name was a FACTOR, not character — surprise!

# R 4.0 (April 2020) flipped the default to FALSE.
# Now character columns stay character:
df <- data.frame(name = c("Ada", "Linus"))
class(df$name)  # "character"

# You can still opt in explicitly:
df <- data.frame(
  size = c("S", "M", "L"),
  stringsAsFactors = TRUE
)

This is the classic R gotcha in old code. Scripts written before April 2020 may rely on the old stringsAsFactors = TRUE default and break subtly under modern R. Tibbles never did this — read_csv() and tibble() have always left strings as strings. When in doubt, check a column with class() and convert deliberately with factor() or as.character().

Putting It All Together

A realistic walk-through: load a CSV of survey responses, inspect it, clean a column into a factor, derive a new column, and pull out a slice — the everyday loop of working with tabular data in R.

library(readr)
library(dplyr)

# 1. Read the file as a tibble (types are guessed)
survey <- read_csv("survey.csv")

# 2. Look before you leap
glimpse(survey)
# Rows: 250
# Columns: 4
# $ id       <dbl> 1, 2, 3, ...
# $ age      <dbl> 34, 51, 22, ...
# $ size     <chr> "M", "L", "S", ...
# $ rating   <dbl> 4, 5, 2, ...

summary(survey$rating)   # min/median/mean/max

# 3. Turn the size text into an ordered factor
survey$size <- factor(
  survey$size,
  levels = c("S", "M", "L"),
  ordered = TRUE
)
levels(survey$size)      # "S" "M" "L"

# 4. Derive a new column (recycled comparison)
survey$is_fan <- survey$rating >= 4

# 5. Slice: ids of large-size fans, base-R style
big_fans <- survey[survey$size == "L" & survey$is_fan,
                   c("id", "rating")]

# 6. Confirm the result
nrow(big_fans)
head(big_fans)

# Drop a working column when done
survey$is_fan <- NULL

Notice the rhythm: read, inspect, clean, derive, slice, confirm. It is the same loop you run in Pandas or SQL — only the syntax changes. The next lesson swaps this base-R machinery for dplyr verbs (filter, select, mutate) chained with the pipe, which reads far closer to a SQL query.

Check Your Understanding

Quick Quiz

1. What is the difference between df$age and df["age"] in R?

df$age (like df[["age"]]) reaches inside and returns the bare column as a vector. df["age"] with single brackets keeps the data frame wrapper and returns a one-column data frame. The Pandas parallel: df$age is like df["age"] (a Series), while R's df["age"] is like Pandas df[["age"]] (a DataFrame).

2. How do you select the first row of a data frame, and why is the comma important?

df[1, ]. R is 1-based, so index 1 is the first row. The comma separates rows from columns in df[rows, cols]; leaving the column side blank keeps all columns. Without the comma, df[1] selects the first column instead.

3. What is the difference between read.csv() and read_csv()?

read.csv() is base R and returns a data.frame. read_csv() (with an underscore) is from the readr package, returns a tibble, is faster on large files, never converts strings to factors, and prints a column-type spec. Modern R code generally uses read_csv().

4. How do you remove a column from a base R data frame?

Assign NULL to it: df$source <- NULL. There is no drop method in base R. The dplyr alternative is select(df, -source).

5. What was stringsAsFactors, and what changed in R 4.0?

It was a default that made data.frame() and read.csv() turn character columns into factors automatically. Until R 4.0 (April 2020) the default was TRUE, which surprised beginners; R 4.0 flipped it to FALSE so strings stay strings. Tibbles never did this conversion at all.
