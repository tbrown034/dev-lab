Key Queries

Counties with fastest population growth (2018-2023)
WITH growth AS (
  SELECT
    g.name, g.state_fips,
    d.population,
    LAG(d.population) OVER (PARTITION BY d.geo_id ORDER BY d.year) AS prev_pop,
    d.year
  FROM demographics d
  JOIN geographies g ON d.geo_id = g.id
  WHERE g.geo_level = 'county'
)
SELECT name, state_fips,
  ROUND((population - prev_pop) * 100.0 / prev_pop, 2) AS growth_pct
FROM growth
WHERE year = 2023 AND prev_pop IS NOT NULL
ORDER BY growth_pct DESC
LIMIT 25;

How I Built This

  • Imported ACS 5-year CSV files using COPY with custom staging tables for data cleaning
  • Normalized raw Census tables into geography, demographics, education, and housing dimensions
  • Window functions (PERCENT_RANK, LAG, NTILE) for rankings and trend analysis
  • Materialized views for pre-computed state-level summaries, refreshed on data import
  • GeoJSON served via JSON API endpoints for D3 choropleth map rendering
  • Full-text search on geography names with tsvector and GIN index

Key Queries

Top employers by total contributions to each party
SELECT
  c.employer,
  SUM(c.amount) FILTER (WHERE ca.party = 'DEM') AS dem_total,
  SUM(c.amount) FILTER (WHERE ca.party = 'REP') AS rep_total,
  SUM(c.amount) AS grand_total,
  RANK() OVER (ORDER BY SUM(c.amount) DESC) AS rank
FROM contributions c
JOIN committees cm ON c.committee_id = cm.id
JOIN candidates ca ON cm.candidate_id = ca.id
WHERE c.employer IS NOT NULL
  AND c.employer NOT IN ('RETIRED', 'SELF-EMPLOYED', 'NOT EMPLOYED')
GROUP BY c.employer
ORDER BY grand_total DESC
LIMIT 30;

How I Built This

  • Bulk imported 20M+ FEC contribution records using COPY with custom delimiter handling
  • Table partitioning by election cycle year for query performance on historical data
  • GIN indexes on tsvector columns and pg_trgm extension for fuzzy name matching
  • Recursive CTE traces money flow: donor to PAC to candidate across committee-to-committee transfers
  • Materialized views pre-compute top donors, employer rankings, and ZIP code aggregates
  • BRIN index on contribution date for efficient time-range queries on the append-only data

Key Queries

Funding equity analysis: per-pupil spending variation by state
SELECT
  d.state,
  COUNT(DISTINCT d.id) AS districts,
  ROUND(AVG(d.total_revenue / NULLIF(e.total, 0)), 0) AS avg_per_pupil,
  ROUND(PERCENTILE_CONT(0.5) WITHIN GROUP
    (ORDER BY d.total_revenue / NULLIF(e.total, 0)), 0) AS median_per_pupil,
  ROUND(STDDEV(d.total_revenue / NULLIF(e.total, 0)), 0) AS stddev_per_pupil,
  ROUND(MAX(d.total_revenue / NULLIF(e.total, 0)) /
    NULLIF(MIN(d.total_revenue / NULLIF(e.total, 0)), 0), 1) AS max_min_ratio
FROM districts d
JOIN enrollment e ON e.school_id IN (
  SELECT id FROM schools WHERE district_id = d.id
)
WHERE e.year = 2023
GROUP BY d.state
ORDER BY max_min_ratio DESC;

How I Built This

  • Imported NCES Common Core of Data (CCD) files spanning 2013-2023 using COPY with staging tables
  • Data cleaning pipeline in SQL: standardized school names, filled missing FIPS codes, deduplicated records
  • GROUPING SETS (school_id, district_id, state, ()) for multi-level aggregation in a single pass
  • CORR(test_scores.proficient_pct, enrollment.free_lunch_pct) for poverty-achievement correlation
  • Materialized views for state-level dashboards, refreshed nightly via pg_cron
  • GiST index on school coordinates for geographic proximity queries (nearest school search)

Key Queries

Agencies with largest homicide clearance rate decline (2019-2023)
WITH clearance_rates AS (
  SELECT
    a.name AS agency,
    a.state,
    cr.year,
    SUM(cr.cleared) * 100.0 / NULLIF(SUM(cr.reported), 0) AS clearance_pct
  FROM crime_reports cr
  JOIN agencies a ON cr.agency_id = a.id
  WHERE cr.offense_type = 'homicide'
    AND cr.year IN (2019, 2023)
    AND a.population > 100000
  GROUP BY a.name, a.state, cr.year
),
pivoted AS (
  SELECT agency, state,
    MAX(clearance_pct) FILTER (WHERE year = 2019) AS rate_2019,
    MAX(clearance_pct) FILTER (WHERE year = 2023) AS rate_2023
  FROM clearance_rates
  GROUP BY agency, state
)
SELECT agency, state,
  ROUND(rate_2019, 1) AS rate_2019,
  ROUND(rate_2023, 1) AS rate_2023,
  ROUND(rate_2023 - rate_2019, 1) AS change
FROM pivoted
WHERE rate_2019 IS NOT NULL AND rate_2023 IS NOT NULL
ORDER BY change ASC
LIMIT 20;

How I Built This

  • Imported FBI UCR data (10+ years, 18K agencies) with COPY and custom ETL scripts for format variations
  • Table partitioning by year on crime_reports for fast time-range queries on 50M+ rows
  • LATERAL JOIN for "top N crimes per agency" without correlated subqueries
  • Statistical functions: REGR_SLOPE for trend lines, CORR for officer-count vs crime-rate analysis
  • Automated monthly report generation using pg_cron and COPY TO for CSV export
  • Materialized views for state-level summaries with CONCURRENTLY refresh during off-peak hours
