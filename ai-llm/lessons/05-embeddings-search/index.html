05 Embeddings & Search — GPS for Meaning

← Back to lessons

Keyword search finds words. Semantic search finds meaning. This lesson teaches you to build search that understands what you actually want, even when you don't use the right words.

Mental Model: GPS Coordinates for Meaning

Imagine every piece of text has GPS coordinates — not on a map, but in a meaning space. Texts about similar topics are close together. Texts about different topics are far apart.

  • "The mayor announced budget cuts" and "City leader reduces spending" are near each other (same meaning, different words).
  • "The mayor announced budget cuts" and "Recipe for chocolate cake" are far apart (completely different topics).

An embedding is this GPS coordinate: a list of numbers (a vector) that represents the meaning of a piece of text. Similar meanings produce similar vectors.

Why this matters: Traditional keyword search for "budget reduction" won't find an article that says "spending cuts." Semantic search with embeddings WILL, because both phrases map to nearby coordinates in meaning space.

What Is a Vector?

A vector is just a list of numbers. An embedding vector for text typically has 1,536 numbers (OpenAI) or similar dimensions. Each number captures one aspect of meaning.

// Simplified 3D embedding (real ones have 1,536 dimensions)
"budget cuts"     → [0.82, 0.15, -0.43]
"spending reduction" → [0.79, 0.18, -0.41]  // Very close!
"chocolate cake"  → [-0.21, 0.89, 0.34]     // Very far!

// "Distance" between vectors = how different the meanings are
// Close vectors = similar meaning
// Far vectors = different meaning

You don't need to understand the math. The embedding model handles the conversion from text to vector. You just need to understand the concept: text → numbers → similarity comparison. The rest is plumbing.

Generating Embeddings

To create embeddings, you call an embedding API. Anthropic partners with Voyage AI for embeddings, or you can use OpenAI's embedding model:

import OpenAI from "openai";

const openai = new OpenAI();

// Generate an embedding for a single text
const response = await openai.embeddings.create({
  model: "text-embedding-3-small",
  input: "The city council voted to reduce the police budget by 15%"
});

const embedding = response.data[0].embedding;
console.log(embedding.length);  // 1536 numbers
console.log(embedding.slice(0, 5)); // [0.023, -0.041, 0.008, ...]
// Batch embedding — more efficient for multiple texts
const articles = [
  "City council approves new budget",
  "Mayor announces infrastructure plan",
  "School board votes on curriculum changes",
  "Police department gets new equipment"
];

const response = await openai.embeddings.create({
  model: "text-embedding-3-small",
  input: articles
});

const embeddings = response.data.map(d => d.embedding);
// embeddings[0] corresponds to articles[0], etc.

Similarity Search: Finding the Closest Match

To find which article is most similar to a query, compute the cosine similarity between the query's embedding and each article's embedding:

// Cosine similarity: how aligned are two vectors?
// 1.0 = identical meaning, 0 = unrelated, -1 = opposite

function cosineSimilarity(a, b) {
  let dot = 0, normA = 0, normB = 0;
  for (let i = 0; i < a.length; i++) {
    dot += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }
  return dot / (Math.sqrt(normA) * Math.sqrt(normB));
}

// Search: embed the query, compare against all stored embeddings
async function search(query, articleEmbeddings, articles) {
  const queryResponse = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: query
  });
  const queryEmbed = queryResponse.data[0].embedding;

  const results = articles.map((article, i) => ({
    article,
    score: cosineSimilarity(queryEmbed, articleEmbeddings[i])
  }));

  return results.sort((a, b) => b.score - a.score);
}

const results = await search("government spending", embeddings, articles);
console.log(results[0]);
// { article: "City council approves new budget", score: 0.87 }

This is the entire concept of semantic search. Embed your documents once, store the vectors, then embed each query and find the closest matches. No keyword matching. No boolean logic. Just "find me the texts whose meaning is closest to this query."

import { ChromaClient } from "chromadb";

const chroma = new ChromaClient();
const collection = await chroma.createCollection({ name: "articles" });

// Add documents (ChromaDB can auto-embed them)
await collection.add({
  ids: ["art-001", "art-002", "art-003"],

    "Mayor announces plan to reduce homeless population",
    "School board votes to add coding curriculum"

// Query — returns most similar documents
const results = await collection.query({
  queryTexts: ["government spending decisions"],
  nResults: 2
});

console.log(results.documents);
// [["City council approves new budget...", "Mayor announces plan..."]]

Journalism Use Case: Searchable Archive

Imagine searching your newspaper's archive not by keyword, but by concept:

  • "stories about government accountability" — finds articles about audits, corruption, transparency, oversight, even if they never use the word "accountability"
  • "coverage of housing affordability" — finds articles about rent, mortgages, zoning, displacement, gentrification
  • "sources who've criticized the police department" — finds quotes and attributions across hundreds of articles

This is the foundation for RAG (Lesson 06). Once you can search by meaning, you can retrieve relevant documents and feed them to Claude as context. That's how you ground AI responses in your actual reporting.

Check Your Understanding

1. What is an embedding?

A list of numbers (vector) that represents the meaning of a piece of text. Similar texts produce similar vectors. It's like GPS coordinates for meaning in a high-dimensional space.

2. Why is semantic search better than keyword search for journalism?

Keyword search only finds exact word matches. Semantic search finds conceptually similar content regardless of specific wording. A search for "government spending" will find articles about "budget allocations," "fiscal policy," and "taxpayer funding" even if those exact words weren't in the query.

3. What is cosine similarity measuring?

How aligned two vectors are in direction. A score of 1.0 means identical meaning, 0 means unrelated, -1 means opposite. It's the standard way to compare embeddings.

4. When should you use a vector database vs. a simple loop?

A simple loop works for up to a few hundred documents. For thousands or millions, you need a vector database that uses indexing algorithms (like HNSW) for fast approximate nearest-neighbor search.
