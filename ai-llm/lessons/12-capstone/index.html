What You're Building

A Newsroom Research Assistant with four core capabilities:

  1. Source Search — Find relevant articles and sources using semantic search (Lesson 05)
  2. Fact Check — Extract claims from text and evaluate them against your archive (Lessons 06, 08)
  3. Draft Generation — Generate article drafts grounded in retrieved sources with citations (Lesson 06)
  4. Research Agent — An autonomous agent that investigates a topic using all available tools (Lesson 07)

The architecture uses every concept from this course: The Claude API (03) for generation. Tools (04) for the agent. Embeddings (05) for search. RAG (06) for grounded generation. Agent patterns (07) for autonomous research. Structured output (08) for data extraction. Streaming (09) for UX. Evals (10) for quality. Ethics (11) for responsible deployment.

Step 3: Implement the Tools

// tools/handlers.js
import Anthropic from "@anthropic-ai/sdk";
import { ChromaClient } from "chromadb";

const claude = new Anthropic();
const chroma = new ChromaClient();

export async function searchArticles({ query, n_results = 5 }) {
  const collection = await chroma.getCollection({ name: "newsroom" });
  const results = await collection.query({
    queryTexts: [query],
    nResults: n_results
  });

  return results.documents[0].map((doc, i) => ({
    text: doc,
    article_id: results.metadatas[0][i].article_id,
    title: results.metadatas[0][i].title,
    date: results.metadatas[0][i].date,
    relevance: results.distances?.[0] ? 1 - results.distances[0][i] : null // 1 - distance assumes cosine distance
  }));
}

export async function extractClaims({ text }) {
  const response = await claude.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 1024,
    tools: [{
      name: "save_claims",
      description: "Save extracted claims",
      input_schema: {
        type: "object",
        properties: {
          claims: { type: "array", items: { type: "object", properties: {
            claim: { type: "string" },
            source_in_text: { type: "string" },
            verifiability: { type: "string", enum: ["verifiable", "subjective", "unclear"] }
          }, required: ["claim", "verifiability"] }}
        },
        required: ["claims"]
      }
    }],
    tool_choice: { type: "tool", name: "save_claims" },
    messages: [{ role: "user", content: `Extract all factual claims from:\n\n${text}` }]
  });

  return response.content.find(b => b.type === "tool_use").input;
}

export async function verifyClaim({ claim }) {
  const results = await searchArticles({ query: claim, n_results: 3 });

  if (results.length === 0) {
    return { verdict: "unverified", evidence: [], note: "No relevant articles found in archive" };
  }

  const response = await claude.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 512,
    system: "You verify claims against source material. Be precise. If the evidence doesn't clearly support or contradict the claim, say so.",
    messages: [{
      role: "user",
      content: `Claim: "${claim}"\n\nEvidence:\n${results.map(r => `[${r.title}] ${r.text}`).join("\n\n")}\n\nDoes the evidence support, contradict, or not address this claim?`
    }]
  });

  return {
    verdict: response.content[0].text,
    evidence: results.map(r => ({ title: r.title, article_id: r.article_id }))
  };
}

Step 4: The Research Agent

// agent/research.js
import Anthropic from "@anthropic-ai/sdk";
import { tools } from "../tools/definitions.js";
import * as handlers from "../tools/handlers.js";

const claude = new Anthropic();

const SYSTEM = `You are a research assistant for the Riverside Tribune newsroom.

You help journalists by:
- Searching the article archive for relevant background
- Extracting and verifying factual claims
- Generating cited article drafts
- Investigating topics across multiple sources

RULES:
- Always cite specific articles when making claims
- Flag information you cannot verify
- When generating drafts, use only information from the archive
- If you can't find relevant sources, say so
- Be transparent about the limits of your search`;

const toolMap = {
  search_articles: handlers.searchArticles,
  get_full_article: handlers.getFullArticle,
  extract_claims: handlers.extractClaims,
  verify_claim: handlers.verifyClaim,
  generate_draft: handlers.generateDraft
};

export async function runResearchAgent(task, onStep) {
  let messages = [{ role: "user", content: task }];
  let steps = 0;
  const MAX_STEPS = 15;

  while (steps < MAX_STEPS) {
    const response = await claude.messages.create({
      model: "claude-sonnet-4-6",
      max_tokens: 2048,
      system: SYSTEM,
      tools,
      messages
    });

    steps++;

    if (response.stop_reason === "tool_use") {
      const toolResults = [];

      for (const block of response.content) {
        if (block.type === "tool_use") {
          onStep?.({ type: "tool_call", name: block.name, input: block.input, step: steps });

          try {
            const result = await toolMap[block.name](block.input);
            toolResults.push({
              type: "tool_result",
              tool_use_id: block.id,
              content: JSON.stringify(result)
            });
            onStep?.({ type: "tool_result", name: block.name, result, step: steps });
          } catch (error) {
            toolResults.push({
              type: "tool_result",
              tool_use_id: block.id,
              content: JSON.stringify({ error: error.message }),
              is_error: true
            });
          }
        }
      }

      messages.push({ role: "assistant", content: response.content });
      messages.push({ role: "user", content: toolResults });
      continue;
    }

    // Agent is done
    const text = response.content.find(b => b.type === "text");
    onStep?.({ type: "done", steps });
    return { answer: text?.text, steps, usage: response.usage };
  }

  return { answer: "Reached maximum research steps. Here's what I found so far.", steps };
}

Step 5: The Streaming Server

// server.js
import express from "express";
import { runResearchAgent } from "./agent/research.js";

const app = express();
app.use(express.json());
app.use(express.static("public"));

app.post("/api/research", async (req, res) => {
  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");

  const { task } = req.body;

  const result = await runResearchAgent(task, (step) => {
    res.write(`data: ${JSON.stringify(step)}\n\n`);
  });

  res.write(`data: ${JSON.stringify({
    type: "final",
    answer: result.answer,
    steps: result.steps
  })}\n\n`);

  res.end();
});

app.listen(3000, () => console.log("Newsroom Assistant running on :3000"));

Quality Checklist

Before calling this project complete, verify:

  1. Search works semantically — "government spending" finds articles about "budget" and "fiscal policy"
  2. Fact-check extracts real claims — not summaries or opinions, but verifiable factual assertions
  3. Drafts cite sources — every factual claim in a draft links back to a specific article
  4. Agent stays on task — doesn't go on tangents or make unnecessary tool calls
  5. Agent stops when appropriate — doesn't keep searching when it already has the answer
  6. Errors are handled gracefully — tool failures don't crash the system
  7. Streaming works — the user sees agent progress in real-time
  8. Ethics are embedded — the system never presents unverified information as fact

Check Your Understanding

1. What are the four core capabilities of the newsroom assistant?

Source search (semantic search over the article archive), fact-checking (claim extraction and verification against sources), draft generation (RAG-based writing with citations), and research agent (autonomous investigation using all tools).

2. Which lessons does each component draw from?

Search: Lessons 05 (embeddings) + 06 (RAG retrieval). Fact-check: Lessons 08 (structured extraction) + 06 (evidence retrieval). Drafts: Lesson 06 (RAG generation). Agent: Lesson 07 (ReAct loop) + 04 (tool dispatch). Server: Lessons 03 (API) + 09 (streaming). Quality: Lesson 10 (evals). Ethics: Lesson 11 (responsible deployment).

3. Why does the agent need a step limit?

Without a limit, a confused agent could loop indefinitely, making API calls and spending money without making progress. The limit forces the agent to work efficiently and report partial results if it can't complete the task.

4. What's the most important ethical safeguard in this tool?

The system never presents unverified information as fact. All claims cite specific sources. The fact-checker explicitly flags claims it cannot verify. And the tool is designed to assist journalists, not replace their judgment.
