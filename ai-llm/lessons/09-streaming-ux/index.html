09 Streaming UX — Making AI Feel Fast

← Back to lessons

Users don't mind waiting if they can see progress. A 5-second spinner feels slow. Watching text appear word by word feels fast, even if it takes the same 5 seconds.

Mental Model: The Teleprinter

Old newsrooms had teleprinters — machines that received wire stories character by character. You could read the story as it came in. You didn't wait for the whole thing to arrive. Streaming AI responses works exactly the same way: show each token as it's generated.

Perceived performance matters more than actual performance. Streaming doesn't make the model faster. It makes the experience faster by reducing time-to-first-token from seconds to milliseconds. Users start reading immediately instead of staring at a spinner.

Server-Side: Streaming from the API

import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

// Method 1: Event-based streaming
const stream = client.messages.stream({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Summarize the key issues in local journalism." }]
});

stream.on("text", (text) => {
  process.stdout.write(text);  // Each chunk as it arrives
});

stream.on("message", (message) => {
  console.log("\n\nDone. Tokens:", message.usage);
});

// Method 2: Async iterator (cleaner for some patterns)
const stream2 = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [{ role: "user", content: "..." }],
  stream: true
});

for await (const event of stream2) {
  if (event.type === "content_block_delta" && event.delta.type === "text_delta") {
    process.stdout.write(event.delta.text);
  }
}

Server-Sent Events (SSE): Streaming to the Browser

To stream AI responses to a web page, you need a bridge: your server streams from Claude's API and re-streams to the browser using Server-Sent Events.

// Express server endpoint
import express from "express";
import Anthropic from "@anthropic-ai/sdk";

const app = express();
const client = new Anthropic();

app.post("/api/chat", async (req, res) => {
  // Set SSE headers
  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");
  res.setHeader("Connection", "keep-alive");

  const stream = client.messages.stream({
    model: "claude-sonnet-4-6",
    max_tokens: 1024,
    system: "You are a helpful newsroom assistant.",
    messages: req.body.messages
  });

  stream.on("text", (text) => {
    // SSE format: "data: ...\n\n"
    res.write(`data: ${JSON.stringify({ text })}\n\n`);
  });

  stream.on("message", (message) => {
    res.write(`data: ${JSON.stringify({ done: true, usage: message.usage })}\n\n`);
    res.end();
  });

  stream.on("error", (error) => {
    res.write(`data: ${JSON.stringify({ error: error.message })}\n\n`);
    res.end();
  });
});
// Browser-side: consuming the SSE stream
async function streamChat(messages, onChunk, onDone) {
  const response = await fetch("/api/chat", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ messages })
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = "";

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split("\n\n");
    buffer = lines.pop();  // Keep incomplete chunk in buffer

    for (const line of lines) {
      if (line.startsWith("data: ")) {
        const data = JSON.parse(line.slice(6));
        if (data.done) {
          onDone(data.usage);
        } else if (data.text) {
          onChunk(data.text);
        }
      }
    }
  }
}

// Usage
const outputEl = document.getElementById("response");
streamChat(
  [{ role: "user", content: "What's happening at city hall?" }],
  (chunk) => { outputEl.textContent += chunk; },
  (usage) => { console.log("Done:", usage); }
);

UX Patterns for Streaming

1. Typing Indicator

// Show a blinking cursor while streaming
const cursor = document.createElement("span");
cursor.className = "typing-cursor";  // CSS: animation: blink 1s infinite
outputEl.appendChild(cursor);

streamChat(messages,
  (chunk) => {
    // Insert text before the cursor
    cursor.insertAdjacentText("beforebegin", chunk);
  },
  () => {
    cursor.remove();  // Remove cursor when done
  }
);

2. Progressive Markdown Rendering

// Render markdown as it streams in
import { marked } from "marked";

let fullText = "";
streamChat(messages,
  (chunk) => {
    fullText += chunk;
    outputEl.innerHTML = marked.parse(fullText);
    // Scroll to bottom
    outputEl.scrollTop = outputEl.scrollHeight;
  },
  () => { console.log("Stream complete"); }
);

3. Skeleton Loading

// Show a skeleton before streaming starts
outputEl.innerHTML = `
  <div class="skeleton-line" style="width: 80%"></div>
  <div class="skeleton-line" style="width: 60%"></div>
  <div class="skeleton-line" style="width: 70%"></div>
`;

// Replace with actual content on first chunk
let started = false;
streamChat(messages,
  (chunk) => {
    if (!started) {
      outputEl.innerHTML = "";  // Clear skeleton
      started = true;
    }
    outputEl.textContent += chunk;
  },
  () => {}
);

Accessibility note: Screen readers may struggle with rapidly changing text content. Consider adding an aria-live="polite" attribute to the output container and debouncing updates. For critical journalism tools, always provide a "copy full response" button after streaming completes.

Streaming with Tool Use

When streaming, tool use events come through as special event types:

const stream = client.messages.stream({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  tools: [...],
  messages: [{ role: "user", content: "..." }]
});

stream.on("text", (text) => {
  // Regular text output
  outputEl.textContent += text;
});

stream.on("contentBlock", (block) => {
  if (block.type === "tool_use") {
    // Show the user what the model is doing
    statusEl.textContent = `Searching: ${block.name}(${JSON.stringify(block.input)})...`;
  }
});

// After the stream ends, check if you need to execute tools
// and continue the conversation

UX opportunity: When the model calls a tool, show the user what's happening: "Searching article archive..." or "Looking up public records..." This builds trust and makes the AI feel more like a collaborator than a black box.

Check Your Understanding

1. Why does streaming make AI feel faster even though the total response time is the same?

Time-to-first-token drops from seconds (waiting for full response) to milliseconds (first token arrives quickly). Users start reading immediately instead of staring at a spinner. Perceived performance is about first visual change, not total completion time.

2. What are Server-Sent Events (SSE)?

A web standard for one-way server-to-client streaming over HTTP. The server sends events in "data: ...\n\n" format. The browser reads them with a ReadableStream or EventSource. SSE is simpler than WebSockets and works over normal HTTP.

3. Why do you need a server in the middle (can't the browser call Claude's API directly)?

Your API key would be exposed in browser JavaScript, which is a security risk. The server acts as a proxy: the browser sends the request to your server, your server calls Claude's API with the key, and streams the response back. The key never leaves the server.

4. What should you show the user when the model is calling a tool during a streamed response?

A status message about what the model is doing: "Searching articles..." or "Looking up records..." This builds trust, shows progress, and makes the AI feel like a transparent collaborator rather than a black box.
