This is not a course you watch. It's one you build inside of. Every lesson was designed around one question: how do you go from "I use Claude" to "I build with Claude"?

The rule: mental models first, syntax second. You won't memorize API signatures. You'll learn the why behind every pattern so you can derive the how on the fly.

What You'll Be Able to Do

After 12 lessons, you should be able to:

Explain how LLMs actually work

Tokens, attention, context windows, temperature — not just vocabulary, but real understanding of what happens when you hit "send."

Build AI-powered applications

From single API calls to multi-turn agents with tool use, RAG pipelines, and streaming UX.

Write prompts that actually work

System prompts, few-shot patterns, chain-of-thought, structured output. Engineering, not guessing.

Ship a journalism AI tool

The capstone: an AI-powered newsroom assistant that searches sources, checks facts, and drafts stories with citations.

How Lessons Work

Each lesson follows the same structure:

  1. Mental model first. An analogy or concept that makes the new idea click before you see any code.
  2. Core concepts. The 2–3 things you need to understand deeply. Explained through comparisons to what you already know — React, JavaScript, journalism workflows.
  3. Code examples. Real Anthropic SDK (JavaScript) code. Short, focused, with inline explanations.
  4. Side-by-side comparisons. The wrong way vs. the right way. Naive approach vs. production approach.
  5. Quiz. Quick checks: "Can you explain this back?" Click to reveal answers.
  6. Exercise. Something to build. Small enough to finish in one sitting, meaty enough to prove understanding.

Every page also has an AI Tutor in the corner. It knows which lesson you're on, what concepts you've covered, and can explain things differently if you're stuck.

The Mental Models

AI is hard to learn because the marketing is louder than the engineering. These analogies cut through the hype:

  • An LLM is a very sophisticated autocomplete — it predicts the next token based on everything before it. Not magic, not sentient. Pattern matching at superhuman scale.
  • A prompt is a briefing document — the same way you'd brief a new reporter on a story. Context, constraints, examples, desired output format.
  • Tool use is giving the model a phone and a rolodex — it can't browse the web itself, but you can let it call functions that do.
  • RAG is an open-book exam — instead of relying on what the model memorized during training, you hand it the relevant pages right before it answers.
  • An agent is an intern with a checklist — it reasons about what to do next, does it, checks the result, and decides the next step. Loop until done.
  • Embeddings are GPS coordinates for meaning — similar ideas are close together in vector space, even if the words are completely different.
  • Evals are unit tests for vibes — you can't assert exact output, but you can measure whether the model is getting better or worse across many examples.

Why This Order

The curriculum is designed as a stack where each layer depends on the one below:

  1. Understand the machine (Lesson 01) — you can't prompt well if you don't know what a token is.
  2. Talk to it well (Lesson 02) — prompting is the foundation of every AI application.
  3. Call it from code (Lessons 03–04) — API + tools turn prompts into applications.
  4. Give it knowledge (Lessons 05–06) — embeddings + RAG solve the "it makes stuff up" problem.
  5. Let it act (Lesson 07) — agents chain tools and reasoning into autonomous workflows.
  6. Make it reliable (Lessons 08–10) — structured output, streaming, and evals make it production-ready.
  7. Use it responsibly (Lesson 11) — ethics and journalism-specific considerations.
  8. Build the thing (Lesson 12) — the capstone puts it all together.

Prerequisites

You should be comfortable with:

  • JavaScript fundamentals — async/await, objects, arrays, destructuring, ES modules.
  • Node.js basics — npm install, running scripts, environment variables.
  • The command line — navigating directories, running commands, reading output.

You do not need prior experience with AI, machine learning, Python, or data science. That's what we're here for.
