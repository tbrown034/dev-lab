You already know Tailwind. That's an advantage, not a crutch. This course goes beneath the utilities to the engine that powers them. By the end, you'll think in CSS the way a systems architect thinks in abstractions.

Why This Track Exists

Most developers learn CSS backwards. They memorize properties, hack at layouts until they work, and never build a mental model of why CSS behaves the way it does. This course fixes that.

You'll learn CSS the same way an investigative journalist learns a beat: understand the system first, then work within it. Every lesson starts with the mental model, then shows you the syntax that expresses it.

The Tailwind advantage. You already know what flex, grid-cols-3, and bg-zinc-900 do visually. Now you'll learn the CSS spec that makes them work. When you read justify-between in Tailwind, you'll understand the flexbox alignment algorithm it triggers. That's the difference between using a tool and understanding it.

What You'll Build

This isn't a "make pretty things" course. You'll build systems:

  • A complete design token architecture — primitive, semantic, and component-level tokens that power multi-brand themes
  • A component CSS system — reusable patterns using modern CSS features (container queries, :has(), nesting)
  • Broadcast-ready TV graphics — lower thirds, election tickers, score bugs, and alert banners built entirely in CSS
  • A capstone design system — everything combined into a production-grade system from scratch

How Lessons Work

Every lesson follows the same rhythm:

  1. Mental model first — We start with the "why" and the conceptual framework. No code yet.
  2. Syntax second — Once you understand the model, the properties and values map onto it naturally.
  3. Side-by-side comparisons — See the wrong way and the right way next to each other. See the CSS and the Tailwind equivalent.
  4. Quizzes — Check your understanding of the mental model, not just syntax recall.
  5. Exercises — Build something that forces you to apply the concept.

Tailwind bridges. Throughout the course, you'll see callouts that connect CSS concepts to Tailwind utilities you already know. When we talk about justify-content: space-between, we'll note that's justify-between in Tailwind. This builds connections in both directions.

The Broadcast Connection

Lessons 11 and 12 focus on something unusual: CSS for television broadcast graphics. Lower thirds, election night overlays, live tickers, and score bugs.

This isn't a gimmick. Broadcast CSS is where design system principles get stress-tested under the hardest constraints:

  • Safe zones — Content must stay within title-safe and action-safe areas
  • Overscan — TVs crop the edges differently than monitors
  • 60fps or nothing — Animations must be butter-smooth on live television
  • Pixel-perfect alignment — Lower thirds must line up exactly when switching between different reporters
  • Theme switching — Election night means switching between candidate colors in real time

If your design system can handle broadcast TV, it can handle anything. The constraints of live television force you to build truly robust, token-driven, performant CSS. The skills transfer directly to any production web application.

Key Mental Models You'll Develop

By the end of this course, you'll think about CSS differently. Here are the mental models we'll build:

  1. CSS is a rules engine, not a programming language — It resolves conflicts, it doesn't execute instructions
  2. Layout modes are universes — Block, flex, and grid each have their own physics. Properties that work in one may not work in another.
  3. Custom properties are a runtime API — They cascade, they scope, they change dynamically. Sass variables compile away.
  4. Tokens are the single source of truth — Colors, spacing, typography all flow from one system
  5. Components are contracts — A component's CSS defines what can vary and what's fixed
  6. Motion has meaning — Every animation should communicate something, not just look cool
