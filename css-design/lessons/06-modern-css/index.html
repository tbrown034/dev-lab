06 Modern CSS — The New Power Features

← Back to lessons

CSS has changed more in the last 3 years than the previous 10. Container queries, :has(), nesting, @layer, and @scope transform CSS from a styling language into a design system language. These features obsolete many JavaScript workarounds.

Container Queries: Components That Respond to Their Container

Media queries respond to the viewport. Container queries respond to the parent element's size. This is the feature that makes truly reusable components possible.

The problem container queries solve

/* Media query approach — the component knows about the page layout */
@media (min-width: 768px) {
  .card { flex-direction: row; }
}
/* PROBLEM: The card doesn't care about the viewport width.
   It cares about how much space IT has.
   A card in a sidebar should be vertical even on a wide screen.
   A card in the main content area should be horizontal. */

/* Container query approach — the component responds to its own space */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card { flex-direction: row; }
}
/* NOW: The card goes horizontal when its container is > 400px,
   regardless of viewport width. Same card in sidebar = vertical.
   Same card in main = horizontal. TRUE component reusability. */

Container queries make components truly portable. A card built with container queries works in a sidebar, a main area, a modal, or a dashboard widget — adapting its layout to whatever space it's given. Media queries tie components to the page; container queries free them.

Container query syntax

/* Step 1: Mark the container */
.wrapper {
  container-type: inline-size;  /* track width changes */
  container-name: card-area;    /* optional name for targeting */
}

/* Shorthand */
.wrapper { container: card-area / inline-size; }

/* Step 2: Query the container */
@container card-area (min-width: 600px) {
  .card { grid-template-columns: 200px 1fr; }
}
@container card-area (min-width: 400px) {
  .card-title { font-size: 1.5rem; }
}

/* Container query units (relative to container size) */
.card-title {
  font-size: clamp(1rem, 3cqi, 2rem);
  /* cqi = 1% of container's inline size */
  /* cqb = 1% of container's block size */
}

:has() — The Parent Selector CSS Never Had

For 20 years, CSS couldn't style a parent based on its children. :has() changes that. It's the most powerful selector ever added to CSS.

/* Style a card differently if it contains an image */
.card:has(img) {
  grid-template-rows: 200px 1fr;
}

/* Style a form group when its input is focused */
.form-group:has(:focus) {
  border-color: var(--accent);
  box-shadow: 0 0 0 3px rgba(168, 85, 247, 0.15);
}

/* Style a label when its sibling checkbox is checked */
label:has(+ input:checked) {
  color: var(--accent);
  font-weight: 600;
}

/* Style body when a modal is open (no JavaScript class needed!) */
body:has(.modal[open]) {
  overflow: hidden;
}

/* Conditional grid: 2 columns if there are enough items */
.list:has(:nth-child(4)) {
  grid-template-columns: 1fr 1fr;
  /* Only use 2 columns when there are 4+ items */
}

:has() eliminates entire categories of JavaScript. Toggle classes on parents when children change? :has() does it in CSS. Conditional layouts based on content? :has(). Form validation styling? :has(:invalid). Every time you'd add a JavaScript event listener just to toggle a parent class, try :has() first.

CSS Nesting: Native Sass-like Syntax

CSS now supports nesting natively. No preprocessor needed.

/* Before nesting */
.card { background: var(--surface); }
.card .title { font-weight: 600; }
.card .title:hover { color: var(--accent); }
.card .body { color: var(--text-dim); }

/* With native CSS nesting */
.card {
  background: var(--surface);

  .title {
    font-weight: 600;

    &:hover {
      color: var(--accent);
    }
  }

  .body {
    color: var(--text-dim);
  }

  /* Nesting media queries */
  @media (min-width: 768px) {
    grid-template-columns: 200px 1fr;
  }
}

/* The & is optional for class/element selectors in modern browsers.
   It's required for pseudo-classes and combinators:
   &:hover, &::before, & + .sibling */

Nesting vs specificity. Unlike Sass, CSS nesting doesn't change specificity by much. .card .title is (0, 2, 0) whether written flat or nested. The key difference from Sass: native nesting uses :is() internally for some patterns, which can affect specificity if you nest inside a list selector. In practice, keep nesting to 2-3 levels deep max.

@layer: Cascade Control (Recap + Advanced)

We introduced @layer in Lesson 01. Here's the advanced usage for design systems:

/* Declare layer order globally (often in your entry CSS file) */
@layer reset, base, tokens, components, utilities, overrides;

/* Import stylesheets into specific layers */
@import url("reset.css") layer(reset);
@import url("tokens.css") layer(tokens);
@import url("components.css") layer(components);

/* Tailwind v4 uses layers internally:
   @layer theme, base, components, utilities;
   This is why utilities always beat components —
   layer order, not specificity. */

/* Unlayered CSS beats ALL layers */
/* This is a powerful escape hatch for one-off overrides */
.page-specific-fix { color: red; }
/* This beats any layered rule, regardless of specificity */

The unlayered gotcha. Any CSS NOT wrapped in a @layer has higher priority than ALL layered CSS. This is intentional — it's an escape hatch. But if you're building a design system with layers, make sure all your CSS is in a layer, or stray unlayered rules will override everything.

@scope: Style Boundaries

@scope limits where styles apply — both where they start (scope root) and where they stop (scope limit). This prevents styles from leaking into nested components.

/* Styles apply from .card down, but stop at .card-actions */
@scope (.card) to (.card-actions) {
  p { color: var(--text-dim); }
  a { color: var(--accent); text-decoration: underline; }
  /* These styles WON'T affect <p> or <a> inside .card-actions */
}

/* Scope without a limit — just a starting point */
@scope (.dashboard) {
  .widget { border-radius: 12px; }
  .chart { aspect-ratio: 16/9; }
}

/* The :scope pseudo-class refers to the scope root */
@scope (.card) {
  :scope { border: 1px solid var(--border); }  /* targets .card itself */
  :scope > h2 { color: var(--accent); }
}

@scope vs BEM vs CSS Modules. All three solve the "style leaking" problem differently. BEM uses naming conventions (.card__title). CSS Modules use build-tool-generated unique class names. @scope solves it in native CSS with actual style boundaries. For design systems, @scope is the cleanest approach — no conventions to enforce, no build tools required.

Combining Modern Features

The real power emerges when you combine these features:

/* A component that:
   - Responds to its container size (container queries)
   - Styles its parent based on children (:has)
   - Uses clean nesting
   - Has scoped styles (@scope)  */

.card-wrapper {
  container: card / inline-size;
}

@scope (.card) {
  :scope {
    display: grid;
    gap: 1rem;
    padding: var(--card-padding, 1.5rem);

    /* Has an image? Show it in a grid row */
    &:has(img) {
      grid-template-rows: 200px auto;
    }

    /* Has a badge? Add extra padding top */
    &:has(.badge) {
      padding-top: 2rem;
    }
  }
}

/* Card responds to container size */
@container card (min-width: 500px) {
  .card:has(img) {
    grid-template-columns: 250px 1fr;
    grid-template-rows: auto;
  }
}

Check Your Understanding

1. What's the difference between a media query and a container query?

Media queries respond to the viewport (window) size. Container queries respond to a parent element's size. Container queries make components portable because they adapt to their local context, not the page layout.

2. Write a CSS rule that styles a form differently when it contains an invalid input.

form:has(:invalid) { border-color: var(--error); }. The :has() selector checks if the form contains any descendant matching :invalid.

3. Why does unlayered CSS beat all layered CSS?

By design, unlayered CSS has higher cascade priority than any layer. This serves as an escape hatch for one-off overrides. It's intentional, but can cause surprises if you're not aware of it.

4. What does @scope (.card) to (.card-footer) do?

Styles inside the @scope block apply to elements inside .card but stop at .card-footer. Elements inside .card-footer (and its descendants) are not affected by the scoped styles.

5. What is cqi as a CSS unit?

cqi is 1% of the container's inline size (width in horizontal writing mode). It's a container query unit that lets you size elements relative to their container, similar to how vw sizes relative to the viewport.

  2. At 500px container width, switch to horizontal layout (image left, text right)
  3. Use :has() to only apply the image layout when an image is present
  4. Test it by placing the same card in a sidebar (narrow) and main area (wide)
  5. Bonus: Use container query units (cqi) for the card title font size

Recap

  1. Container queries make components respond to their container, not the viewport
  2. :has() is the parent selector — style ancestors based on descendants
  3. CSS nesting is now native — no preprocessor needed
  4. @layer controls cascade order explicitly — layer order beats specificity
  5. @scope creates style boundaries — prevents leaking without naming conventions
  6. Combined, these features enable CSS-native component systems that rival CSS-in-JS
