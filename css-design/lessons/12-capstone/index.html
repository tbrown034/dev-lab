This is the final project. You'll build a complete, broadcast-ready design system from scratch. Tokens, components, themes, animations, and live broadcast overlays — everything from this course, combined into one system.

The Project: "Studio" Design System

You're building a design system called Studio. It powers a fictional broadcast news network that covers news, sports, and elections. The system must support:

  • 3 brands — News (red/orange), Sports (green/emerald), Elections (blue/purple)
  • 2 themes — Dark (on-air) and Light (web/mobile)
  • Core UI components — buttons, cards, inputs, badges
  • Broadcast components — lower thirds, ticker, score bug, results bar
  • Smooth animations — all broadcast-quality (60fps, transform/opacity only)
  • Safe zone compliance — all broadcast elements within title-safe boundaries

This project proves you understand CSS at a systems level. It's not about making pretty things — it's about building an architecture that's maintainable, themeable, performant, and production-ready. This is the kind of work that gets you hired at design-driven companies.

  --ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);

/* tokens/semantic.css */
/* Map primitives to purposes. Theme-switchable. */

:root {
  /* Surfaces */
  --color-bg: var(--gray-950);
  --color-surface: var(--gray-900);
  --color-surface-elevated: var(--gray-800);

  /* Text */
  --color-text: var(--gray-50);
  --color-text-secondary: var(--gray-400);
  --color-text-muted: var(--gray-600);

  /* Actions & Feedback */
  --color-action: var(--blue-500);
  --color-success: var(--green-500);
  --color-error: var(--red-500);
  --color-warning: var(--amber-400);

  /* Borders */
  --color-border: var(--gray-800);
}

/* Light theme override */
[data-theme="light"] {
  --color-bg: var(--gray-50);
  --color-surface: white;
  --color-text: var(--gray-900);
  ...
}

/* Brand overrides */
[data-brand="news"] { --color-action: var(--red-600); }
[data-brand="sports"] { --color-action: var(--green-600); }
[data-brand="elections"] { --color-action: var(--blue-600); }

Start here. Everything else depends on tokens. Get this right and the rest follows naturally. Get it wrong and you'll be fighting overrides everywhere.

Phase 4: Theme Engine

Wire everything together so the entire system responds to theme and brand changes.

/* The theme engine: ONE data attribute changes everything */
<body data-brand="news" data-theme="dark">
  <!-- All components automatically use news brand, dark theme -->
</body>

/* JavaScript: switch brands */
document.body.dataset.brand = 'sports';
/* Every component updates: buttons turn green, lower thirds turn green,
   tickers turn green, score bugs use sports styling. Zero CSS changes. */

/* JavaScript: switch themes */
document.body.dataset.theme = 'light';
/* Every surface, text color, border color updates.
   Brand colors stay the same. Components adapt. */
