04 CSS Grid Mastery — Two Dimensions

← Back to lessons

Grid is the first CSS layout system designed for two dimensions. Rows AND columns, simultaneously. Where flexbox distributes items along a line, grid places items on a plane.

The Mental Model: A Spreadsheet

A CSS Grid container is like a spreadsheet. You define the rows and columns, then place items into cells. Items can span multiple cells. The grid tracks (rows/columns) can be fixed, flexible, or content-sized.

/* Define a 3-column, 2-row grid */
.grid {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr;
  gap: 1rem;
}

/*
   ┌──200px──┬────1fr────┬──200px──┐
   │ sidebar │  header   │  aside  │  ← auto height (fits content)
   ├─────────┼───────────┼─────────┤
   │ sidebar │   main    │  aside  │  ← 1fr height (fills remaining)
   └─────────┴───────────┴─────────┘
*/

The key difference from flexbox. In flex, items flow in one direction and you control that flow. In grid, you define the structure first (the lines), then place items into it. Grid is structure-first; flex is content-first.

The fr Unit: Fractional Space

fr stands for "fraction of remaining space." It's grid's killer feature for flexible layouts:

/* Three equal columns */
grid-template-columns: 1fr 1fr 1fr;

/* Sidebar (fixed) + main (flexible) + aside (fixed) */
grid-template-columns: 250px 1fr 200px;

/* Main gets 2x the flexible space as sidebar */
grid-template-columns: 1fr 2fr;

/* How fr works:
   Total space: 900px
   Fixed: 250px + 200px = 450px
   Remaining: 900 - 450 = 450px
   1fr = 450px (all to the middle column) */

Tailwind connection. grid-cols-3 becomes grid-template-columns: repeat(3, minmax(0, 1fr)). The minmax(0, 1fr) prevents items from overflowing their columns (Tailwind is smarter than just 1fr). Custom column sizes use arbitrary values: grid-cols-[250px_1fr_200px].

repeat(), auto-fit, and auto-fill

repeat() — shorthand for repeated tracks

/* Instead of: 1fr 1fr 1fr 1fr */
grid-template-columns: repeat(4, 1fr);

/* Mix repeat with fixed: sidebar + 3 equal columns */
grid-template-columns: 250px repeat(3, 1fr);

auto-fill vs auto-fit — responsive grids without media queries

/* auto-fill: create as many columns as fit, even if empty */
grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));

/* auto-fit: like auto-fill, but collapse empty tracks */
grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
auto-fill (keeps empty tracks)
/* 3 items in a 1200px container:

   [item][item][item][    ][    ]

   Empty tracks remain, items
   DON'T stretch to fill. */
auto-fit (collapses empty tracks)
/* 3 items in a 1200px container:

   [  item  ][  item  ][  item  ]

   Empty tracks collapse, items
   STRETCH to fill all space. */

The responsive card grid one-liner. grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)) creates a responsive grid that adjusts columns automatically — no media queries needed. This single line replaces 20+ lines of media query breakpoints.

Placing Items on the Grid

Grid lines (numbered from 1)

/* Grid lines are numbered starting at 1 */
/*  col 1   col 2   col 3   col 4 (line numbers)
     |       |       |       |
     v       v       v       v
     ┌───────┬───────┬───────┐
     │       │       │       │
     └───────┴───────┴───────┘
*/

.header {
  grid-column: 1 / -1;  /* span all columns (-1 = last line) */
  grid-row: 1;
}

.sidebar {
  grid-column: 1 / 2;   /* first column */
  grid-row: 2 / 4;      /* span rows 2-3 */
}

/* Shorthand with span */
.featured {
  grid-column: span 2;  /* take up 2 columns from wherever placed */
}

Named grid areas — the readable approach

.layout {
  display: grid;
  grid-template-columns: 250px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header  header"
    "sidebar main    aside"
    "footer  footer  footer";
  gap: 1rem;
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.aside   { grid-area: aside; }
.footer  { grid-area: footer; }

/* This reads like a visual blueprint of the page.
   Want to move the sidebar to the right?
   Just swap the names in the template. */

Named areas are self-documenting. When someone reads grid-template-areas, they see the layout immediately. No counting grid lines. This is invaluable on teams and in design systems where layout intent needs to be obvious.

Subgrid: Alignment Across Nesting

Subgrid solves the "nested alignment" problem. Without it, a card grid where each card has a title, body, and footer can't align those internal parts across cards:

/* The problem: each card has its own internal grid.
   If Card A's title is longer, its body starts lower than Card B's body. */

/* The solution: subgrid */
.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: subgrid;  /* Wait — this goes on the parent? No. */
}

/* Actually: subgrid goes on the CHILD that spans parent tracks */
.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-auto-rows: auto auto 1fr auto;  /* 4 rows per card */
}

.card {
  display: grid;
  grid-row: span 4;
  grid-template-rows: subgrid;  /* inherit parent's row tracks */
}

/* Now all card titles align, all bodies align, all footers align
   — even if content lengths differ */

Subgrid is the holy grail of card layouts. Before subgrid, you couldn't align internal card parts across a grid without JavaScript or fixed heights. Now a single grid-template-rows: subgrid makes the card inherit its parent's row tracks. Titles line up. Footers line up. Content fills the middle.

Grid vs Flex: The Decision

Use Flexbox when...
• Layout is one-dimensional
  (row OR column)

• Content drives the layout
  (items size themselves)

• You need items to wrap
  naturally like text

• It's a component-level
  layout (navbar, button
  group, card row)

• You want items to "flex"
  and share space along
  one axis
Use Grid when...
• Layout is two-dimensional
  (rows AND columns)

• Structure drives the layout
  (you define the grid first)

• Items need to align in
  both directions

• It's a page-level layout
  (sidebar + main + aside)

• You need items to span
  multiple rows AND columns
  (dashboards, magazines)

They work together. A common pattern: grid for the page layout, flex for component-level layouts within grid cells. The nav is flexbox inside a grid cell. The card content is flexbox inside a grid item. Don't choose one — use both.

Common Grid Patterns

Pattern 1: Responsive card grid

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 1.5rem;
}
/* Tailwind: grid grid-cols-[repeat(auto-fit,minmax(300px,1fr))] gap-6 */

Pattern 2: Holy grail layout

.page {
  display: grid;
  grid-template: "header header" auto
                 "sidebar main" 1fr
                 "footer footer" auto / 250px 1fr;
  min-height: 100vh;
}
/* grid-template combines areas, rows, and columns in one */

Pattern 3: Overlap without absolute positioning

.hero {
  display: grid;
}
.hero > * {
  grid-area: 1 / 1;  /* all children occupy the same cell */
}
.hero-image { z-index: 1; }
.hero-text  { z-index: 2; align-self: end; }
/* Stack elements without position: absolute. Clean layering. */

Pattern 4: Dense auto-placement (masonry-like)

.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  grid-auto-flow: dense;  /* fill gaps with smaller items */
}
.gallery .wide { grid-column: span 2; }
.gallery .tall { grid-row: span 2; }

Check Your Understanding

1. What's the difference between auto-fill and auto-fit?

auto-fill creates as many columns as fit and keeps empty tracks. auto-fit collapses empty tracks so items stretch to fill all available space. With fewer items than columns, auto-fit makes items wider.

2. What does grid-column: 1 / -1 mean?

Span from the first grid line to the last grid line — i.e., take up the full width of the grid. -1 means "the last line" (like negative indexing in Python).

3. When would you use subgrid?

When nested elements need to align with their parent's grid tracks. The classic case: card grids where titles, bodies, and footers should align across all cards regardless of content length.

4. What does 1fr actually mean?

One fraction of the remaining available space after fixed-size tracks are subtracted. If you have 200px 1fr 1fr in a 1000px container, each 1fr gets (1000 - 200) / 2 = 400px.

5. How do you overlap two elements using grid (no position: absolute)?

Place both children in the same grid area: grid-area: 1 / 1 on both. They'll stack, and you can control z-order with z-index. This is cleaner than absolute positioning because the parent still sizes to its content.

Recap

  1. Grid = two-dimensional — rows AND columns simultaneously
  2. fr distributes remaining space after fixed tracks are laid out
  3. auto-fit collapses empty tracks, auto-fill preserves them
  4. Named areas make layouts self-documenting and easy to rearrange
  5. Subgrid aligns nested content across parent grid tracks
  6. Grid for page layout, flex for component layout — use both
  7. Grid overlap replaces many absolute positioning hacks
  8. repeat(auto-fit, minmax(min, 1fr)) is the responsive one-liner
