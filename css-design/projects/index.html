Broadcast Design System

A complete lower-third graphics system with multiple variants for news, sports, and weather. Built with CSS custom properties for instant re-theming across show brands.

How I Built This

  • All variants share a base .lt-bar component, themed via --lt-primary, --lt-secondary, and --lt-text custom properties
  • Slide-up entrance uses @keyframes ltSlideUp with translateY(100%) to translateY(0), simulating broadcast reveal timing
  • Breaking news variant adds an animated red banner using @keyframes pulse for the urgency indicator
  • Sports variant uses flexbox with fixed-width team badges and a fluid game-clock section
  • The system exports as a CSS file with layer ordering: @layer base, variants, overrides
  • All lower thirds scale via clamp() typography for screen sizes from mobile preview to 4K broadcast output

How I Built This

  • Seamless loop by duplicating content and using translateX(-50%) in the keyframe animation
  • Speed variants controlled by --ticker-speed custom property on the container
  • Pause-on-hover uses animation-play-state: paused on the :hover state
  • Breaking news mode increases speed and adds a red background pulse animation to the label
  • Stock ticker items use conditional coloring with .up / .down utility classes
  • The track uses overflow: hidden with absolutely-positioned content for the scroll illusion

How I Built This

  • Vote bars use percentage-based widths with transition: width 1s ease for live counting animation

  • @media print styles convert the package to a black-and-white friendly format for press handouts
Project 04

Multi-brand Token System

A CSS design token architecture supporting multiple broadcast brands (news, sports, entertainment) from a single codebase. Custom properties cascade from brand to component level.

custom properties @layer :root data attributes cascade layers

How I Built This

  • Three-tier token architecture: global → brand → component, each in its own @layer
  • Brand switching via [data-brand="news"], [data-brand="sports"], etc. on the root element
  • Components reference semantic tokens like var(--brand-primary) instead of raw values
  • Spacing scale uses a modular formula: --space-1 through --space-8 with a 1.25 ratio
  • Typography tokens include --font-heading, --font-body, --font-mono per brand
  • The system exports a tokens.css file consumed by Tailwind v4, standard CSS, or JavaScript tooling
